## 0. 이 페이지의 성격 (설계 전제)

1. **YMYL 등급 = 중간(YMYL 인접).** 연봉 실수령액(#5)과 동급이고 근속연수(#7, YMYL-낮음)보다
   한 단계 높다. 급여·법정 지급액이라 오정보 리스크가 있고 통상임금·부분월 등 단순화한 부분이
   있어(기획 §1-3), **연봉 계산기와 동일하게 상단 amber `role="note"` 근사 고지 배너를 항상 노출**
   한다(§6-1). 결과에 위험/경고 색을 부여하지 않고, 강조는 순수 "값 확인" 목적의 Accent(에메랄드)
   하나로 고정한다.
2. **인터랙션 = 버튼식(실시간 아님) 확정.** 세트 파트너 #7 근속연수 계산기, 선례 #5 연봉 계산기가
   모두 `form onSubmit` + "계산하기" 버튼 방식이다. 퇴직금도 **여러 값을 넣고 한 번 확인하는 단발
   계산**이므로 버튼식이 맞다(사이트 "단발=버튼 / 연속 변환=실시간" 이원화 원칙). → `form`/
   `onSubmit`/"계산하기" 버튼 사용, 실시간 파생 렌더 금지.
3. **입력 5개 · 출력(대표 1 + 보조 4)** — 기획 §7 확정안 그대로. 입력이 date 2개 + 금액 3개로
   연봉(4개)과 비슷하게 많고, 결과 정보량은 연봉(공제 10줄+)보다 적다(대표값 + 보조 4). 그래서
   **연봉처럼 접이식으로 숨길 만큼 많지 않아, 보조 4값을 항상 노출**한다(§3-0 근거). 무스크롤은
   목표가 아니며 연봉과 동일하게 계산 후 결과로 스무스 스크롤한다(§5).
4. **YMYL-중간 CTA 금지(연봉과 동일).** 퇴직·이직·전직을 부추기거나 오인시키는 2차 버튼/문구를
   계산기 카드 안팎에 절대 넣지 않는다(§6-3). "퇴직하면 얼마?", "이직 준비", "받는 법" 류 카피·버튼
   금지. 계산 버튼 1개만 존재.
5. **세트 일관성(#7).** 재직일수 기준이 근속연수 계산기와 완전히 동일(양 끝 포함 +1, 기획 §1-2)
   하므로, 결과 카드의 재직일수·근속 표기를 #7 결과 카드와 **같은 시각 언어**로 맞춘다(총 재직일수
   `N,NNN일` + `N년 M개월`). 관련 계산기 링크는 #7을 반드시 포함(기획 §4 override — 개발/마스터
   확인 사안, 이 문서 범위 밖이나 §7에 재확인).

## 2. 입력 설계 (date 2 + 금액 3 + 계산 버튼)

### 2-1. 배치 원칙 — **2개 그룹으로 시각 분절 + 금액 필드 그룹핑**

입력 5개를 **의미 단위 2그룹**으로 나눠 소제목(구분선 + 작은 라벨)으로 분절한다. 필드가 5개로
많아 한 덩어리로 쌓으면 "무엇을 어디까지 넣어야 하나"가 흐려지므로, "① 재직 기간(날짜 2개) → ②
임금 정보(금액 3개)"로 나눠 입력 흐름을 명확히 한다(계산 파이프라인 순서와도 일치: 재직일수 →
임금총액).

| 그룹 | 소제목(작은 라벨) | 필드 | 배치 |
|---|---|---|---|
| ① 재직 기간 | "재직 기간" | 입사일 / 퇴사일 | 각 전체 폭, 세로 스택(from→to 시간순) |
| ② 임금 정보 | "임금 정보 (세전 기준)" | 3개월 급여 총액(필수) / 연간 상여금(선택) / 연차수당(선택) | 급여 총액 전체 폭, 상여+연차 2열 |

- **그룹 소제목**: `mt-2 text-xs font-semibold text-brand-text-secondary`(연봉 접이식 소그룹 헤더
  "4대 보험"/"세금"과 동일 톤) + 위쪽 얇은 구분선 `border-t border-brand-border/60 pt-3`은 두 번째
  그룹(임금 정보) 앞에만. 첫 그룹(재직 기간) 소제목은 구분선 없이. **과설계 방지**: 별도 카드
  분할이나 접이식이 아니라 소제목 라벨 1줄로만 분절한다(같은 흰 카드 안).
- **금액 필드 그룹핑 근거**: 3개월 급여 총액은 **필수·대표 입력**(억 단위 가능)이라 전체 폭 1줄
  확보(연봉 계산기가 연봉을 전체 폭으로 둔 판단과 동일 — 2열로 묶으면 자릿수가 잘려 보임).
  연간 상여금·연차수당은 **둘 다 선택·기본 0**인 부속 입력이라 2열로 나란히 묶어 "추가 수당 한
  쌍"임을 시각적으로 인지시키고 폼 길이를 줄인다(§5-2에서 375px 자릿수 폭 안전 확인).

### 2-2. 필드별 스펙

design-system 3-1 + 연봉/근속연수의 입력 스타일을 **그대로** 재사용한다. 세 계산기가 동일한
`inputBase` 상수 문자열이므로 그대로 뽑아 쓴다.

`inputBase` = `"h-12 rounded-lg border px-4 text-base text-brand-text outline-none transition-colors
focus:border-brand-primary focus:ring-4 focus:ring-brand-primary/15"` (오류 시 `border-brand-warning`,
평시 `border-brand-border` 토글).

| 순서 | 필드 | id | 라벨(표시) | 컨트롤 | 필수 | placeholder / 기본값 | 도움말(Caption) |
|---|---|---|---|---|---|---|---|
| 1 | 입사일 | `startDate` | **"입사일"** | `type="date"` | 필수 | 미입력 | "재직을 시작한 첫날(첫 근무일)을 선택하세요." |
| 2 | 퇴사일 | `endDate` | **"퇴사일 (마지막 근무일, 미입력 시 오늘)"** | `type="date"` | 선택 | 미입력=오늘 | "마지막 근무일을 선택하세요. 비워 두면 오늘까지 계산합니다. 퇴사 예정일 등 미래 날짜도 입력할 수 있습니다." |
| 3 | 최근 3개월 급여 총액 | `recentPay` | **"최근 3개월 급여 총액 (세전, 원)"** | `type="text"` + `inputMode="numeric"`, 천단위 콤마 | 필수 | placeholder `9,000,000` | "세전 기준, 퇴사일 직전 3개월간 받은 급여 3개월치의 합계입니다." |
| 4 | 연간 상여금 | `annualBonus` | **"연간 상여금 (원)"** | `type="text"` + `inputMode="numeric"`, 천단위 콤마 | 선택 | placeholder `0`, 빈값=0 | "최근 1년간 받은 상여금 (없으면 비워 두세요)." |
| 5 | 연차수당 | `annualLeavePay` | **"연차수당 (원)"** | `type="text"` + `inputMode="numeric"`, 천단위 콤마 | 선택 | placeholder `0`, 빈값=0 | "전년도 발생 미사용 연차수당 (없으면 비워 두세요)." |

- **라벨 매핑 주의(자기검증 §8)**: 표시 라벨은 기획 §7-1 필드명과 1:1 대응한다. 4·5번은 화면에
  단위 `(원)`를 붙였고, "총액/전년도 발생/미사용" 등 의미 한정은 도움말 Caption으로 이관했다(라벨
  길이 축소 목적, 의미 동일). **입력 항목 자체는 5개 그대로, 추가·삭제·치환 없음.**
- **date input `max` 미설정**: 퇴사일 미래(퇴사 예정일) 허용(기획 §1-0). 입사일에도 상한 없음.
  (연봉과 달리 미래 차단 불필요 — 근속연수 계산기와 동일 판단.)
- **금액 3필드 천단위 콤마 = 필수**: 연봉 컴포넌트의 `formatAmountInput(raw)`(숫자만 남기고
  `toLocaleString("ko-KR")`)을 **세 필드 모두에 재사용**. 이 함수는 `[^\d]`(숫자 외)를 제거하므로

## 3. 결과 카드 설계 — [기획 §7-2·§7-3에 대한 디자인팀 확정안]

### 3-0. 정보구조 확정 (대표 1 + 보조 4, **접이식 미도입 → 4값 항상 노출**)

기획 §7-2 출력(대표값 1 = 예상 퇴직금, 보조값 4 = 1일 평균임금 / 산정기간 총일수 / 임금총액 /
재직일수+근속)을 **그대로 채택**한다. 접이식(`<details>`) 도입 여부에 대한 디자인팀 결정:

> **결정: 접이식 미도입. 보조 4값을 항상 노출(Tier ② 단일 블록).**

- **근거 — 정보량이 접이식 문턱 아래다.** 연봉 계산기가 접이식을 쓴 이유는 공제 항목이 8줄+(4대
  보험 4 + 세금 2 + 중간값 2)이라 모바일에서 히어로 아래 10행이 쌓여 대표값 위계가 뭉개지기
  때문이었다(연봉 §3-0). 퇴직금은 보조값이 **4줄뿐**이라, 근속연수 계산기(보조 2값 항상 노출, 접이식
  없음)에 훨씬 가깝다. 4줄은 히어로 아래에 두어도 카드가 콤팩트하다(§5 실측 ~300px대).
- **근거 — 4값이 곧 '계산 근거'라 숨기면 YMYL 신뢰가 떨어진다.** 임금총액·산정기간 총일수·1일
  평균임금은 "예상 퇴직금이 어떻게 나왔는지"의 투명한 근거다. YMYL-중간에서 이 근거를 기본으로
  보여주는 것이 신뢰에 유리하다. 접이식으로 접으면 오히려 "왜 이 금액?"에 답을 감추게 된다.
- **정보 은닉 아님 + 항목 무변경**: 기획 §7-2가 정한 4값을 그대로, 전부, 항상 노출한다. 추가
  항목(상여 3/12 반영액, 퇴직일 D_ret 등)은 **만들지 않는다**(출력 항목 증가 = 기획 이탈 방지).

**3-tier 정리(연봉의 3-tier를 이 계산기 규모에 맞게 축약 — Tier ③ 없음):**

| Tier | 내용 | 강조/배치 | 노출 |
|---|---|---|---|
| **① 대표값** | **예상 퇴직금 `N원`** + **[세전] 배지** | Result 스케일(`text-4xl sm:text-[2.5rem]`, Bold, Accent, tabular-nums). "원"은 `text-base` Text Secondary | 항상 |
| ① 세전 각주 | "세전(퇴직소득세 공제 전) 금액입니다." | `text-xs text-brand-text-secondary` | 항상 |
| ① 요약 문구 | 기획 §2 1문단(동적) 압축 1~2줄 | `mt-2 text-sm text-brand-text-secondary` | 항상 |
| ①-보조 | "오늘 기준" 표기 | `text-xs text-brand-text-secondary` | 퇴사일 미입력 시에만(§3-4 A) |
| **② 보조값 4행** | 임금총액(3개월) / 평균임금 산정기간 총일수 / 1일 평균임금 / 재직일수+근속 | `flex justify-between py-1.5 text-sm`(연봉 `AmountRow`·근속연수 문자열 행 패턴) | 항상 |

- **1년 미만 상태**는 Tier ① 히어로를 금액 대신 안내로 치환한다(§3-4 B, 별도 렌더 분기).

### 3-1. 결과 카드 공통 골격 (연봉·근속연수 결과 카드와 동일)

`mt-6 rounded-xl border-l-4 border-brand-accent bg-gradient-to-br from-emerald-50 to-white
p-4 sm:p-6`. Accent 고정(상태별 색 분기 없음 — 퇴직금 금액에 위험/경고 색 부여 안 함). 계산 후에만
조건부 렌더(`result && (...)`). 갱신 시 `transition-colors duration-200`(연봉 패턴). **`resultRef`로
계산 후 스무스 스크롤**(연봉 패턴 재사용 — 입력 폼이 5필드로 길어 결과를 놓치지 않게, §5-1).

### 3-2. Tier ① — 대표값(예상 퇴직금) 렌더 (항상)

연봉의 단일 금액 히어로(`formatWon` + "원" 작게 분리) 패턴을 그대로 쓴다.

```
예상 퇴직금  [세전]              ← 라벨: text-sm font-medium text-brand-text-secondary
                                   [세전] 배지: ml-2 rounded bg-brand-border/40 px-1.5 py-0.5
                                   text-xs font-medium text-brand-text-secondary
20,562,239 원                     ← 히어로: text-4xl sm:text-[2.5rem] font-bold tabular-nums
                                   text-brand-accent, "원"은 text-base font-medium text-secondary
세전(퇴직소득세 공제 전) 금액입니다.  ← 각주: mt-1 text-xs text-brand-text-secondary
입사일 2019년 1월 1일부터 퇴사일    ← 동적 요약(§3-5): mt-2 text-sm text-brand-text-secondary
2024년 12월 31일까지…예상 퇴직금은
약 2,056만원입니다.
```

- 히어로 컨테이너: `mt-1 flex flex-wrap items-baseline gap-2 tabular-nums`. **`flex-wrap`**: 장기근속
  고소득 극단값(예 `180,000,000원`, 9자리)이 375px 카드 안쪽(약 311px)을 넘을 때 "원" 단위가
  줄바꿈으로 안전하게 흐르도록(가로 오버플로 0). 일반값(8자리 이하)은 한 줄. design-system Result
  스케일 "모바일 28px 이하 축소 금지" 준수(36px OK).
- **[세전] 배지 + 세전 각주 = 이중 세전 명시**(기획 §7-2 "세전임을 배지/각주로 명시"). 배지는
  라벨 옆 인라인, 각주는 히어로 아래 1줄. YMYL-중간 오정보(세후로 오인) 방어의 결과 지점 보강.
- "약"·"원" 단위 텍스트는 히어로보다 작게(Body) Text Secondary — design-system 단위 규칙.

### 3-3. Tier ② — 보조값 4행 (항상)

Tier ① 아래 구분선(`mt-4 pt-4 border-t border-brand-border/60`, 연봉 Tier② 패턴) 다음, 보조 4값을
**계산 흐름 순서**로 나열한다. 원 금액 행은 연봉 `AmountRow`(숫자+원 전용) 재사용, 일수·근속 등
문자열 값 행은 근속연수의 "문자열 값 라벨-값 행"(`flex justify-between`, 값 문자열) 재사용.

```
임금총액 (3개월)            10,500,000 원      ← AmountRow (원)
평균임금 산정기간 총일수     92일               ← 문자열 값 행
1일 평균임금               114,130 원          ← AmountRow (원)
재직일수                   2,192일 (근속 6년 0개월)  ← 문자열 값 행(복합)
```

- 각 행: `flex items-center justify-between py-1.5 text-sm`. 라벨(좌) `text-brand-text-secondary`,
  값(우) `tabular-nums font-semibold text-brand-text`(**Accent 아님** — 컬러 강조는 히어로 전용,
  시선 위계 유지).
- **표시 순서 = 계산 흐름 순**(임금총액 → 산정기간 총일수 → 1일 평균임금 → 재직일수). 사용자가
  "임금총액 ÷ 산정일수 = 1일 평균임금"의 근거를 위에서 아래로 따라 읽게 해 YMYL 투명성을 높인다.
  **이는 기획 §7-2 4항목의 표시 순서 제안일 뿐, 항목의 추가·삭제·치환이 아니다**(§8 자기검증·§9
  기획 확인 요청에 명시). 기획이 원 순서(1일 평균임금 우선)를 원하면 회신만으로 재배열 가능.
- **금액 콤마**: 임금총액·1일 평균임금은 `formatWon`(천단위 콤마) + "원". 1일 평균임금은 기획
  §1-4 앵커 표기가 소수(114,130.43)이나 **결과 카드 표시는 원단위 반올림 정수**(`114,130 원`)로
  통일 권장 — 히어로/보조 모두 정수 원단위라 일관. (엔진 내부는 전 정밀도 유지, 기획 §1-1·§8-5.
  표시 반올림 자릿수 최종은 개발 라운딩 정책 확정 시 고정 — §9 개발 확인.)
- **재직일수 복합 값**: `{재직일수}일 (근속 {N}년 {M}개월)` 한 행. 재직일수는 정수부 천단위 콤마
  (`2,192일`), 근속은 년·개월(일 생략, 근속연수 계산기 히어로는 일까지지만 여기선 보조라 년·개월로
  요약). #7 근속연수 계산기와 **동일 재직일수**임을 사용자가 교차 확인 가능(세트 일관성, §0-5).
  값이 길어도(`2,192일 (근속 6년 0개월)` ≈ 18자) 라벨("재직일수" 4자)이 짧아 375px에서 한 줄에
  들어옴(§5-2). 극단 시 라벨이 줄바꿈되고 값은 우측 유지 → 오버플로 0.

### 3-4. 상태별 UI — (A) 오늘 기준 / (B) 1년 미만 / (C) 입력검증

**(A) "오늘 기준" 표기 (퇴사일 미입력 시):**

- 근속연수 계산기의 `usedToday` caption을 그대로 확장한다.
- **Tier ① 라벨 동적화**: 퇴사일 직접 입력 시 라벨 "예상 퇴직금", **퇴사일 미입력 시 "오늘 기준
  예상 퇴직금"**(기획 §7-2 "오늘(2026년 X월 X일) 기준 예상 퇴직금").
- **caption 1줄**: Tier ① 요약 문구 아래 `text-xs text-brand-text-secondary` —
  "2026년 7월 14일(오늘)을 마지막 근무일로 가정해 계산한 예상 퇴직금입니다."(날짜는
  `formatKoreanDatePlain` 등 기존 유틸 재사용). 퇴사일을 직접 입력한 경우 이 줄을 **숨긴다.**
- 동적 요약 문구(§3-5)도 퇴사일 자리에 "오늘(2026년 7월 14일)"을 자연어로 넣어 caption과 호응.

**(B) 1년 미만 상태 (재직일수 < 365, 기획 §7-2·§1-1 STEP 2):**

금액을 산출하지 않으므로 Tier ① 히어로(에메랄드 대형 숫자)를 렌더하지 않고, **안내 톤 카드**로
치환한다. 사용자를 놀래키지 않도록 위험/경고 색이 아닌 **중립 안내** 톤을 쓴다.

- **카드 골격**: `mt-6 rounded-xl border border-brand-border bg-brand-surface p-4 sm:p-6`
  (에메랄드 `border-l-4`·히어로 없음 — "금액이 나온 결과"와 시각적으로 구분되는 차분한 안내).
  좌측 `ⓘ`(`text-brand-text-secondary`, `aria-hidden`) + 본문 구성:
  ```
  ⓘ 법정 퇴직금 지급 대상이 아닙니다        ← text-base font-semibold text-brand-text (Accent 아님)
     계속근로기간이 1년(365일) 미만입니다.   ← text-sm text-brand-text-secondary
     재직일수  243일 (근속 0년 8개월)         ← 문자열 값 행(재직일수만 표시, 금액 없음)
     법정 퇴직금은 계속근로 1년 이상일 때 지급 대상이 됩니다. 다만 회사 규정에
     따라 1년 미만에도 퇴직금이 별도로 정해진 경우가 있으니 회사 규정을 확인해 주세요.
                                            ← text-xs text-brand-text-secondary (FAQ 3과 일관)
  ```
- **금액(예상 퇴직금·1일 평균임금·임금총액)은 어디에도 표시하지 않는다**(YMYL: 대상 아닌 금액
  노출은 오정보). 재직일수·근속만 노출해 "얼마나 다녔는지"는 확인시켜 준다.
- amber(상단 배너 색)를 재사용하지 않는다 — 상단 배너와 색이 겹쳐 "경고 중첩"으로 읽히지 않게,
  1년 미만은 중립 회색 안내로 둔다.

**(C) 입력검증 에러**: §2-4대로 결과 카드가 아니라 **원인 필드 하단 인라인**으로 처리(퇴사일<입사일,
입사일 미입력, 급여 미입력). 에러 시 결과 카드는 렌더하지 않는다.

### 3-5. Tier ① 요약 문구 = 기획 §2 1문단 동적 배치

기획 §2 1문단(앵커 A 예시: "입사일 2019년 1월 1일부터…예상 퇴직금은 약 2,056만원입니다.")은
정적 텍스트가 아니라 **사용자 입력값 기반 동적 생성**이어야 한다(기획 §8-8, 기존 라이브 계산기
전부 동일). 이 문서는 **배치 위치만 확정**: Tier ① 히어로/세전 각주 바로 아래(`mt-2 text-sm
text-brand-text-secondary`). 연봉·근속연수의 `getResultSummary(...)`처럼 컴포넌트 내부 함수로
입력·결과를 받아 1~2문장 조립. **정적 텍스트 금지.**

- 길이: 모바일 2줄 이내 권장(길면 히어로 위계 방해). 금액은 "약 N,NNN만원"으로 반올림(연봉의
  `formatApprox` 패턴 참고)해 간결하게(상세 숫자는 Tier ②에 있음).
- 퇴사일 미입력이면 퇴사일 자리에 "오늘(2026년 7월 14일)"을 넣고 §3-4(A) caption과 호응.
- **YMYL-중간 카피 가드**: 이 문구에 퇴직·이직·전직을 부추기거나 암시하는 표현 금지(§6-3). 사실
  서술만. 정식 카피 톤은 기획 §2 담당(§9).

## 4. (없음 — 결과 모드 분기 불필요)

연봉·대출과 달리 결과 구조 분기(상환방식·모드 A/B 등)가 없다. 입력이 무엇이든 결과 구조는 §3의
단일 위계(히어로 + 요약 + 보조 4행)로 동일하며, **예외는 1년 미만 안내 카드(§3-4 B) 하나뿐**이다.
(§ 번호는 선행 스펙과 대응 유지 위해 자리만 남김.)

## 5. 모바일(375px) 반응형 규칙

### 5-1. 무스크롤 목표 — 연봉과 동일하게 문자 그대로는 불가, 스무스 스크롤로 절충

iPhone SE급(375×667, 실사용 높이 약 550~600px) 개략 높이 추정.

| 영역 | 예상 높이 |
|---|---|
| 근사 고지 배너(2~3줄, §6-1) | 약 72~92px |
| 그룹① 소제목 "재직 기간" | 약 24px |
| 입사일 필드(라벨+`h-12`+도움말) | 약 88px |
| gap-4 | 16px |
| 퇴사일 필드(라벨 1~2줄+`h-12`+도움말 2줄) | 약 108px |
| 구분선 + 그룹② 소제목 "임금 정보" | 약 32px |
| 3개월 급여 총액 필드(라벨+`h-12`+도움말) | 약 88px |
| gap-4 | 16px |
| 상여+연차 2열 행(라벨+`h-12`+도움말) | 약 92px |
| 계산 버튼(`h-12`) + 개인정보 캡션 | 약 84px |
| 카드 상하 padding | 32px |
| **입력 폼까지 소계** | **약 660~700px + 배너** ≈ **730~790px** |
| 결과 카드(히어로+각주+요약 2줄+구분선+보조 4행, padding) | 약 320~360px |

→ 입력 폼만으로 실사용 뷰포트를 넘어선다. **연봉 계산기와 동일 판단**: design-system 4장 원칙 1을
문자 그대로 달성 불가(입력 5개). 절충안:

1. **상단 배너 광고를 넣지 않는다**(모바일·데스크톱 공통). 최상단은 근사 고지 배너 우선(§6-1).
2. **입력 폼 압축**하되 라벨·48px 터치타깃 유지(YMYL 명확성 우선). 도움말 Caption이 세로를 늘리면
   포커스 시에만 노출하거나 핵심 필드에만 두는 조정을 개발 재량으로 허용(강제 아님).
3. **계산 버튼 클릭 시 결과 영역(`resultRef`)으로 스무스 스크롤**(연봉 컴포넌트 구현 그대로
   재사용, 강력 권장). 5필드 폼이라 결과가 접힘선 아래로 밀리므로 필수적이다(근속연수와 다른 점 —
   근속연수는 date 2개라 스크롤 불필요했으나 퇴직금은 연봉급 폼 길이라 스크롤 채택).

### 5-2. 결과 카드 가로 오버플로 0 규칙 (필수)

- **다열 `<table>` 미사용.** Tier ②의 4행 모두 `flex items-center justify-between`(라벨 좌 줄바꿈
  허용 / 값 우 `tabular-nums` 고정) → **375px에서 가로 스크롤이 구조적으로 발생 불가**(연봉·근속연수
  스펙과 동일 원칙).
- **히어로 금액**: 최장 현실값(장기근속 고소득 `180,000,000원`, 9자리)까지 `text-4xl`(36px) +
  `flex-wrap`으로 "원" 단위 줄바꿈 흐름 → 오버플로 0(§3-2). 일반값(8자리 이하)은 한 줄.
- **재직일수 복합 값**(`2,192일 (근속 6년 0개월)`): 라벨이 짧아 한 줄. 극단 시 라벨 줄바꿈, 값 우측
  유지 → 오버플로 0(§3-3).

### 5-3. 상여+연차 2열의 모바일 처리 — 자릿수 폭 검증

`grid grid-cols-2 gap-3`(연봉 2열 패턴)을 375px에서도 유지한다. 검산: 카드 안쪽 약 343px −
gap 12px = 각 열 약 165px, input `px-4`(좌우 16px) 제외 텍스트 폭 약 133px. 현실 상여금 최댓값
`60,000,000`(10자리, 콤마 포함)도 16px 폰트 기준 약 90px로 133px 안에 들어옴 → **자릿수 잘림
없음**(연봉이 억 단위+긴 라벨 때문에 전체 폭으로 뺀 것과 달리, 상여/연차는 라벨이 짧고 금액이
급여 총액보다 작아 2열 안전). 라벨("연간 상여금"·"연차수당")이 짧아 2줄 방지. 좁은 극단(≤360px)
에서 잘림 우려 시 `max-[359px]:grid-cols-1` 폴백 허용(개발 재량, 기본은 2열 유지).

### 5-4. 부수 규칙

- 모든 입력·버튼 `h-12`(48px) 터치타깃(design-system 4장 원칙 3).
- 데스크톱에서 카드 폭이 넓어도 히어로/보조행 구조 동일(리플로우만), `max-w-3xl` 내라 과폭 없음.
- **다크모드 없음**: 기존 계산기 전부 `brand-*` 라이트 토큰만 사용(코드 확인). 이 컴포넌트도 `dark:`
  분기 미도입(사이트 전역 다크모드는 별도 톤앤매너 결정 사안, 범위 밖).

## 6. YMYL(중간) 반영 — 근사 배너 · CTA 금지 · 광고

### 6-1. 상단 근사 고지 배너 (amber `role="note"`, 항상 노출) — 카피 제안

마스터 확정: "연봉 계산기와 동일한 amber `role="note"` 스타일, 항상 노출". **연봉 컴포넌트의 배너
마크업을 그대로 재사용**한다.

- 위치: 컴포넌트 최상단, 계산기 카드 위(§1 와이어프레임). 계산 여부와 무관하게 항상 노출.
- 스타일(연봉과 동일): `mb-6 flex items-start gap-2 rounded-lg border border-amber-200 bg-amber-50
  px-4 py-3`, `role="note"`. 좌측 `ⓘ`(`text-brand-warning`, `mt-0.5`, `aria-hidden`), 본문
  `text-sm text-amber-900`. **광고 유닛 아님**(“광고” 캡션 금지).
- **카피 제안(기획 §7-3 초안 반영, YMYL-중간 톤, 퇴직·이직 부추김 배제 — 최종은 기획팀 확정):**

  > **1안(권장):** "예상 퇴직금은 **세전(퇴직소득세 공제 전)** 금액이며, 법정 산식으로 계산한
  > 참고용 값입니다. 통상임금·부분월·무급휴직 등에 따라 실제 지급액과 다를 수 있으니, 정확한
  > 금액은 회사 규정과 고용노동부 퇴직금 계산기로 확인해 주세요."

  > **2안(간결):** "예상 퇴직금은 세전 금액이며, 통상임금·부분월 등에 따라 실제 지급액과 다를 수
  > 있습니다. 참고용으로만 활용해 주세요."(기획 §7-3 초안 그대로)

- **디자인팀 의견**: 연봉 배너(3줄)와 동일 지면 예산에서 1안이 세전·근사·공식출처를 3중으로 담아
  YMYL-중간에 적합. 다만 최종 문안은 **기획팀 확정 사안**이므로 "카피 제안"으로 명시한다(§9).
- **YMYL 톤**: 경고가 아니라 안내. "지급받을 수 있다/받아야 한다" 류 권유·부추김 표현 금지, 사실
  고지만.

### 6-2. 결과 지점 근사 고지 (3중 고지 정합)

상단 배너(§6-1) + interpretation ③·④단(기획 §2, 통상임금 한계·세전) + FAQ 2·4(기획 §3)로 근사
한계가 이미 3중 고지된다. **결과 카드 내 별도 인라인 노트는 두지 않는다**(연봉은 접이식 세금 소계
아래 1줄을 뒀으나, 퇴직금은 접이식이 없고 세전 각주(§3-2)·1년미만 안내(§3-4 B)가 결과 지점 고지를
이미 담당하므로 중복 회피). 세전 각주 = 결과 지점의 핵심 고지.

### 6-3. CTA 디자인 금지 원칙 (YMYL-중간, 퇴직·이직 부추김 절대 금지)

- 계산기 카드 안에는 **"퇴직금 계산하기" 버튼 1개만** 존재. "퇴직하면 얼마?", "이직 준비",
  "퇴직금 더 받는 법", "노무사 상담 신청" 류 2차 버튼/링크를 결과 카드나 주변에 **절대** 추가하지
  않는다.
- 결과·요약 문구(§3-5)에도 퇴직·이직·전직을 부추기거나 암시하는 표현 금지. 사실 서술만.
- 공식 기관(고용노동부·홈택스 등) 참조가 필요하면(주로 interpretation 본문, 범위 밖) **일반 본문
  링크 스타일**(밑줄 + Primary 인라인)만, 박스형 CTA 버튼 금지(버튼은 계산 버튼 전용 예약).
- "관련 계산기" 카드(공통 템플릿)는 내부 링크로만(기획 §4: 연봉+근속연수+만나이 override 제안).
  외부 채용/이직/노무 상담 서비스 연결 카드 금지.

### 6-4. 광고 배치 (design-system.md 5장 + 이 페이지 규칙)

- 상단 배너 광고: 넣지 않음(§5-1, 근사 고지 배너가 그 자리).
- 인터랙티브 클러스터(입력 ↔ 버튼 ↔ 결과 카드) 내부/사이에 광고 미삽입(오인 클릭·흐름 방해).
- 본문 중간/하단 광고: 공통 템플릿 처리(범위 밖). design-system 5장 준수(위아래 margin 24px+,
  "광고" Caption, 결과 카드의 에메랄드 틴트/좌측 강조선·amber 배너와 유사하지 않은 **중립 배경
  (White/Gray 50 + 얇은 회색 테두리)**). 급여/퇴직/재테크 광고 소재가 "퇴직금 더 받기" 류 문구를
  포함하더라도 사이트 UI와 명확히 구분되게 감싼다. 클릭 유도 배치 금지.

## 8. 개발팀 전달 체크리스트 — 기획 §7 대비 자기검증 (불일치 0 확인)

### 8-1. 입력 5개 대조 (기획 §7-1 ↔ 본 스펙 §2-2)

| # | 기획 §7-1 입력 | 타입/필수/기본 | 본 스펙 §2-2 | 일치 |
|---|---|---|---|---|
| 1 | 입사일 | date / 필수 / — | `startDate` date 필수 | ✅ |
| 2 | 퇴사일(마지막 근무일, 미입력 시 오늘) | date / 선택 / 오늘 | `endDate` date 선택, 미입력=오늘, 라벨 동일 | ✅ |
| 3 | 최근 3개월 급여 총액(세전, 콤마) | number / 필수 / — | `recentPay` text+numeric 콤마 필수, "세전" 도움말 | ✅ |
| 4 | 연간 상여금 총액 | number / 선택 / 0 | `annualBonus` text+numeric 콤마 선택, 빈값=0 | ✅ |
| 5 | 연차수당 | number / 선택 / 0 | `annualLeavePay` text+numeric 콤마 선택, 빈값=0 | ✅ |

→ **입력 5개 100% 일치. 추가·삭제·치환 없음.** 라벨은 §2-2 표대로 화면 표기(단위 `(원)`·의미
한정은 도움말로 이관)했을 뿐 항목·타입·필수·기본값 불일치 0.

### 8-2. 출력 대조 (기획 §7-2 ↔ 본 스펙 §3)

| 기획 §7-2 출력 | 본 스펙 배치 | 일치 |
|---|---|---|
| 대표값: 예상 퇴직금 `N원`, 세전 배지/각주 | Tier① 히어로 + [세전] 배지 + 세전 각주(§3-2) | ✅ |
| 보조 (a) 1일 평균임금 `N원` | Tier② 행(§3-3) | ✅ |
| 보조 (b) 평균임금 산정기간 총일수 `N일`(89~92) | Tier② 행(§3-3) | ✅ |
| 보조 (c) 임금총액(3개월) `N원` | Tier② 행(§3-3) | ✅ |
| 보조 (d) 재직일수 `N일` + 근속 `N년 M개월` | Tier② 복합 행(§3-3) | ✅ |
| 1년 미만: 대표값 자리 안내 + 재직일수만, 금액 미표시 | §3-4 B 안내 카드 | ✅ |
| 퇴사일 미입력: "오늘(YYYY년 X월 X일) 기준" 표기 | §3-4 A 라벨 동적 + caption | ✅ |

→ **출력(대표 1 + 보조 4 + 2개 상태) 100% 일치. 항목 추가·삭제 없음.** Tier② **표시 순서만**
계산 흐름 순으로 제안했고(§3-3), 이는 항목 변경이 아니라 배열 순서 제안이다(§9 기획 확인 요청).

### 8-3. 마스터 확정 결정 반영 확인

| 마스터 확정 | 반영 위치 |
|---|---|
| 입력 5개 확정 | §2-2, §8-1 ✅ |
| 출력 대표1+보조4 | §3, §8-2 ✅ |
| 1년 미만 = 금액 미산출 + 안내 | §3-4 B ✅ |
| 상단 amber `role="note"` 배너, 항상 노출 | §6-1 ✅ |
| 버튼식(실시간 아님) | §0-2, §2-3 ✅ |

### 8-4. 개발 구현 유의(스펙 준수 항목)

- 금액 3필드 `formatAmountInput` 재사용(콤마·음수 차단), 결과 금액 `formatWon`.
- `resultRef` 계산 후 스무스 스크롤(§5-1, 연봉 패턴).
- 필드별 인라인 검증(`FieldError`, 근속연수 패턴): 입사일 미입력 / 급여 미입력·0 / 퇴사일<입사일.
- `type="date"`에 `max` 미설정(미래 퇴사일 허용, §2-2).
- 동적 요약 `getResultSummary`(§3-5, 정적 금지). 1년 미만·오늘 기준 상태 분기(§3-4).
- 엔진 출력이 기획 §1-4 앵커 A~E와 일치하는지 회귀 테스트(A 20,562,239 / B 21,178,856 / C
  12,952,686(N_avg 89) / D 6,994,521(N_avg 90) / E 지급대상 아님). 표시 반올림 자릿수는 개발
  라운딩 정책 확정 후 고정(기획 §8-5).
- 구현 후 375×667 / 390×844 / 1280에서 §5-2 가로 오버플로 0, §5-1 스크롤 동작을 디자인팀이 재검수.

## 9. 마스터 확인 필요 / 타팀 이관 (핑퐁)

**기획팀에 되돌릴 이슈 (출력 항목 추가·삭제·치환): 없음.**
입력 5개·출력(대표1+보조4)을 기획 §7 확정안 그대로 채택했고, 화면상 항목을 바꿀 필요가 발견되지
않았다(§8 자기검증 불일치 0). 아래는 **콘텐츠·순서 확인 요청**(항목 변경 아님)이다.

**기획팀 확인 요청 (카피 — 콘텐츠 소관)**
- **상단 근사 고지 배너 최종 카피(§6-1)**: 1안(권장, 세전·근사·공식출처 3중)/2안(간결) 중 택1 또는
  수정. 디자인은 "카피 제안"까지만 담당, 최종 문안은 기획 확정.
- **동적 요약 문구(§3-5) 문안·톤**: 기획 §2 1문단 기반 1~2줄 축약 템플릿. 퇴사일 미입력 시 "오늘"
  표현 및 §3-4(A) caption과의 날짜 표현 중복 수위.
- **Tier② 보조 4값 표시 순서(§3-3)**: 디자인은 계산 흐름 순(임금총액→산정일수→1일평균→재직일수)을
  제안. 기획 §7-2 열거 순(1일평균 우선)을 원하면 회신만으로 재배열(항목 변경 아님).
- **1년 미만 안내 카드 문구(§3-4 B)** 및 **재직일수 근속 표기 자릿수**(일 생략 년·개월 요약) 확인.

**마스터 팀장 확인 필요**
1. **상단 amber 배너 = 이미 마스터 확정 반영**(연봉과 동일, 항상 노출). 별도 승인 불요, 반영 보고만.
2. **design-system 신규 요소 = 없음**: 본 컴포넌트는 연봉·근속연수의 기존 토큰·클래스·함수만
   재사용(§7). 신규 시각 언어 도입 없음(마스터의 "컴포넌트 내 사용만" 판단 승계).
3. **1년 미만 안내 카드 톤(§3-4 B) = 중립 회색 안내로 제안**(amber 상단 배너와 색 중첩 회피).
   위험/경고 색이 아닌 안내 톤이 맞는지 확인 요망(YMYL-중간, 오정보 아닌 "대상 아님" 안내).

**개발팀 전달 사항 (마스터 경유)**
- §7 매핑대로 `components/SeverancePayCalculator.tsx`·`lib/severance.ts` 신설, `CALCULATOR_COMPONENTS`
  매핑, `lib/calculators.ts` 콘텐츠 확보 후 `status: "live"`.
- §8-4 유의 항목 준수(콤마·스크롤·인라인 검증·`max` 미설정·동적 요약·앵커 회귀·라운딩 정책).
- 관련링크 override(기획 §4·§8-7: 연봉+근속연수+만나이) 및 라운딩 정책(기획 §8-5)은 개발/마스터
  확정 사안.

## 10. 대안으로 검토했으나 채택하지 않은 안

- **보조 4값을 접이식(`<details>`)으로 기본 접힘**(연봉 방식): 4값은 곧 계산 근거라 접으면 YMYL
  투명성이 떨어지고, 정보량이 접이식 문턱(연봉 8줄+) 아래라 근속연수처럼 항상 노출이 맞다(§3-0).
- **상여+연차를 전체 폭 세로 스택**(연봉이 연봉·비과세를 전체 폭으로 둔 방식): 두 필드는 선택·기본
  0이고 금액이 급여 총액보다 작아 2열에서 자릿수 잘림이 없어(§5-3 검산), 2열로 폼 길이를 줄이는
  편이 낫다. 급여 총액만 전체 폭 유지.
- **1년 미만 결과 영역 빨간/amber 경고 배너**: "대상 아님"은 오류가 아니라 정상 안내라 경고 색은
  과하고, 상단 amber 배너와 색이 겹친다. 중립 회색 안내 카드로 처리(§3-4 B).
- **금액 히어로에 소수점 1일 평균임금 노출**: 히어로는 예상 퇴직금 단일 대표값. 1일 평균임금은
  보조값(Tier②)이며 표시 정수 원단위로 통일(내부 전 정밀도 유지, §3-3).
- **입력을 5개 한 덩어리로 쌓기**: 필드가 많아 "어디까지 넣나"가 흐려짐. "재직 기간 / 임금 정보"
  2그룹 소제목으로 분절(§2-1). 단 카드 분할·접이식 같은 과설계는 하지 않고 소제목 라벨로만.

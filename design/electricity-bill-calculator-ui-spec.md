## 0. 이 페이지의 성격 (설계 전제)

1. **YMYL 등급 = 낮음~중간(기획 §0·YMYL 판단 승계).** 연봉·퇴직(중간)보다 한 단계 낮다("이미 쓴
   전기의 요금 추정"이지 건강·법률·금융 의사결정 아님). 다만 (1) 요금표·부담금 요율이 매년/분기
   개정되어 오정보 리스크가 있고 (2) 복지할인·아파트 고압·검침주기로 실제 청구와 달라지므로,
   **연봉·퇴직과 동일하게 상단 amber `role="note"` 근사 고지 배너를 항상 노출**한다(§6-1). 결과에
   위험/경고 색을 부여하지 않고, 강조는 순수 "값 확인" 목적의 Accent(에메랄드) 하나로 고정한다.
   누진 상위 단계를 "위험/경고 색(빨강)"으로 칠하지 않는다 — 공포 조장·상업적 부추김 금지(§6-3).
2. **인터랙션 = 버튼식(실시간 아님) 확정.** 이 계산기는 **상단 amber 근사 고지 배너를 쓰는
   YMYL 계산기 family(연봉·퇴직·대출)** 에 속하며, 그 family는 전부 `form onSubmit` + "계산하기"
   버튼이다(사이트 "단발=버튼 / 연속 변환=실시간" 이원화 원칙, 단위변환만 실시간). 입력이 2개로
   가볍지만 **사용량 타이핑 중 부분 입력 깜빡임(1→15→150)** 을 막고 "확인 후 결과"의 YMYL 리듬을
   유지하기 위해 버튼식으로 둔다. → `form`/`onSubmit`/"계산하기" 버튼 사용, 실시간 파생 렌더 금지.
   (실시간 채택 여부는 §10 대안 검토 + §9 마스터 확인 항목으로 남긴다.)
3. **입력 2개 · 출력(대표 1 + 보조 8) — 기획 §7 확정안 그대로.** 입력은 사용량(number) + 계절
   (select) 2개뿐이라 매우 가볍고, 결과 정보량은 보조 8값(a~h)으로 연봉(공제 8줄+)에 가깝다.
   그래서 **연봉의 3-tier(대표값 히어로 + 핵심 요약행 항상 노출 + 세부 접이식)를 채택**한다(§3-0
   근거). 여기에 이 계산기 고유의 **누진단계 게이지(항상 노출)** 를 결합한다(§3-3, 채택 결정).
4. **YMYL CTA 금지.** 절전상품·특정 요금제 가입·에너지 컨설팅 유도 등 상업적 2차 버튼/문구를
   계산기 카드 안팎에 절대 넣지 않는다(§6-3). 계산 버튼 1개만 존재. 공공요금 계산기의 신뢰도가
   핵심이다.

## 2. 입력 설계 (사용량 1 + 계절 1 + 계산 버튼)

### 2-1. 배치 원칙

입력이 2개뿐이라 그룹 분절(퇴직의 소제목)이나 접이식 없이 **같은 흰 카드 안에 세로 스택**한다.
과설계 금지.

| 순서 | 필드 | 데스크톱 배치 | 모바일 배치 |
|---|---|---|---|
| 1 | 월 사용량 | 전체 폭 (우측 `kWh` 접미 라벨) | 전체 폭 |
| 2 | 계절 | 전체 폭(권장) 또는 사용량과 2열 | 전체 폭 |

- 데스크톱에서 사용량+계절을 2열(`grid-cols-2`)로 묶는 안도 가능하나, **계절 셀렉트 아래 상태문구
  (§2-5) 1줄이 붙어 2열이면 높이가 어긋난다.** 기본은 **둘 다 전체 폭 세로 스택** 권장(상태문구가
  계절 바로 아래 자연스럽게 붙음). 2열 채택 시 상태문구 자리 확보를 개발이 처리(§9 개발 재량).

### 2-2. 필드별 스펙

design-system + 연봉/퇴직의 입력 스타일을 **그대로** 재사용한다.

`inputBase` = `"h-12 rounded-lg border px-4 text-base text-brand-text outline-none transition-colors
focus:border-brand-primary focus:ring-4 focus:ring-brand-primary/15"` (오류 시 `border-brand-warning`,
평시 `border-brand-border` 토글) — 연봉·퇴직과 동일 상수.

| 순서 | 필드 | id | 라벨(표시) | 컨트롤 | 필수 | placeholder / 기본값 | 도움말(Caption) |
|---|---|---|---|---|---|---|---|
| 1 | 월 사용량 | `usage` | **"월 사용량 (kWh)"** | `type="text"` + `inputMode="numeric"`, 천단위 콤마, 우측 `kWh` 접미 | **필수** | placeholder `350` | "한 달 동안 사용한 전력량(kWh). 검침 사용량 또는 고지서의 사용량을 입력하세요." |
| 2 | 계절 | `season` | **"계절"** | `select`(연봉 `selectBase` 스타일) | 선택 | 기본값=**현재월 자동**(7·8월→`summer`, 그 외→`other`) | "여름(7·8월)은 누진구간이 완화됩니다." + 상태문구(§2-5) |

- **계절 옵션(2개, 값은 §7-2와 1:1):** `하계(7·8월)`(value `summer`) / `기타계절(그 외)`(value
  `other`). 순서는 하계·기타계절. 기본 선택값은 §2-5 자동 결정.
- **사용량 = 정수 kWh + 천단위 콤마 재사용**: 연봉의 `formatAmountInput(raw)`(숫자만 남기고
  `toLocaleString("ko-KR")`)을 사용량 필드에 **그대로 재사용**. 이 함수는 `[^\d]`(숫자 외)를
  제거하므로 **음수 기호·문자·소수점 입력이 구조적으로 불가** → 기획 §1-0 "사용량 ≥ 0, 음수 불가,
  정수 권장"이 입력 단계에서 원천 차단된다(별도 음수 검증 로직 불필요, §2-4). kWh는 원(₩)이
  아니지만 "숫자+천단위 콤마" 서식 재사용 취지는 동일하며, 통상 4자리 이하라 콤마가 필수는
  아니나(예 `1,200`) 큰 값 가독성을 위해 서식 적용.
- **우측 `kWh` 접미 라벨**: 사용량 input 우측에 `kWh` 단위를 시각적으로 붙인다(input 내부 우측
  `pr-12` + absolute `text-sm text-brand-text-secondary` 접미, 또는 라벨에 이미 `(kWh)`가 있으므로
  input 우측 접미는 생략 가능 — 개발 재량. 라벨 `(kWh)` 만으로 단위 인지 충족).
- **계절 select 스타일 = `selectBase`(연봉 §2-2)**: `inputBase`와 동일 높이(`h-12`)·테두리·radius·
  포커스 링 + 우측 `pr-9`에 chevron(▾) `pointer-events-none` 겹침 + `appearance-none`. 연봉에서
  이미 도입·사용 중이므로 그대로 재사용(신규 아님).
- 라벨: `text-sm font-medium text-brand-text-secondary`. 각 필드 컨테이너 `flex flex-col gap-1.5`.
  Caption: `text-xs text-brand-text-secondary`.

### 2-3. 계산 버튼 + 개인정보 캡션

- 버튼: 연봉·퇴직과 동일 `h-12 w-full rounded-lg bg-brand-primary px-5 text-base font-semibold
  text-white transition-colors hover:bg-brand-primary-hover`. 라벨 **"전기요금 계산하기"**.
- 버튼 아래 개인정보 캡션(`mt-2 text-xs text-brand-text-secondary`): "입력하신 사용량은 브라우저
  안에서만 계산되며 서버에 저장·전송되지 않습니다."(기획 §2 ⑤단과 일관). 사용량 1개라 민감도가
  낮지만 신뢰도 목적으로 유지.

### 2-4. 입력 검증 (기획 §1-0) — 필드별 인라인, 퇴직과 동일 방식

퇴직 `FieldError`(필드+메시지) 패턴을 그대로 쓴다. **결과 영역 빨간 배너 방식 미사용**, 오류 원인
필드 하단에 인라인 노출.

| 검증 | 트리거 | 표기 위치·스타일 | 문구 예 |
|---|---|---|---|
| 사용량 미입력(필수) | 빈값 또는 숫자화 후 값 없음 | `usage` 테두리 `border-brand-warning` + 하단 `text-xs text-brand-warning` `role="alert"` | "월 사용량을 입력해주세요." |
| 사용량 0 | 숫자화 후 `= 0` | (개발 재량) 필수 안내로 처리 권장 — "월 사용량을 입력해주세요." | 상동 |
| 음수 사용량 | — | **불필요**(§2-2 콤마 서식이 음수·문자·소수점 원천 차단) | — |
| **사용량 매우 큰 값(> 1,000kWh)** | 계산은 정상 수행 | **에러 아님** — 결과 카드 내 인라인 note(§3-6) | (§3-6 note) |
| 계절 미선택 | — | 발생 안 함(항상 자동 기본값 존재) | — |

- 사용량 `0`은 수학적으로는 계산 가능(기본요금만 부과)하나 사용자의 미입력 실수일 확률이 높아
  필수 안내로 처리 권장(개발 재량). 정확한 트리거·문구는 개발 재량이며 본 문서는 **시각 스타일과
  배치**만 규정(연봉·퇴직과 동일 원칙).
- 에러 시 결과 카드는 렌더하지 않는다(퇴직 §2-4·§3-4 C와 동일).

### 2-5. 계절 자동설정 상태문구 (마스터 확정 항목 — 계절 select 아래 노출)

기획 §7-3·§8-8 요청 반영. 계절 기본값이 현재월로 자동 결정되므로, 사용자 혼동 방지를 위해 **계절
셀렉트 바로 아래**에 상태 caption을 노출한다. 스타일 `mt-1 flex items-start gap-1 text-xs
text-brand-text-secondary`, 좌측 `ⓘ`(`aria-hidden`).

- **자동 상태(현재 셀렉트 값 = 현재월 자동값):**
  - 7·8월: "현재 {M}월 기준 **하계(7·8월)** 요금으로 자동 선택되었습니다."
  - 그 외: "현재 {M}월 기준 **기타계절** 요금으로 자동 선택되었습니다."
- **사용자 변경 상태(셀렉트 값 ≠ 현재월 자동값):** "현재월"을 주장하지 않는다.
  - 하계로 변경: "**하계(7·8월)** 요금으로 계산합니다. (여름철 예상 요금 미리보기)"
  - 기타계절로 변경: "**기타계절** 요금으로 계산합니다."
- `{M}`(현재 월)은 클라이언트 현재 날짜 기준(기획 §8-7: 빌드 타임 고정 방지 위해 클라이언트에서
  결정 권장). 정확한 문안은 기획 확정 사안이므로 위는 "제안"이다(§9 기획 회신 요청).

## 3. 결과 카드 설계 — [기획 §7-2·§7-3·§8-8에 대한 디자인팀 확정안]

### 3-0. 정보구조 확정 (대표 1 + 누진 게이지 + 요약 3행 + 세부 접이식 4행)

기획 §7-2 출력(대표값 1 = 최종 청구요금, 보조 8 = a~h)을 **그대로 채택**하고, **연봉의 3-tier를
이 계산기에 맞춰 적용**한다. 접이식/게이지 채택에 대한 디자인팀 결정:

> **결정 1 — 3-tier 채택.** 대표값(최종 청구요금) 히어로 + 핵심 요약 3행 항상 노출 + 세부 4항목
> 접이식(기본 접힘). 보조값 8개(a~h)를 전부 유지하되 위계를 나눈다.
>
> **결정 2 — 누진단계 게이지 채택(항상 노출).** 사용량이 현재 계절 누진구간 어디에 있는지 막대로
> 보여준다(§3-3). 이 계산기의 이름이 곧 "누진세"이므로, 누진 구조 전달이 핵심 체류·재방문 가치다.

**보조 8값(a~h)의 tier 배정 — 기획 §7-2와 1:1(항목 추가·삭제·치환 없음):**

| Tier | 항목 | 노출 | 근거 |
|---|---|---|---|
| **① 대표값** | 최종 청구요금 `N원` + "참고용 예상 청구액" 배지 | 항상 | 사용자 핵심 질문 "이번 달 얼마?" |
| ① chip | (h) 적용 누진단계·계절 (예: "기타계절 2단계 적용 · 월 350kWh") | 항상 | 대표값의 맥락. 게이지가 시각 보강 |
| ① 요약 | 기획 §2 1문단(동적) 1~2줄 | 항상 | §3-5 |
| **게이지** | 누진단계 시각화 (h 시각 표현) | 항상 | §3-3. 새 화폐 항목 추가 아님 |
| **② 요약 3행** | (e) 전기요금계 / (f) 부가가치세 / (g) 전력산업기반기금 | 항상 | "전기요금계 → +부가세 +기금 = 최종"의 검산 가능한 큰 그림 |
| **③ 세부 4행** | (a) 기본요금 / (b) 전력량요금 / (c) 기후환경요금 / (d) 연료비조정요금 | 접이식(기본 접힘) | 전기요금계(e)를 구성하는 4항목의 상세 근거 |

- **결정 근거 — 왜 3-tier(연봉형)이고 퇴직형(전부 노출)이 아닌가:** 보조값이 8개로 퇴직(4개)의
  2배다. 여기에 누진 게이지(약 90~120px)까지 항상 노출되므로, 8행을 전부 펼치면 모바일 카드
  높이가 500px대로 치솟아 대표값 위계가 뭉개진다(연봉이 접이식을 쓴 것과 동일 상황). 그래서
  **연봉 3-tier**를 채택한다.
- **결정 근거 — 왜 (e·f·g)를 항상 노출하고 (a·b·c·d)를 접는가:** (e)전기요금계→(f)+부가세→(g)+기금이
  곧 "최종 청구요금이 어떻게 만들어졌나"의 핵심 흐름이라 항상 보여 검산 가능하게 둔다(요약 3행 =
  연봉의 "세전/4대보험/세금" 3행과 동형). (a·b·c·d)는 전기요금계(e)를 다시 쪼갠 상세라 접이식으로.
- **정보 은닉 아님 + YMYL 투명성 보강:** (b)전력량요금의 핵심인 **누진 구간별 단가·경계는 게이지
  (항상 노출)에 텍스트로 이미 드러난다.** 따라서 (b)를 접이식에 두어도 누진 정보는 감춰지지 않는다.
  접이식 라벨을 "요금 항목 자세히 보기"로 명확히 노출해 CS "안 보인다" 리스크를 대비한다.
- **항목 무변경:** 기획 §7-2 8항목을 그대로, 전부 노출한다(접이식은 접었을 뿐 삭제 아님). 추가
  항목(구간별 q1/q2/q3 금액, 슈퍼유저 단가 등)은 **만들지 않는다**. 게이지에 표시되는 구간 단가
  (120/214.6/307.3원)는 **상수 라벨**이지 새 화폐 출력 항목이 아니다(§8 대조표 명시).

### 3-1. 결과 카드 공통 골격 (연봉·퇴직 결과 카드와 동일)

`mt-6 rounded-xl border-l-4 border-brand-accent bg-gradient-to-br from-emerald-50 to-white
p-4 sm:p-6`. Accent 고정(상태별 색 분기 없음 — 요금 금액·누진 단계에 위험/경고 색 부여 안 함).
계산 후에만 조건부 렌더(`result && (...)`). 갱신 시 `transition-colors duration-200`. **`resultRef`로
계산 후 스무스 스크롤**(연봉·퇴직 패턴 재사용).

### 3-2. Tier ① — 대표값(최종 청구요금) 렌더 (항상)

연봉/퇴직의 단일 금액 히어로(`formatWon` + "원" 작게 분리) 패턴 + 퇴직의 참고 배지 패턴을 결합.

```
예상 청구요금  [참고용 예상 청구액]   ← 라벨: text-sm font-medium text-brand-text-secondary
                                       배지: ml-2 rounded bg-brand-border/40 px-1.5 py-0.5
                                       text-xs font-medium text-brand-text-secondary
70,640 원                             ← 히어로: text-4xl sm:text-[2.5rem] font-bold tabular-nums
                                       text-brand-accent, "원"은 text-base font-medium text-secondary
기타계절 2단계 적용 · 월 350kWh        ← (h) chip: mt-1 text-xs font-medium text-brand-text-secondary
                                       (인라인 pill: rounded-full bg-brand-border/30 px-2 py-0.5)
월 350kWh 사용 시(기타계절) 기본요금과   ← 동적 요약(§3-5): mt-2 text-sm text-brand-text-secondary
전력량요금 등을 더한 예상 청구요금은
약 7만 원입니다.
```

- 히어로 컨테이너: `mt-1 flex flex-wrap items-baseline gap-2 tabular-nums`. **`flex-wrap`**: 고사용
  세대 극단값(예 `730,000원`대, 6~7자리)이 375px 카드 안쪽(약 311px)을 넘을 때 "원" 단위가
  줄바꿈으로 흐르도록(가로 오버플로 0). 일반값(5~6자리)은 한 줄. design-system Result 스케일
  "모바일 28px 이하 축소 금지" 준수(36px OK).
- **"참고용 예상 청구액" 배지 = 대표값 옆 항상 노출**(기획 §7-2). 상단 amber 배너(§6-1)와 함께
  결과 지점의 근사 고지를 이중으로 담당. 배지는 경고 색이 아닌 중립 회색(`bg-brand-border/40`).
- **(h) 적용 누진단계·계절 chip**: "{계절} {단계}단계 적용 · 월 {N}kWh" 형태. 게이지(§3-3)의 요약과
  호응하며, 게이지를 접거나 스크린리더로 읽을 때도 단계·계절이 텍스트로 즉시 파악되게 한다.
- "약"·"원" 단위 텍스트는 히어로보다 작게(Body) Text Secondary — design-system 단위 규칙.

### 3-3. 누진단계 게이지 (항상 노출) — [기획 §7-2 "누진 시각화(선택)"·§8-8에 대한 채택 결정]

> **채택.** Tier ① 요약 문구 아래, Tier ② 요약 3행 위에 **가로 막대(3구간 세그먼트)** 를 항상
> 노출한다. 사용자가 "지금 몇 단계인지 / 다음 단계 경계까지 얼마 남았는지"를 한눈에 보게 해
> 체류·재방문(예: "여름엔 얼마 남지?")을 유도한다.

#### 3-3-1. 구조

현재 계절의 3구간을 **좌→우 세그먼트 막대**로 표현한다. 경계는 계절에 따라 동적(§7-2 상수):

- 기타계절: `1단계 0–200` · `2단계 200–400` · `3단계 400+`
- 하계(7·8월): `1단계 0–300` · `2단계 300–450` · `3단계 450+`

3단계는 상한이 없으므로 **게이지 시각 상한(visual max)** 을 둔다: **기타 600 / 하계 600**
(= b2 + (b2 − b1), 3단계 세그먼트가 2단계와 같은 시각 폭이 되게 함 — 기타 seg3=600−400=200=seg2,
하계 seg3=600−450=150=seg2). (v1.0 초안의 "기타 500"은 공식·rationale과 모순되는 오타로, 2026-07-14
개발 검수 회신에서 공식 값 600/600으로 정정. 순수 시각 파라미터라 금액·앵커 무영향.) 사용량이
visual max를 초과하면 마커를 막대 우측 끝에 고정하고 3단계 세그먼트 끝에 `→`(더 큼) 표시 + 설명
문장에서 실제 값 노출.

```
1단계        2단계 ▲          3단계
■■■■■■■■│■■■■■■■■■■■■■│■■■■■■■■→
0        200            400
                     ↑ 350kWh (현재)
현재 350kWh는 기타계절 누진 2단계 구간입니다. 3단계(400kWh)까지 50kWh 남았습니다.
```

- **세그먼트 폭**: 실제 kWh 경계에 비례(visual max 대비 %). 각 세그먼트 내부에 `N단계` 텍스트
  라벨(작게). 경계 tick 숫자(`200`·`400` 또는 `300`·`450`)는 막대 **아래** 경계 위치에 배치.
- **현재 위치 마커**: 사용량 U 위치에 세로선(`w-0.5 bg-brand-text`) + 위쪽 작은 라벨 버블
  `{N}kWh (현재)`. 마커가 막대 좌/우 끝 8px 이내면 라벨을 막대 안쪽으로 정렬(오버플로 방지).
- **활성 세그먼트 강조**: 사용자의 현재 단계 세그먼트는 `bg-brand-primary/15 border border-brand-primary`
  + 라벨 `font-semibold text-brand-primary`. 비활성 세그먼트는 `bg-brand-border/40` + 라벨
  `text-brand-text-secondary`.
- **설명 문장(항상, 게이지 아래)**: "현재 {N}kWh는 {계절} 누진 {단계}단계 구간입니다.
  {다음경계}kWh까지 {잔여}kWh 남았습니다." / 이미 3단계면 "…누진 3단계 구간입니다. (최고 단계)".
  스타일 `mt-2 text-xs text-brand-text-secondary`.

#### 3-3-2. 접근성 (색만으로 구분 금지 — 필수)

- **색만으로 단계를 구분하지 않는다.** 활성 세그먼트는 (1) 채움색 + (2) **테두리** + (3) **굵은
  텍스트 라벨** 3중으로 구분한다. 비활성/활성이 색약 사용자에게도 텍스트(`N단계`)와 굵기·테두리로
  구분된다.
- **위험/경고 색(빨강) 사용 금지.** 상위 단계를 빨강으로 칠하면 공포 조장이 되고 YMYL·상업적
  부추김 금지 원칙에 어긋난다(§0-1). 단계 상승은 **단가 텍스트 라벨**(120 / 214.6 / 307.3원/kWh,
  §3-3-3)로만 전달하고, 활성 세그먼트는 브랜드 뉴트럴(primary/15)로만 강조.
- **텍스트 대체(스크린리더)**: 게이지 컨테이너에 `role="img"` + `aria-label`에 §3-3-1 설명 문장과
  동일한 텍스트를 넣거나, 시각 설명 문장(항상 노출)을 게이지의 접근 가능한 설명으로 연결
  (`aria-describedby`). 마커·세그먼트의 순수 그래픽 요소는 `aria-hidden`. → 스크린리더 사용자는
  "현재 350kWh는 기타계절 누진 2단계 구간, 3단계(400kWh)까지 50kWh 남음"을 텍스트로 그대로 듣는다.
- (h) chip(§3-2)이 단계·계절을 텍스트로 이미 노출하므로, 게이지가 렌더되지 않는 극단 환경에서도
  단계 정보는 유실되지 않는다.

#### 3-3-3. 단가 라벨(교육용, 선택 노출 — 새 출력 항목 아님)

- 각 세그먼트에 구간 단가(`120원` / `214.6원` / `307.3원`, kWh당)를 작은 라벨로 병기하면 "왜 많이
  쓰면 비싼가"가 직관적으로 전달된다. **데스크톱에서 세그먼트 하단에 노출**, **모바일(375px)에서는
  공간 부족 시 생략하고 접이식 Tier ③(전력량요금 근처)이나 설명 문장으로 이관**(§3-3-4).
- 이 단가는 **상수 라벨**이며 기획 §1-2 값과 일치. **화폐 출력 항목(a~h)에 추가되는 것이 아니다**
  (§8 대조표에 명시 — 출력 항목 증가 아님).

#### 3-3-4. 모바일(375px) 게이지 레이아웃

- 막대는 **가로 100%** (세로 높이 약 28~36px). 세로로만 늘어 가로 오버플로 0.
- 세그먼트 내부 라벨은 `1단계/2단계/3단계`만(짧게). **단가 라벨은 모바일에서 생략**하고 설명
  문장에 흡수("2단계 구간(kWh당 214.6원)…"은 길어지므로, 단가는 Tier ③ 전력량요금 행 근처에서만).
- 경계 tick 숫자(`200`·`400`)는 막대 아래 절대 위치. 마커 라벨 버블은 막대 위. 세 요소가 세로로
  분리 배치되어 375px에서 겹치지 않음.
- 마커가 막대 양 끝 근처일 때 라벨 정렬(좌끝→`text-left`, 우끝→`text-right`)로 카드 밖 이탈 방지.
- 게이지 총 세로 높이(라벨+막대+tick+설명 문장): 약 90~120px 추정(§5-1 높이표 반영).

### 3-4. Tier ② — 요약 3행 (항상)

Tier ① + 게이지 아래 구분선(`mt-4 pt-4 border-t border-brand-border/60`) 다음, "전기요금계 → +부가세
+기금 = 최종"의 큰 그림 3행을 나열한다. 연봉 요약 3행 패턴 재사용.

```
전기요금계                62,690 원      ← AmountRow (원)
부가가치세 (10%)         + 6,269 원      ← AmountRow, 값 앞 '+' (더해지는 항목 명확화)
전력산업기반기금 (2.7%)  + 1,690 원      ← AmountRow, 값 앞 '+'
```

- 각 행: `flex items-center justify-between py-1.5 text-sm`. 라벨(좌) `text-brand-text-secondary`,
  값(우) `tabular-nums font-semibold text-brand-text`(**Accent 아님** — 컬러 강조는 히어로 전용).
- **부가세·기금 값 앞 `+`**: 전기요금계에 "더해져" 최종이 됨을 시각화(연봉이 공제에 `−`를 붙인 것과
  대칭). 전기요금계 62,690 + 부가세 6,269 + 기금 1,690 = 최종 70,640(히어로)이 검산됨.
- **요율 병기**: 부가가치세 `(10%)`·전력산업기반기금 `(2.7%)`을 라벨에 병기해 근거를 투명화(기획
  §1-2·§2 본문의 2.7%와 일치 — 3.7% 아님). 요율이 상수 변경되면 라벨도 함께 갱신(개발: 라벨을
  상수에서 파생 권장, §7).

### 3-5. Tier ③ — 세부 4행 (접이식, 기본 접힘)

Tier ② 아래 `<details>` 접이식(연봉 §3-4 패턴). summary는 클릭 영역 전체(터치타깃 확보).

```
▸ 요금 항목 자세히 보기        ← summary: text-sm font-medium text-brand-primary, chevron 회전
  ─────────────────────────
  기본요금                1,600 원      ← AmountRow (원)
  전력량요금              56,190 원      ← AmountRow (원)
  기후환경요금             3,150 원      ← AmountRow (원)
  연료비조정요금           1,750 원      ← AmountRow (원)
```

- 각 상세 행: `flex justify-between py-1 text-sm`(padding만 `py-1`로 촘촘). 네 항목 합 = 전기요금계
  (Tier ②)임이 검산됨(1,600+56,190+3,150+1,750 = 62,690).
- **표시 순서 = 계산 파이프라인 순**(기본요금→전력량→기후환경→연료비조정, 기획 §1-1 STEP 1~4).
  이는 기획 §7-2 (a~d) 항목의 **표시 순서일 뿐, 항목의 추가·삭제·치환이 아니다**(§8·§9 명시).
- (선택) 전력량요금 행 아래에 §3-3-3 구간 단가(120/214.6/307.3원)를 `text-xs` 보조 라벨로 병기
  가능(모바일에서 게이지가 단가를 생략했을 때의 이관 위치). 새 화폐 항목 아님.

### 3-6. 상태별 UI — 사용량 큰 값(> 1,000kWh) 인라인 note

퇴직처럼 "1년 미만" 특수 케이스는 **없다**(task 확인). 대신 사용량 극단값 처리:

- **사용량 > 1,000kWh**: 결과는 정상 렌더(3단계 누진까지 계산). Tier ② 아래(또는 전력량요금 행
  근처)에 인라인 note 1줄: `text-xs text-brand-text-secondary` — "월 1,000kWh를 초과하는 경우
  실제로는 별도 단가(슈퍼유저 요금)가 적용될 수 있어 계산 결과와 다를 수 있습니다."(기획 §1-2
  슈퍼유저 v1 미구현 고지). 게이지 마커는 visual max 우측 끝 고정 + `→`.
- **사용량 0**: §2-4대로 필수 안내로 처리 권장(결과 미렌더). 개발이 0을 계산 대상으로 두면(기본요금
  만 부과) 정상 렌더도 허용하나, 흔치 않은 케이스라 필수 안내 우선 권장.
- **계절별 결과 차이**: 동일 사용량이라도 계절 변경 시 경계(200/400 ↔ 300/450)가 달라 단계·전력량
  요금·게이지가 전부 갱신된다. 이는 정상 재계산이며 별도 상태 UI 불필요 — 게이지·chip·요약이
  자동 반영. (예: 500kWh는 하계 3단계 110,280원 / 기타계절 3단계 더 비쌈, 기획 §1-4 앵커 C.)
- **경고 색 미사용**: 위 note·큰 값·최고 단계 모두 중립 회색 안내. 상단 amber 배너와 색 중첩을
  피하고(퇴직 §3-4 B 원칙 승계), 공포 조장 금지.

### 3-7. Tier ① 요약 문구 = 기획 §2 1문단 동적 배치

기획 §2 1문단(앵커 B: "월 350kWh…예상 청구요금은 약 70,640원…")은 정적 텍스트가 아니라 **사용자
입력값 기반 동적 생성**이어야 한다(기획 §8-7, 기존 라이브 계산기 전부 동일). 이 문서는 **배치 위치만
확정**: Tier ① chip 아래(`mt-2 text-sm text-brand-text-secondary`). 연봉·퇴직의 `getResultSummary(...)`
처럼 컴포넌트 내부 함수로 입력·결과를 받아 1~2문장 조립. **정적 텍스트 금지.**

- 길이: 모바일 2줄 이내 권장. 금액은 "약 N만 원" 반올림으로 간결하게(상세 숫자는 Tier ②·③에 있음).
- 계절·단계를 문장에 자연어로 반영("월 {N}kWh 사용 시({계절}) … 누진 {단계}단계가 적용되어…").
- **CTA 가드**: 절전상품·요금제 가입을 부추기는 표현 금지(§6-3). 사실 서술만. 정식 카피 톤은 기획
  §2 담당(§9).

## 5. 모바일(375px) 반응형 규칙

### 5-1. 무스크롤 목표 — 입력 2개라 입력부는 짧으나 결과 카드가 길어 스무스 스크롤 채택

iPhone SE급(375×667, 실사용 높이 약 550~600px) 개략 높이 추정.

| 영역 | 예상 높이 |
|---|---|
| 근사 고지 배너(2~3줄, §6-1) | 약 72~92px |
| 월 사용량 필드(라벨+`h-12`+도움말) | 약 88px |
| gap-4 | 16px |
| 계절 select(라벨+`h-12`+상태문구 1줄) | 약 96px |
| 계산 버튼(`h-12`) + 개인정보 캡션 | 약 84px |
| 카드 상하 padding | 32px |
| **입력 폼까지 소계** | **약 316~336px + 배너** ≈ **390~430px** |
| 결과 카드(히어로+배지+chip+요약 2줄+게이지 100~120px+구분선+요약 3행+접이식 summary, padding) | 약 380~430px |

→ 입력 폼(약 400px)은 배너 포함해도 한 화면에 거의 들어오지만(입력 2개라 연봉/퇴직보다 짧다),
**결과 카드는 게이지(약 100~120px) 때문에 접힘 상태로도 약 400px**라 입력+결과가 한 화면에 다
들어오진 않는다. 절충안:

1. **상단 배너 광고를 넣지 않는다**(모바일·데스크톱 공통). 최상단은 근사 고지 배너 우선(§6-1).
2. **계산 버튼 클릭 시 결과 영역(`resultRef`)으로 스무스 스크롤**(연봉·퇴직 구현 그대로 재사용).
   결과 카드가 게이지 때문에 접힘선 아래로 밀리므로 채택.
3. **Tier ③ 접이식이 결과 카드 높이를 억제하는 장치**다. 세부 4행을 기본 접힘으로 두어 카드 초기
   높이를 억제하고, 대표값(최종 청구요금)은 스크롤 직후 즉시 above-the-fold에 온다.

### 5-2. 결과 카드 가로 오버플로 0 규칙 (필수)

- **다열 `<table>` 미사용.** Tier ②·③의 모든 행은 `flex items-center justify-between`(라벨 좌 줄바꿈
  허용 / 값 우 `tabular-nums` 고정) → 375px에서 가로 스크롤 구조적으로 발생 불가(연봉·퇴직 원칙).
- **히어로 금액**: 최장 현실값(고사용 세대 6~7자리, 예 `730,000원`대)까지 `text-4xl`(36px) +
  `flex-wrap`으로 "원" 단위 줄바꿈 흐름 → 오버플로 0(§3-2).
- **누진 게이지**: 가로 100% 막대라 세로로만 늘어 폭 이슈 없음. 마커 라벨은 양 끝에서 안쪽 정렬로
  이탈 방지(§3-3-4). 단가 라벨은 모바일에서 생략(§3-3-3·§3-3-4)해 세그먼트 내 텍스트 넘침 방지.
- **(h) chip**: `flex-wrap` 허용 pill이라 좁으면 줄바꿈, 오버플로 0.

### 5-3. 부수 규칙

- 모든 입력·select·버튼 `h-12`(48px) 터치타깃(design-system 4장 원칙 3).
- 계절 select은 모바일에서 전체 폭(§2-1). 상태문구(§2-5)가 바로 아래 붙어 2열 어긋남 없음.
- **다크모드 없음**: 기존 계산기 전부 `brand-*` 라이트 토큰만 사용. 이 컴포넌트도 `dark:` 분기
  미도입(사이트 전역 다크모드는 별도 톤앤매너 결정 사안, 범위 밖).

## 6. YMYL(낮음~중간) 반영 — 근사 배너 · CTA 금지 · 광고

### 6-1. 상단 근사 고지 배너 (amber `role="note"`, 항상 노출) — 카피 1·2안 + 권장

마스터 확정: "YMYL 낮음~중간, 상단 amber `role="note"` 배너, 항상 노출". **연봉·퇴직 컴포넌트의
배너 마크업을 그대로 재사용**한다.

- 위치: 컴포넌트 최상단, 계산기 카드 위(§1 와이어프레임). 계산 여부와 무관하게 항상 노출.
- 스타일(연봉·퇴직과 동일): `mb-6 flex items-start gap-2 rounded-lg border border-amber-200
  bg-amber-50 px-4 py-3`, `role="note"`. 좌측 `ⓘ`(`text-brand-warning`, `mt-0.5`, `aria-hidden`),
  본문 `text-sm text-amber-900`. **광고 유닛 아님**("광고" 캡션 금지).
- **카피(기획 §7-3 초안·§8-8 위임 반영, YMYL-낮음~중간 톤 — 최종 문안은 기획 확정):**

  > **1안(권장):** "한전 주택용 저압 요금표를 기준으로 한 **참고용 예상 청구액**입니다. 복지할인·
  > 아파트 일괄계약(고압)·검침 주기 등에 따라 실제 청구액과 다를 수 있으며, 요금표·부담금 요율은
  > 매년/분기 개정될 수 있습니다. 정확한 금액은 한국전력공사 사이버지점에서 확인해 주세요."

  > **2안(간결):** "한전 주택용 저압 요금표 기준의 예상 청구액이며, 복지할인·아파트 일괄계약(고압)·
  > 검침 주기 등에 따라 실제 청구액과 다를 수 있습니다. 참고용으로만 활용해 주세요."(기획 §7-3
  > 초안 그대로)

- **디자인팀 권장 = 1안.** 연봉·퇴직 배너(2~3줄)와 동일 지면 예산에서 1안이 (참고용 + 저압 기준 +
  복지할인·고압·검침 차이 + 요율 개정 + 공식 출처)를 다중으로 담아 YMYL-낮음~중간의 핵심 면책
  5요소를 모두 커버한다. 2안은 요율 개정·공식 출처가 빠져 다소 약하다. 다만 최종 문안은 기획팀
  확정 사안이라 "카피 제안"으로 명시(§9).
- **YMYL 톤**: 경고가 아니라 안내. "절약하면 얼마 아낀다", "요금제 바꾸면 유리" 류 권유·부추김
  표현 금지, 사실 고지만.

### 6-2. 결과 지점 근사 고지 (3중 고지 정합)

상단 배너(§6-1) + "참고용 예상 청구액" 배지(§3-2) + interpretation ④·⑤단(기획 §2, 개정 가능성·
복지할인·고압·검침 차이) + FAQ 3·5(기획 §3)로 근사 한계가 다중 고지된다. 결과 카드 내 별도 인라인
노트는 **큰 값(>1,000kWh) note(§3-6)** 만 두고 중복 회피한다. "참고용 예상 청구액" 배지 = 결과
지점의 핵심 고지.

### 6-3. CTA 디자인 금지 원칙 (YMYL, 상업적 부추김 절대 금지)

- 계산기 카드 안에는 **"전기요금 계산하기" 버튼 1개만** 존재. "절전상품 보기", "요금제 비교/가입",
  "에너지 컨설팅 신청", "전기요금 아끼는 법" 류 2차 버튼/링크를 결과 카드나 주변에 **절대** 추가하지
  않는다.
- 결과·요약 문구(§3-7)·게이지 설명(§3-3)에도 절전상품·요금제 전환을 부추기는 표현 금지. 사실
  서술만(예: "3단계까지 50kWh 남음"은 사실 고지, "지금 절약하면 요금 폭탄 피하세요"는 금지).
- 누진 상위 단계를 **빨강/경고 색으로 칠하지 않는다**(§3-3-2) — 공포 조장 = 상업적 부추김의 변형.
- 공식 기관(한전 사이버지점·고객센터 123) 참조가 필요하면(주로 interpretation 본문, 범위 밖) **일반
  본문 링크 스타일**(밑줄 + Primary 인라인)만, 박스형 CTA 버튼 금지(버튼은 계산 버튼 전용 예약).
- "관련 계산기" 카드(공통 템플릿)는 내부 링크로만(기획 §4: 단위변환+BMI+만나이 자동, override
  불필요). 외부 에너지·절전·요금제 서비스 연결 카드 금지.

### 6-4. 광고 배치 (design-system 5장 + 이 페이지 규칙)

- 상단 배너 광고: 넣지 않음(§5-1, 근사 고지 배너가 그 자리).
- 인터랙티브 클러스터(입력 ↔ 버튼 ↔ 결과 카드/게이지) 내부/사이에 광고 미삽입(오인 클릭·흐름 방해).
- 본문 중간/하단 광고: 공통 템플릿 처리(범위 밖). design-system 5장 준수(위아래 margin 24px+,
  "광고" Caption, 결과 카드의 에메랄드 틴트/좌측 강조선·amber 배너·누진 게이지와 유사하지 않은
  **중립 배경(White/Gray 50 + 얇은 회색 테두리)**). 전기·에너지·절전 광고 소재가 "요금 아끼기" 류
  문구를 포함하더라도 사이트 UI와 명확히 구분되게 감싼다. 클릭 유도 배치 금지.

## 10. 대안으로 검토했으나 채택하지 않은 안

- **인터랙션 실시간(입력 변경 시 즉시 재계산)**: 입력 2개라 가볍고 게이지 마커가 실시간 이동하면
  체류시간에 유리. 그러나 (1) amber 근사 배너 YMYL family(연봉·퇴직·대출)가 전부 버튼식이라
  일관성 저해, (2) 사용량 타이핑 중 부분값(1→15→150) 깜빡임 → 버튼식 채택(§0-2, §9 마스터 확인).
- **누진 게이지 미채택(텍스트 chip만)**: (h) chip만으로도 단계·계절은 전달되나, "누진세" 계산기의
  핵심 교육 가치(경계까지 얼마 남았나·왜 비싼가)를 시각으로 못 준다 → 게이지 채택(§3-3).
- **누진 상위 단계를 빨강 그라데이션으로 강조**: 시각적으로 "요금 폭탄"을 직관 전달하나 공포 조장·
  상업적 부추김에 해당하고 색약 접근성도 나쁨 → 뉴트럴 색 + 텍스트 라벨 3중 구분으로(§3-3-2).
- **보조 8값을 전부 항상 노출(퇴직형)**: 투명성 최고이나 게이지(100~120px)까지 더하면 모바일 카드
  500px대 → 대표값 위계 훼손 → 연봉 3-tier로 절충, (a·b·c·d)만 접이식(§3-0). 항목 삭제 아님(접었을
  뿐, 게이지가 누진 정보 보강).
- **게이지 구간 단가를 모바일에서도 세그먼트 안에 표시**: 375px에서 세그먼트 폭이 좁아 `214.6원`이
  넘침 → 모바일은 단가 생략하고 설명 문장/Tier ③으로 이관(§3-3-3·§3-3-4).

# 계산기 허브 — 디자인 시스템 스펙 (v1.0, 2026-07-08)

> 이 문서는 디자인팀이 작성한 스타일 가이드이다. 실제 코드(Tailwind config, 컴포넌트 등)는
> 개발팀이 프로젝트 스캐폴딩 완료 후 이 스펙을 참고해 구현한다. 본 문서 자체는 코드가 아니다.

---

## 1. 색상 팔레트 (라이트 모드 기준)

애드센스 사이트 특성상 "신뢰감 + 가독성 + 눈 피로도 낮음"을 최우선으로 한다.
과도한 원색·비비드 컬러는 지양하고, 블루 계열 메인 + 뉴트럴 그레이 조합을 사용한다.

| 용도 | 색상명 | HEX | 비고 |
|---|---|---|---|
| Primary (브랜드/버튼/링크) | Blue 600 | `#2563EB` | 액션 버튼, 강조 링크 |
| Primary Hover | Blue 700 | `#1D4ED8` | 버튼 hover/active |
| Accent (결과 강조) | Emerald 500 | `#10B981` | 계산 결과값, 성공 상태 |
| Warning (주의/경고) | Amber 500 | `#F59E0B` | 입력 오류, 유의사항 |
| Danger (위험/비만 등 강조) | Red 500 | `#EF4444` | 결과 상태 중 위험 수준 표시 |
| Background (기본) | Gray 50 | `#F9FAFB` | 페이지 배경 |
| Surface (카드/입력영역) | White | `#FFFFFF` | 카드, 입력 필드 배경 |
| Border | Gray 200 | `#E5E7EB` | 카드/입력 테두리 |
| Text Primary | Gray 900 | `#111827` | 본문/제목 |
| Text Secondary | Gray 500 | `#6B7280` | 설명, 캡션, 보조 텍스트 |
| Text Disabled | Gray 300 | `#D1D5DB` | 비활성 텍스트 |

- 다크모드는 이번 단계에서 정의하지 않음 (향후 별도 스펙 필요 시 마스터 팀장 확인 후 진행).
- 색상 대비는 WCAG AA 기준(본문 텍스트 대 배경 4.5:1 이상) 충족하도록 위 조합 선정.

## 2. 타이포그래피

### 추천 폰트
- **본문/전체 기본**: `Pretendard` (한글 가독성 우수, 웹폰트 무료, 다양한 굵기 지원)
- **대체(폴백)**: `Noto Sans KR` → 시스템 폰트(`-apple-system, Malgun Gothic, sans-serif`)
- 숫자(계산 결과)는 가독성을 위해 `tabular-nums` 속성 적용 권장 (자릿수 정렬).

### 스케일

| 레벨 | 크기 (px/rem) | 굵기 | 용도 |
|---|---|---|---|
| Display | 32px / 2rem | 700 (Bold) | 페이지 대제목 (H1) |
| Heading 1 | 24px / 1.5rem | 700 | 섹션 제목 (H2) |
| Heading 2 | 20px / 1.25rem | 600 (SemiBold) | 카드/블록 제목 (H3) |
| Body | 16px / 1rem | 400 (Regular) | 본문 텍스트 |
| Body Small | 14px / 0.875rem | 400 | 보조 설명, FAQ 답변 |
| Caption | 12px / 0.75rem | 400 | 출처, 저작권, 광고 라벨 |
| Result (계산 결과 전용) | 36~40px / 2.25~2.5rem | 700 | 계산기 결과값 — 모바일에서도 28px 이하로 축소 금지 |

- 줄 간격(line-height): 본문 1.6, 제목 1.3 기준.
- 모바일에서는 Display/H1을 24px로 한 단계 축소 (섹션 5 반응형 원칙 참고).

## 3. 계산기 UI 컴포넌트 스타일 가이드

### 3-1. 입력 필드 (Input)
- 배경 `#FFFFFF`, 테두리 `1px solid #E5E7EB`, radius `8px`
- 포커스 시 테두리 `#2563EB` + 얇은 outline/box-shadow (`0 0 0 3px rgba(37,99,235,0.15)`)
- 높이 최소 48px (모바일 터치 타겟 확보), 내부 padding `12px 16px`
- 라벨은 입력창 위쪽에 배치, Body Small 크기 + Text Secondary 컬러
- 에러 상태: 테두리 `#F59E0B` 또는 Red 500(`#EF4444`), 하단에 에러 메시지 Caption 크기로 표시

### 3-2. 결과 표시 영역 (가장 중요 — 시각적 위계 최상위)
- 카드 형태, 배경은 살짝 구분되는 톤(`#F0FDF4` 연한 그린 틴트 or `#EFF6FF` 연한 블루 틴트) + 좌측 또는 상단에 Accent 컬러 강조선(4px)
- 결과 숫자는 Result 스케일(36~40px, Bold), 컬러는 Accent(`#10B981`) 또는 Primary
- 단위(원, %, kg 등)는 결과 숫자보다 작은 크기(Body)로 옆에 배치, Text Secondary 컬러
- 결과 카드는 입력 필드 바로 아래, 스크롤 없이 보이는 위치(above the fold) 배치 권장 → 체류시간·재사용성 확보
- 결과가 갱신될 때 살짝 배경색 페이드 or 미세한 scale 트랜지션(0.15~0.2s) 제안 — 입력 변화 인지 향상 (기능 로직 아님, CSS transition 범위)

### 3-3. FAQ 아코디언
- 기본 상태: 질문(Body, SemiBold) + 우측 chevron 아이콘, 하단 얇은 구분선(`#E5E7EB`)
- 펼침 상태: 답변(Body Small, Text Secondary), padding `16px`, 배경은 Surface 유지
- 아코디언 항목 간 간격 `8px`, 전체를 카드 하나로 감싸 하나의 블록처럼 보이게 구성
- 클릭 영역은 질문 전체 줄(텍스트+아이콘)로 확보 — 모바일 터치 편의성

### 3-4. 관련 계산기 링크 카드
- Grid 배치: 데스크톱 3~4열, 태블릿 2열, 모바일 1열(섹션 4 참고)
- 카드 스타일: 배경 White, 테두리 `#E5E7EB`, radius `12px`, padding `16px`, hover 시 그림자(`shadow-sm → shadow-md`)와 테두리 색이 Primary로 살짝 변화
- 카드 내부: 아이콘(선택) + 계산기 이름(Body, SemiBold) + 한 줄 설명(Caption, Text Secondary)
- 카드 자체가 링크 전체이며, 광고 배너와 시각적으로 명확히 구분되는 위치/스타일 유지 (아래 5번 항목 참고)

## 4. 모바일 우선 반응형 브레이크포인트 원칙

기준은 Tailwind 기본 브레이크포인트를 그대로 채택 (개발팀 커스텀 없을 시 기본값 사용 권장):

| 브레이크포인트 | 최소 너비 | 대상 |
|---|---|---|
| 기본(no prefix) | 0px~ | 모바일 (기본 설계 기준, 1열 레이아웃) |
| `sm` | 640px~ | 큰 모바일 / 작은 태블릿 |
| `md` | 768px~ | 태블릿 (관련 링크 카드 2열 전환) |
| `lg` | 1024px~ | 데스크톱 (3~4열 전환, 사이드바 광고 노출 가능) |
| `xl` | 1280px~ | 와이드 데스크톱 |

### 설계 원칙
1. 모바일 화면에서 "입력 필드 → 결과 표시"가 스크롤 없이 한 화면(1스크린)에 들어오는 것을 최우선 목표로 한다.
2. 계산기 핵심 영역(입력+결과)은 항상 최상단, FAQ/관련 링크/광고는 그 아래 순서로 배치.
3. 터치 타겟 최소 44x44px 확보 (버튼, 입력필드, 아코디언 헤더 등).
4. 데스크톱에서는 좌측 콘텐츠(2/3) + 우측 사이드바(1/3, 광고·관련 링크) 2단 레이아웃 고려 가능.

## 5. 광고(애드센스) 배치 가이드라인

**원칙: 사용자가 계산기를 쓰는 흐름을 방해하지 않는 선에서, 자연스러운 콘텐츠 여백에 배치한다.**
**클릭 유도 배치, 콘텐츠로 오인되는 배치, 실수 클릭 유발 배치는 애드센스 정책 위반이므로 절대 금지.**

### 권장 배치 위치
1. **상단 배너 (선택적)**: 헤더 아래, 계산기 입력 영역 위. 크기를 크게 잡아 입력 영역을 아래로 밀어내지 않도록 높이 제한(모바일 기준 최대 100px 권장).
2. **본문 중간 (콘텐츠 사이 자연 삽입)**: 결과 표시 영역과 FAQ 섹션 "사이", 또는 FAQ와 관련 링크 카드 "사이"의 여백에 배치. 반드시 위아래에 명확한 여백(margin 24px 이상)을 두어 콘텐츠와 시각적으로 분리.
3. **사이드바 (데스크톱 lg 이상에서만)**: 우측 사이드바 영역에 sticky 광고 1개. 스크롤 시 따라오되 화면의 30% 이하 크기.
4. **하단 (푸터 위)**: 관련 링크 카드 섹션 다음, 페이지 최하단.

### 금지 사항 (정책 준수)
- 계산기 "결과 표시 영역" 바로 위/아래에 결과처럼 보이게 디자인된 광고 배치 금지 (오인 클릭 유도).
- 버튼(계산하기, 초기화 등) 근처 30px 이내 광고 배치 금지.
- 광고 주변에 "여기를 클릭하세요", 화살표 등 클릭 유도 그래픽/문구 배치 금지.
- 광고와 콘텐츠 카드의 스타일(테두리, 배경)을 유사하게 만들어 구분을 어렵게 하는 디자인 금지 — 광고 영역은 콘텐츠 카드와 다른 여백/배경으로 명확히 분리하고, 필요시 작은 "광고" 라벨(Caption, Text Secondary) 표기.
- 모바일에서 화면 전체를 덮는 팝업/인터스티셜 형태의 자체 삽입 광고 레이아웃 제안 금지 (UX 저해 + 정책 리스크).

### 레이아웃 우선순위 요약
콘텐츠(입력→결과) 최우선 노출 > 자연스러운 여백 활용 광고 배치 > 광고 개수보다 사용자 체류시간·재방문 우선.

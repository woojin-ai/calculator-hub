## 0. 이 페이지의 성격 (설계 전제)

1. **같은 `date` 카테고리(만나이·D-Day)와 완전히 같은 언어로 만든다.** 새 디자인 언어를 발명하지
   않는다. 입력은 `type="date"`, 인터랙션은 **"계산하기" 버튼식**(만나이·D-Day와 동일), 결과는
   에메랄드 히어로 카드 1장. 사이트 일관성이 최우선.
2. **인터랙션 = 버튼식(실시간 아님) 확정.** 같은 카테고리의 `AgeCalculator`·`DdayCalculator`가
   모두 `form onSubmit` + "계산하기" 버튼 방식이다(코드 확인 완료). 단위변환만 실시간이었던 것은
   "연속 탐색·스왑" 도구라는 예외였고(마스터가 "단발 계산=버튼 / 연속 변환=실시간"으로 이원화
   승인), 근속연수 계산기는 **날짜 2개를 넣고 한 번 확인하는 단발 계산**이므로 만나이·D-Day와
   같은 버튼식이 맞다. → **`form`/`onSubmit`/"계산하기" 버튼 사용, 실시간 파생 렌더 금지.**
3. **YMYL 등급 = 낮음**(기획 §0). 만나이·D-Day와 동급의 순수 날짜 연산. 연봉·대출 같은 상단 근사
   고지 배너·CTA 가드가 **불필요**하다. 결과에 위험/경고 색을 부여하지 않고, 강조는 순수 "값 확인"
   목적의 Accent(에메랄드) 하나로 고정한다(만나이·D-Day 결과 카드와 동일).
4. **연차는 출력이 아니다(기획 §7-2, 마스터 승인).** 결과 카드에 "예상 연차 N일" 같은 개인화 출력
   UI를 **만들지 않는다.** 연차는 결과해석·FAQ 텍스트(공통 템플릿, 범위 밖)로만 등장한다. 결과
   카드가 표시하는 값은 §1-1의 **3개(재직일수·근속기간·소수 근속연수)뿐**이다.
5. **무스크롤 달성 계산기.** 입력이 date 2개뿐이고 결과도 3값으로 콤팩트해서(연봉처럼 공제 10줄이
   없다) 375px에서 입력 → 버튼 → 결과가 사실상 한 흐름에 들어온다(§7-4). design-system 4장 원칙
   1을 문자 그대로 만족하는 몇 안 되는 계산기다.

## 2. 입력 설계 (date 2개 + 계산 버튼)

### 2-1. 결정: **입사일·종료일 세로 스택 (데스크톱·모바일 동일), 계산 버튼식**

**근거:**

- **세로 스택**: 만나이(입력 1개)·D-Day(입력 1개)가 모두 단일 컬럼 `flex flex-col gap-4` 폼이다.
  근속연수는 입력이 2개지만 **"입사일 → 종료일"의 시간 순서(from→to)** 라 위→아래 세로 배치가
  의미적으로 자연스럽고, 기존 date 폼 패턴과 그대로 이어진다. 데스크톱에서 2열로 나란히 두는 안도
  검토했으나(§12), 이중 레이아웃으로 QA 부담만 늘 뿐 이득이 적어 **양쪽 뷰포트 모두 세로 스택으로
  통일**한다.
- **버튼식**: §0-2 근거. 같은 카테고리 만나이·D-Day와 동일하게 `form onSubmit` + "계산하기" 버튼.
  실시간 파생 렌더(단위변환 방식) 아님.
- **편의 기능은 과설계 지양**: "오늘로 설정" 버튼, 캘린더 프리셋(입사 N주년 등) 같은 편의는 기존
  date 계산기에 없고, 종료일 미입력 시 자동으로 오늘을 쓰는 것(§1-0)으로 충분하다. **추가하지
  않는다.** (만나이·D-Day 톤과 맞춤.)

### 2-2. 필드별 스펙

design-system 3-1 + 만나이/D-Day의 date input 스타일을 **그대로** 재사용한다. 연봉의 `inputBase`
상수와 동일 문자열이므로 `inputBase` 상수로 뽑아 재사용해도 좋다.

| 순서 | 필드 | id | 라벨 | 컨트롤 | 필수 | 도움말(Caption, 선택 노출) |
|---|---|---|---|---|---|---|
| 1 | 입사일 | `startDate` | **"입사일"** | `type="date"` | 필수 | "재직을 시작한 첫날(첫 근무일)을 선택하세요." |
| 2 | 종료일 | `endDate` | **"퇴사일 (미입력 시 오늘)"** | `type="date"` | 선택 | "비워 두면 오늘 날짜까지 계산합니다. 퇴사 예정일 등 미래 날짜도 입력할 수 있습니다." |

- **입력 스타일(만나이·D-Day와 동일)**:
  `h-12 rounded-lg border px-4 text-base text-brand-text outline-none transition-colors
  focus:border-brand-primary focus:ring-4 focus:ring-brand-primary/15`, 기본 테두리
  `border-brand-border`, 오류 시 `border-brand-warning`로 토글(만나이의 `error ? ... : ...` 패턴
  그대로).
- 라벨: `text-sm font-medium text-brand-text-secondary`(만나이·D-Day 라벨과 동일).
- 각 필드 컨테이너: `flex flex-col gap-1.5`(만나이 패턴).
- **`max` 속성**: 만나이는 미래 생일을 막으려 `max={오늘}`을 걸지만, **이 계산기는 미래 종료일을
  허용(기획 §1-0, 퇴사 예정일 조회 수요)** 하므로 `startDate`·`endDate` 어디에도 `max`를 걸지
  않는다. (입사일에 상한을 걸 이유도 없다.)
- 도움말 Caption은 `text-xs text-brand-text-secondary`(연봉 도움말 톤). 세로를 늘리므로 필수는
  아니며, 최소한 종료일의 "미입력 시 오늘/미래 허용" 안내 1줄은 권장.

### 2-3. 계산 버튼

- 만나이·D-Day와 동일: `h-12 rounded-lg bg-brand-primary px-5 text-base font-semibold text-white
  transition-colors hover:bg-brand-primary-hover`. 라벨 **"근속연수 계산하기"**.
- 폭: 만나이·D-Day는 카드 폭에 맞춰 자연 확장(명시적 `w-full` 없이 flex 컨테이너 폭). 연봉은
  `w-full` 명시. 어느 쪽이든 전체 폭이 되게 하면 된다(48px 터치타깃 확보).
- 개인정보 안내 캡션(연봉의 폼 하단 문구와 동일 톤)을 버튼 아래에 1줄 넣는 것을 권장:
  "입력하신 입사일·퇴사일은 브라우저 안에서만 계산되며 서버에 저장·전송되지 않습니다."
  (기획 §3 FAQ 5번·interpretation ⑤단과 일관, YMYL-낮음이라 필수는 아니나 신뢰도에 도움.)

### 2-4. 입력 검증 (기획 §1-0)

- **필수(입사일 미입력)**: 입사일 없이 계산 시 `startDate` 테두리 `border-brand-warning` + 하단
  `text-xs text-brand-warning` 메시지 `role="alert"`. 문구 예: "입사일을 선택해주세요."(만나이의
  "생년월일을 입력해주세요." 톤).
- **`종료일 < 입사일`(핵심 검증, 기획 §1-0)**: 계산하지 않고 안내한다. **하드 에러 배너가 아니라
  안내 수준**(YMYL-낮음). 위치·톤은 아래 §3-4에서 확정.
- **미래 종료일**: 허용. 오류 아님(퇴사 예정일 조회).
- 정확한 검증 트리거·문구 세부는 개발팀 재량이며, 본 문서는 **시각 스타일과 배치**만 규정한다
  (만나이·D-Day와 동일 원칙).

## 3. 결과 카드 설계 — [기획 §7-3에 대한 디자인팀 확정안]

### 3-0. 위계 확정 (기획 §7-3 제안 채택 + 근거)

기획팀 §7-3 제안(= (b) `N년 M개월 D일`을 히어로 대표값, (a) 재직일수·(c) 소수 근속연수를 보조)을
**그대로 확정한다.** 근거:

- 이 페이지에 오는 핵심 질문은 "내가 **몇 년 몇 개월** 다녔나?"이고, 사람이 근속을 말할 때 가장
  직관적으로 쓰는 단위가 `N년 M개월 D일`이다. 총 재직일수(1,993일)는 정확하지만 "그래서 몇 년?"이
  한눈에 안 들어와 히어로로는 약하다. → **(b)를 히어로**로.
- (a) 재직일수는 "정확한 총 일수"로 퇴직금·검증에 쓰이는 보조 근거값, (c) 소수 근속연수는 "대략
  몇 년" 게이지 → 둘 다 히어로 아래 **보조 2행**으로.
- 만나이·D-Day도 "대표값 1개(에메랄드 히어로) + 보조 정보 몇 줄" 구조다. 이 위계가 그 패턴과
  정확히 일치한다.

**3-tier 정리(연봉의 3-tier를 이 계산기 규모에 맞게 축약):**

| Tier | 내용 | 강조/배치 | 노출 |
|---|---|---|---|
| **① 대표값** | **(b) 근속기간 `N년 M개월 D일`** | Result 스케일(`text-4xl sm:text-[2.5rem]`, Bold, Accent). **숫자는 크게(Accent), 년/개월/일 단위는 작게**(`text-base` Text Secondary) | 항상 |
| ① 요약 문구 | 기획 §2 1문단(동적) 압축 1~2줄 | `mt-2 text-sm text-brand-text-secondary` | 항상 |
| ①-보조 | "오늘 기준" 표기 | `text-xs text-brand-text-secondary` | 종료일 미입력 시에만 |
| **② 보조값 2행** | (a) 총 재직일수 `N일` / (c) 근속연수(소수) `약 N.N년` | `flex justify-between py-1.5 text-sm`(연봉 `AmountRow`와 동일 패턴, 단 값은 문자열) | 항상 |

- **접이식(연봉 Tier ③) 없음.** 값이 3개뿐이라 숨길 것이 없다. 3값 모두 항상 노출해도 카드가
  콤팩트하다(§7-4).

### 3-1. 결과 카드 공통 골격 (만나이·D-Day·연봉 결과 카드와 동일)

`mt-6 rounded-xl border-l-4 border-brand-accent bg-gradient-to-br from-emerald-50 to-white
p-4 sm:p-6`. Accent 고정(상태별 색 분기 없음 — 근속연수에 위험/경고 색 부여 안 함). 계산 후에만
조건부 렌더(`result && (...)`, 만나이·D-Day와 동일). 갱신 시 `transition-colors duration-200`은
선택(버튼식이라 플리커 우려 없음).

- **결과 스크롤**: 만나이·D-Day는 계산 후 `scrollIntoView`를 쓰지 않는다(폼이 짧아 결과가 버튼
  바로 아래 노출). 이 계산기도 입력이 date 2개로 짧으므로 **동일하게 스무스 스크롤 없이** 둔다
  (연봉의 `resultRef` 스크롤은 입력 폼이 길 때의 장치라 여기선 불필요). 개발팀이 필요하다고
  판단하면 추가는 무해하나 기본은 만나이·D-Day와 통일.

### 3-2. Tier ① — 대표값(근속기간) 렌더 (항상)

`N년 M개월 D일`은 단일 숫자가 아니라 3부분 복합값이라, **AgeCalculator의 "숫자 크게 + 단위 작게"
분리 렌더**(`<span big>{age}</span><span small>세</span>`)를 3부분으로 확장한다. 히어로 폭을
줄이고 design-system 3-2 단위 규칙(값은 크게, 단위는 작게)에 맞다.

```
근속기간                                  ← 라벨: text-sm font-medium text-brand-text-secondary
5 년  5 개월  14 일                        ← 히어로: 숫자 span = text-4xl sm:text-[2.5rem] font-bold
                                             tabular-nums text-brand-accent
                                           단위(년·개월·일) span = text-base font-medium
                                             text-brand-text-secondary
입사일 2020년 3월 2일부터 2025년 8월 15일   ← 동적 요약(§3-5): mt-2 text-sm text-brand-text-secondary
까지 총 1,993일 근무한 것으로 계산됩니다.
```

- 히어로 컨테이너: `mt-1 flex flex-wrap items-baseline gap-x-2 gap-y-1 tabular-nums`
  (AgeCalculator의 `flex items-baseline gap-2 tabular-nums` 확장). **`flex-wrap`**: 최장값
  ("10년 11개월 30일" 등)이 375px 카드 안쪽(약 311px)을 넘을 극단 상황에서 줄바꿈으로 안전하게
  흐르게 한다(가로 오버플로 0). 일반값은 한 줄에 들어온다.
- 각 부분: `<span>{y}</span><span 단위>년</span>` 형태로 년/개월/일 3세트. 숫자는 Accent 큰 폰트,
  단위는 작은 Text Secondary. (예: **5**`년` **5**`개월` **14**`일`.)
- 히어로 값 컨테이너에 `aria-label`로 "근속기간 5년 5개월 14일"처럼 자연어 낭독을 보장하는 것을
  권장(분리 span이 스크린리더에서 "5 년 5 개월"로 끊겨 읽히지 않도록). 필수는 아님.
- 0 표기: `0년 0개월 1일`(앵커 E)처럼 0인 자리도 **생략하지 않고 표시**한다(엔진이 항상 3부분을
  주므로). "0년"을 감추면 위계가 흔들리고 앵커 검증도 어긋난다.

### 3-3. Tier ② — 보조값 2행 (항상)

Tier ① 아래 구분선(`mt-4 pt-4 border-t border-brand-border/60`, 연봉 Tier② 패턴) 다음, 보조값
2행을 나열한다. 연봉 `AmountRow`와 **같은 시각 패턴**(`flex items-center justify-between py-1.5
text-sm`, 라벨 좌 `text-brand-text-secondary` / 값 우 `tabular-nums`)이되, 값이 "원 금액"이
아니라 문자열(`1,993일`, `약 5.5년`)이라 `AmountRow`를 그대로 쓰지 못한다 → **문자열 값 전용
경량 행**을 컴포넌트 내부에 둔다(§7 매핑, §11-신규요소).

```
총 재직일수            1,993일           ← 라벨 좌 / 값 우(tabular-nums, font-semibold)
근속연수(소수)         약 5.5년
```

- 각 행: `flex items-center justify-between py-1.5 text-sm`.
  - 라벨(좌): `text-brand-text-secondary`.
  - 값(우): `tabular-nums font-semibold text-brand-text`(Accent 아님 — 컬러 강조는 히어로 전용,
    시선 위계 유지. 연봉 `AmountRow` `emphasis`와 동일 톤).
- (a) 재직일수: 정수부 **천단위 콤마**(`toLocaleString("ko-KR")`) + `일`. 예 `1,993일`.
- (c) 소수 근속연수: 기획 §1-1대로 소수 **첫째 자리 반올림** + "약 " 접두 + `년`. 예 `약 5.5년`.
  (앵커 E는 `약 0.0년` — 0.0도 그대로 표기.)
- **"근속연수(소수)" 라벨**: (c)가 "5.5년"처럼 소수라 (b) 근속기간과 헷갈릴 수 있으므로 라벨에
  "(소수)"를 붙여 구분한다. 대안 라벨 "환산 근속연수"도 가능(개발팀/기획 카피 재량, 의미는 동일).

### 3-4. "오늘 기준" 표기 + `종료일 < 입사일` 안내 (기획 §7-3 디자인 반영 요청 응답)

**(A) "오늘 기준" 표기 (종료일 미입력 시):**

- **위치**: Tier ① 동적 요약 문구 **바로 아래**, `text-xs text-brand-text-secondary` 1줄.
  문구 예: **"2026년 7월 14일 오늘을 기준으로 계산했습니다."**(날짜는 `formatKoreanDate` 등 기존
  D-Day 유틸 재사용). 종료일을 직접 입력한 경우엔 이 줄을 **숨긴다.**
- **왜 여기인가**: 사용자가 종료일을 비웠을 때 "그럼 언제까지 계산된 거지?"가 곧바로 궁금해진다.
  대표값(히어로) 직후·요약 문구 옆에 붙여야 "이 결과가 오늘까지 기준"임이 즉시 이해된다. 동적
  요약 문구 자체도 "…오늘(2026년 7월 14일)까지…"로 종료일을 자연어로 포함하게 조립하면
  (§3-5) 이 caption과 함께 이중으로 명확해진다. (요약이 이미 날짜를 포함하면 caption은 짧게
  "오늘 기준"만 남겨 중복을 줄여도 된다 — 카피 최종은 기획, §11.)
- 시각적으로 과하지 않게: 별도 배지/박스 없이 Caption 텍스트 1줄로 충분(YMYL-낮음, 만나이·D-Day의
  보조 정보 톤과 동일).

**(B) `종료일 < 입사일` 안내:**

- **위치·톤**: 결과 카드를 렌더하지 않고, **`endDate` 입력 필드 하단**에 만나이·D-Day의 인라인
  검증 메시지와 **동일한 방식**으로 노출한다 — `endDate` 테두리 `border-brand-warning` + 하단
  `text-xs text-brand-warning` `role="alert"`. 문구 예(기획 §1-0): **"퇴사일이 입사일보다 빠릅니다.
  날짜를 다시 확인해주세요."**
- **왜 결과 카드가 아니라 입력부인가**: 오류의 원인이 "입력한 종료일"이므로, 사용자가 고쳐야 할
  그 필드 옆에 메시지를 두는 것이 만나이의 "미래 생일" 검증(입력 필드 하단 `role="alert"`)과 동일
  동선이고 수정이 빠르다. 결과 영역에 빨간 배너를 띄우는 방식은 만나이·D-Day에 없는 언어라
  도입하지 않는다(YMYL-낮음, 사이트 일관성).

### 3-5. Tier ① 요약 문구 = 기획 §2 1문단 동적 배치

기획 §2 1문단(앵커C 예시: "입사일 2020년 3월 2일부터…약 5.5년입니다.")은 정적 텍스트가 아니라
**사용자 입력값 기반 동적 생성**이어야 한다(기획 §7-6, 기존 라이브 계산기 전부 동일). 이 문서는
**배치 위치만 확정**: Tier ① 히어로 바로 아래(`mt-2 text-sm text-brand-text-secondary`). BMI·연봉의
`getResultSummary(...)`처럼 컴포넌트 내부 함수로 입력(입사일·종료일)·결과(재직일수·근속기간·소수년)를
받아 1~2문장을 조립한다. **정적 텍스트 금지.**

- 길이: 모바일 2줄 이내 권장(길면 히어로 위계 방해). 3값을 모두 자연어로 녹이되 간결하게. 예(동적):
  "입사일 {입사일}부터 {종료일 또는 '오늘'}까지 총 {재직일수}일 근무해, 근속기간은 {N년 M개월
  D일}입니다."
- 종료일 미입력이면 "{종료일}" 자리에 "오늘(2026년 7월 14일)"을 넣고 §3-4(A) caption과 호응.
- 정식 카피 톤은 기획 §2 담당(§11). YMYL-낮음이라 이직/퇴사 부추김 표현은 넣지 않는다(기획도 §2에
  명시).

## 5. 표시 / 포맷 규칙

- **천단위 콤마**: (a) 재직일수는 정수부 천단위 콤마(`toLocaleString("ko-KR")`) — `1,993일`,
  `1,097일`. 근속기간(b)의 각 숫자(년/개월/일)는 최대 3자리라 콤마 불필요. 모든 결과 숫자는
  `tabular-nums`(design-system 2장, 자릿수 정렬).
- **소수 근속연수(c)**: 소수 **첫째 자리 반올림 고정 표기**(기획 §1-1). `5.5년`, `3.0년`, `0.0년` —
  후행 0을 **절삭하지 않는다**(단위변환과 다른 규칙). "3년"이 아니라 "3.0년"으로 표기해야 "소수
  환산값"임이 드러나고 앵커(3.0/5.5/0.0)와 일치한다.
- **date input 값 표시**: 브라우저 네이티브 `type="date"`가 로캘 형식으로 표기(만나이·D-Day와
  동일, 별도 서식 없음). 결과·요약의 날짜 표기는 D-Day의 `formatKoreanDate`("2025년 8월 15일")
  재사용.
- **입력에는 서식을 적용하지 않는다**(date input이므로 콤마·마이너스 이슈 없음).

## 6. 반응형 (모바일 375 / 데스크톱 1280)

### 6-1. 입력부 — 세로 스택 (양쪽 뷰포트 동일)

§2-1대로 입사일·종료일·버튼을 `flex flex-col gap-4` 세로 스택으로 통일한다. 데스크톱(1280,
`max-w-3xl` ≈ 768px 내)에서도 동일 구조(폭만 넓어짐). date input은 브라우저 네이티브 위젯이라
375px에서도 잘림 없이 표기된다.

### 6-2. 결과 카드 — 가로 오버플로 0 규칙 (필수)

- **히어로(근속기간)**: `flex flex-wrap items-baseline`(§3-2)로 최장값도 줄바꿈으로 흐른다. 일반값
  ("5년 5개월 14일" ≈ 375px 카드에서 한 줄)은 `text-4xl`(36px)로 무리 없이 들어간다. design-system
  Result 스케일 "모바일 28px 이하 축소 금지"는 지켜진다(줄바꿈으로 대응하므로 폰트 축소 불필요).
- **보조값 2행**: `flex items-center justify-between`(라벨 좌 줄바꿈 허용 / 값 우 `tabular-nums`
  고정). 다열 `<table>` 미사용 → **375px에서 가로 스크롤이 구조적으로 발생할 수 없다**(연봉·단위변환
  스펙과 동일 원칙).
- **요약 문구**: 일반 문단이라 세로로만 늘어남, 폭 이슈 없음.

### 6-3. 무스크롤 목표 — 이 계산기는 달성

iPhone SE급(375×667, 실사용 높이 약 550~600px) 개략 높이 추정.

| 영역 | 예상 높이 |
|---|---|
| 입사일 필드(라벨 + `h-12` + 도움말 1줄) | 약 84px |
| gap-4 | 16px |
| 종료일 필드(라벨 + `h-12` + 도움말 1줄) | 약 84px |
| gap-4 | 16px |
| 계산 버튼(`h-12`) + 개인정보 캡션 | 약 72px |
| 카드 상하 padding(`p-4`) | 32px |
| **입력 폼까지 소계** | **약 304px** |
| 결과 카드(히어로 + 요약 2줄 + 구분선 + 보조 2행, padding 포함) | 약 210~240px |
| **입력 + 결과 합계** | **약 514~544px** |

→ 도움말 Caption을 접거나 한 줄로 줄이면 실사용 뷰포트(약 550~600px) 안에 입력~결과가 대체로
들어온다. 최소한 **입력부는 above-the-fold**이고 결과는 버튼 바로 아래 노출되어, 만나이·D-Day와
동일한 "짧은 폼 + 즉시 결과" 경험을 준다. 연봉처럼 결과 스크롤 장치가 필요 없다(§3-1).

### 6-4. 부수 규칙

- date input·버튼 모두 `h-12`(48px) 유지 — 터치타깃(design-system 4장 원칙 3, §8).
- 데스크톱에서 카드 폭이 넓어도 히어로/보조행 구조는 동일(리플로우만). 히어로가 과도하게
  넓어지지 않도록 카드 자체가 `max-w-3xl` 안이라 문제없음.

## 7. 접근성 / 입력 상태 처리

- **라벨 연결**: 두 date input 모두 `<label htmlFor>`로 명시 연결(만나이·D-Day 패턴). 스크린리더
  낭독 보장.
- **터치타깃**: date input·버튼 `h-12`(48px ≥ design-system 44/48px 기준).
- **대비**: 라벨/값 모두 기존 `text-brand-text`·`text-brand-text-secondary` 토큰 사용(design-system
  대비 기준 통과분). 결과 값(Accent `#10B981`)은 히어로 큰 폰트에만 적용, 본문 텍스트에는 미사용
  (대비 안전).
- **검증 메시지**: `role="alert"` + 해당 입력 테두리 `border-brand-warning`(만나이·D-Day와 동일).
  §2-4·§3-4(B).
- **히어로 낭독**: 분리 span 히어로에 자연어 `aria-label` 권장(§3-2).
- **결과 갱신 안내**: 버튼식이라 만나이·D-Day처럼 별도 `aria-live`는 필수 아님(계산 클릭 후 결과가
  DOM에 나타나며 포커스 흐름상 인지 가능). 단위변환(실시간)의 `aria-live`는 여기 불필요.

### 다크모드

- **현재 사이트에 다크모드 없음.** 기존 계산기 컴포넌트(만나이·D-Day·연봉·단위변환) 모두 `dark:`
  변형 클래스를 쓰지 않고 `brand-*` 라이트 토큰만 사용한다(코드 확인). 이 컴포넌트도 **동일하게
  다크모드 분기를 두지 않는다.** (사이트 전역 다크모드는 별도 톤앤매너 결정 사안 — 도입 시 전
  계산기 일괄 대응, 이 문서 범위 밖.)

## 8. 컴포넌트 / 데이터 매핑 (개발팀 참고 — 그대로 구현)

## 9. 디자인시스템 신규 요소 (편입 판단)

이 컴포넌트가 새로 쓰는 시각 요소:

- (a) **`N년 M개월 D일` 분리-span 히어로**(숫자 크게/단위 작게 3세트, `flex-wrap`) — AgeCalculator의
  2-span 히어로를 3-span으로 확장한 변형.
- (b) **문자열 값 라벨-값 행**(연봉 `AmountRow`의 "숫자+원" 아닌 자유 문자열 버전, §3-3).

→ **정식 design-system.md 편입은 보류**(마스터가 단위변환·연봉 스펙에서 내린 것과 동일 판단:
"컴포넌트 내 사용만, 정식 편입은 신규 패턴이 누적되면 일괄 재검토"). 둘 다 **기존 패턴의 경미한
변형**이라 새 컴포넌트 언어 도입이 아니다. 최종 판단은 마스터(§11).

## 10. 광고 배치 (design-system.md 5장 + 이 페이지 규칙)

- **상단 배너 광고: 넣지 않는다.** 입력(date 2개) + 결과를 above-the-fold에 두는 것이 이 콤팩트
  도구의 핵심(§6-3). YMYL-낮음이라 근사 고지 배너도 없어 최상단은 계산기 카드가 차지한다.
- **인터랙티브 클러스터(입력 ↔ 버튼 ↔ 결과 카드) 내부/사이에는 광고를 넣지 않는다.** 하나의 도구
  흐름이라 중간 삽입은 오인 클릭·흐름 방해.
- 본문 중간/하단 광고: 공통 템플릿이 처리(범위 밖). **결과 카드 다음(설명글/FAQ 사이)** 여백부터
  배치 가능. design-system 5장 공통 규칙(위아래 margin 24px+, "광고" Caption 라벨, 결과 카드의
  에메랄드 틴트/좌측 강조선과 유사하지 않은 중립 배경으로 명확히 구분) 준수. 클릭 유도 배치 금지.

## 11. 마스터 확인 필요 / 타팀 이관 (핑퐁)

**마스터 팀장 확인 필요**
1. **결과 위계 확정 승인(§3-0)**: 기획 §7-3 제안대로 (b) `N년 M개월 D일`을 히어로, (a)·(c)를 보조
   2행으로 확정했다. 이 위계가 맞는지 확인 요청(기획 제안 그대로라 조정 불필요할 것으로 봄).
2. **design-system.md 신규 요소 편입 = 보류로 제안(§9)**: 분리-span 히어로·문자열 값 행 2종. 기존
   패턴 변형이라 단위변환·연봉 때와 동일하게 "컴포넌트 내 사용만, 정식 편입은 추후 일괄"로 제안.
   최종 판단 요망.

**기획팀 확인 요청(카피 — 콘텐츠 소관)**
- **동적 요약 문구 1~2줄 템플릿(§3-5)** 최종 문안·톤. "오늘 기준" caption(§3-4 A)과 요약 문구의
  날짜 표현 중복을 어느 선까지 둘지(요약이 날짜 포함이면 caption은 "오늘 기준"만 짧게).
- **`종료일 < 입사일` 안내 문구**(§3-4 B), **입사일 미입력 문구**(§2-4) 최종 카피.
- **보조값 (c) 라벨** "근속연수(소수)" vs "환산 근속연수" 중 택1(§3-3, 의미 동일).

**개발팀 전달 사항**
- `components/ServicePeriodCalculator.tsx`(§8) 신설 — **`AgeCalculator`/`DdayCalculator` 구조 복제**
  (form onSubmit + "계산하기" 버튼, 실시간 아님). `lib/service-period.ts`(기획 §1-2 알고리즘,
  §7-5 3건 준수). `CALCULATOR_COMPONENTS` 매핑 추가, `lib/calculators.ts` 콘텐츠 확보 후
  `status: "live"`.
- 엔진 출력이 기획 §1-3 앵커 A~E(1,097 / 1,096 / **1,993·5년5개월14일·5.5년(대표)** / 1,425 / 1)와
  일치하는지 회귀 테스트(앵커 D "오늘"은 접속일마다 바뀌므로 동작 확인용, 수치 고정 테스트는
  A·B·C·E).
- 결과 요약은 동적 생성(§3-5), 소수 근속연수 후행 0 절삭 금지(§5), (a)는 천단위 콤마(§5).

## 12. 대안으로 검토했으나 채택하지 않은 안

- **데스크톱 입사일/종료일 2열 나란히**: 폭 활용엔 좋으나 "입사→종료" 시간 순서 메타포가 좌우로
  가면 약해지고, 데스크톱/모바일 이중 레이아웃으로 QA·오버플로 리스크만 증가. 만나이·D-Day의
  단일 컬럼 폼과도 어긋난다 → 양쪽 세로 스택 통일(§2-1).
- **실시간 계산(단위변환 방식)**: 근속연수는 날짜 2개를 넣고 한 번 확인하는 단발 계산이라 실시간
  이점이 없고, 같은 카테고리 만나이·D-Day가 버튼식이다. 사이트 이원화 원칙("단발=버튼")대로
  버튼식 채택(§0-2).
- **(a) 재직일수를 히어로로**: "총 일수(1,993일)"는 정확하나 "몇 년?"이 한눈에 안 들어와 대표값
  으로 약함. (b) 근속기간을 히어로, (a)를 보조로(§3-0).
- **연차 예상 일수 출력 UI**: 기획 §7-2·마스터 승인대로 **출력 아님**. YMYL(회계연도/입사일 기준
  상이) 리스크로 개인화 연차 숫자는 만들지 않고 해설·FAQ 텍스트로만 처리. 결과 카드에 넣지 않음.
- **`종료일 < 입사일` 결과 영역 빨간 배너**: 만나이·D-Day에 없는 언어. 오류 원인 필드(종료일)
  하단 인라인 검증으로 처리(§3-4 B).
- **"오늘로 설정" 버튼 등 date 편의 프리셋**: 기존 date 계산기에 없고 종료일 자동-오늘로 충분.
  과설계 지양(§2-1).

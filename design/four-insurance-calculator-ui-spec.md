## 0. 이 페이지의 성격 (설계 전제)

1. **YMYL 등급 = 낮음~중간(기획 §0 승계).** 연봉(중간)보다 약간 낮다("이미 정해진 요율의 단순
   곱셈"으로 세금 근사 리스크 없음). 다만 (1) 요율이 매년 갱신되어 하드코딩 방치 시 오정보,
   (2) 사업주 부담(고용보험 규모별)·산재보험(업종별)이 조건에 따라 달라 "참고용" 고지가 필요하다.
   → **연봉·전기요금·중도상환과 동일하게 상단 amber `role="note"` 안내 배너를 항상 노출**한다(§6-1).
   결과에 위험/경고 색(빨강)을 부여하지 않고, 강조는 순수 "값 확인" 목적의 Accent(에메랄드/
   `brand-primary`) 하나로 고정한다.
2. **인터랙션 = 버튼식(실시간 아님) 확정(마스터 지시).** 상단 amber 근사 배너 YMYL family(연봉·
   퇴직·전기요금·중도상환)와 동일하게 `form onSubmit` + **"계산하기"** 버튼을 쓴다. 실시간 계산은
   사이트 원칙상 단위변환 전용이다. 급여(민감정보) 타이핑 중 부분값 깜빡임을 막고 "확인 후 결과"의
   YMYL 리듬을 유지한다. → 실시간 파생 렌더 금지.
3. **입력 2개(급여 1 + 규모 1) · 출력 = 근로자/사업주/합계 3열 구조가 핵심 — 기획 §7 확정안 그대로.**
   입력은 가볍지만(월급 1개 + 규모 select), **출력의 핵심 차별점은 "근로자 부담 / 사업주 부담 /
   합계"를 함께 보여주는 3열 구조**다(기존 연봉 계산기가 근로자분만 보여준 것과 다름). 연봉의
   3-tier를 이 계산기에 맞춰 "대표값(근로자 합계) + 3-way 요약(근로자/사업주/총) + 4보험 항목별
   내역(근로자/사업주/소계)"으로 적용한다(§3-0).
4. **YMYL CTA 금지.** 급여·노무·보험 상품/컨설팅 가입 유도 등 상업적 2차 버튼/문구를 계산기 카드
   안팎에 절대 넣지 않는다(§6-3). 계산 버튼 1개만 존재. 4대 보험료 정보의 신뢰도가 핵심이다.
5. **연봉 계산기의 보조 페이지(정합 최우선, 기획 §배경).** 근로자 열은 연봉 계산기에 같은 월
   과세대상급여를 넣었을 때와 **원 단위까지 동일**해야 한다. 이는 로직 영역이나(개발), 화면에서도
   입력 라벨을 연봉 계산기와 같은 개념("월 과세대상 급여")으로 못박아 사용자가 두 계산기를 오가며
   혼동하지 않게 한다(§2-1).

## 2. 입력 설계 (월 과세대상 급여 1 + 사업장 규모 1 + 계산 버튼)

### 2-1. 배치 원칙 — 2개 전체 폭 세로 스택(2열 미채택)

입력 2개를 **모두 전체 폭 세로 스택**한다(데스크톱·모바일 동일). 2열(`grid-cols-2`)을 쓰지 않는
이유: 규모 select 아래 보조문구가 2~3줄로 길고(사업주 열에만 영향 설명), 급여 필드 아래 보조문구도
2줄이라 2열로 묶으면 좌/우 caption 높이가 어긋난다(전기요금 §2-1·중도상환 §2-0에서 확인된 문제).
YMYL 안내를 온전히 노출하는 편을 택한다. 2개뿐이라 스택이 길지 않다.

### 2-2. 필드별 스펙 (순서 고정 ①② = 기획 §7-1과 1:1)

`inputBase` = `"h-12 rounded-lg border px-4 text-base text-brand-text outline-none transition-colors
focus:border-brand-primary focus:ring-4 focus:ring-brand-primary/15"` (오류 시 `border-brand-warning`,
평시 `border-brand-border` 토글) — 연봉·퇴직·전기요금·중도상환과 동일 상수. 각 필드 컨테이너
`flex flex-col gap-1.5`, 라벨 `text-sm font-medium text-brand-text-secondary`, 보조문구(Caption)
`text-xs text-brand-text-secondary`. 우측 단위 접미(`원`)는 전기요금 `kWh` 어드먼트 패턴(`relative`
컨테이너 + `pr-12` + `absolute right-4 text-brand-text-secondary`) 재사용.

### 2-4. 입력 검증 (기획 §1-0) — 필드별 인라인, `role="alert"`, 기존 계산기와 일관

퇴직·전기요금·중도상환의 `FieldError`(필드 하단 인라인 메시지) 패턴을 쓴다. **결과 영역 빨간 배너

`border-brand-warning` 토글(`aria-invalid`). **경보색은 `brand-warning`(amber 계열)이며 빨강 아님**

### 2-5. 사업장 규모 상태 문구 (마스터 지시 — "사업주 열에만 영향" 명시)

기획 §7-3 요청 반영. 규모 select가 **오직 사업주 고용보험 부담에만 영향**을 준다는 사실을 두 지점에서
전달해 "규모를 바꿨는데 근로자 부담이 왜 안 변하지?" 혼동을 방지한다.

- **입력 지점(항상)**: 규모 select 아래 Caption(§2-2 보조문구) — "회사(사업주)의 고용보험 부담
  계산에만 쓰입니다. 근로자 부담과 국민연금·건강보험·장기요양보험에는 영향을 주지 않습니다."
- **결과 지점(계산 후)**: Tier③ 하단 각주 — "사업주 열은 사업장 규모(**{선택된 규모}**) 기준입니다."
  (규모를 바꿔 재계산하면 사업주 고용보험·사업주 합계·총 합계만 갱신되고 근로자 열은 불변임을

## 3. 결과 카드 설계 — [기획 §7-2·§7-3에 대한 디자인팀 확정안]

### 3-0. 정보구조 확정 (근로자 합계 히어로 + 3-way 요약 + 4보험 항목별 내역, 전부 항상 노출)

기획 §7-2 출력(대표값 = 근로자 부담 합계 + 사업주 부담 합계 + 총 합계 / 4보험 × 근로자·사업주·소계 /
"참고용 예상 보험료" 배지 / "산재보험 제외" 각주)을 **그대로 채택**하고, 연봉의 3-tier를 이 계산기의
"3열(근로자/사업주/합계)이 핵심"이라는 성격에 맞춰 적용한다.

> **결정 1 — 대표값(히어로) = 근로자 부담 합계.** 사용자가 가장 궁금한 값은 "내 급여에서 매달
> 실제로 빠지는 4대 보험료"이므로 이 숫자 하나만 Accent 히어로로 키운다(마스터 확정).
>
> **결정 2 — Tier② = 근로자/사업주/총 3-way 요약(항상 노출).** 이 계산기의 **핵심 차별점인 "3열
> 구조"를 합계 수준에서 한 시야에** 보여준다. 데스크톱은 3셀 가로 병렬(grid-cols-3), 모바일은 3행
> 라벨-값 스택. 히어로(근로자 합계)와 값이 겹치지만, 이는 연봉이 히어로(월 실수령)를 요약행에서
> 다시 보여 검산하게 한 것과 동형이며 "근로자↔사업주↔총" 비교의 축이 된다.
>
> **결정 3 — Tier③ 4보험 항목별 내역 = 접이식 미채택, 항상 노출.** 전기요금(보조 8값)은 접이식이
> 필요했지만, 이 계산기는 **"회사가 각 보험을 얼마 내는지 항목별로 보는 것"이 곧 페이지의 존재
> 이유**다(연봉과의 차별점). 4개 보험뿐이라 카드가 과하게 길지 않고, 항목을 접으면 이 계산기의
> 핵심 가치를 숨기는 셈이 된다(중도상환이 근거 4행을 항상 노출한 것과 동일 판단). → **항상 노출.**
> 다만 기획팀이 "모바일 높이 부담이 커 접이식이 낫다"고 판단하면 회신만으로 `<details>` 기본
> 접힘으로 조정 가능(§9).

**출력 항목 tier 배정 — 기획 §7-2와 1:1(추가·삭제·치환 없음):**

| Tier | 항목 | 노출 | 근거 |
|---|---|---|---|
| **① 대표값** | 근로자 부담 합계 `N원` + "참고용 예상 보험료" 배지 | 항상 | 사용자 핵심 질문 "매달 얼마 빠지나?" |
| ① chip | 월 과세급여 · 사업장 규모 (예: "월 과세급여 300만원 · 150인 미만") | 항상 | 대표값의 맥락 |
| ① 요약 | 기획 §2 1문단(동적) 1~2줄 | 항상 | §3-7 |
| **② 3-way 요약** | 근로자 부담 합계 / 사업주 부담 합계(산재 제외 각주) / 총 합계 | 항상 | **3열 핵심**을 합계 수준에서(§3-3) |
| **③ 항목별 내역** | 국민연금 / 건강보험 / 장기요양 / 고용보험 × (근로자 / 사업주 / 소계) + 합계 행 | 항상 | 4보험 개별 부담(§3-4). 회사 부담 항목별 = 차별점 |
| ③ 노사절반 안내 | "국민연금·건강·장기요양은 회사가 같은 금액 함께 부담(노사 절반)…" 1줄 | 항상 | 대칭값의 의미 전달(§3-5) |
| ③ 각주 | "사업주 열 = 규모 기준 · 산재보험 제외(전액 사업주·업종별)" | 항상 | 산재 안내 + 규모 상태(§2-5·§3-5) |

- **항목 무변경**: 기획 §7-2 항목(대표 3값 + 4보험×근로자/사업주/소계 + 배지 + 산재 각주)을 그대로
  전부 노출. 추가 화폐 항목(예 산재 추정액, 총급여 대비 %)은 **만들지 않는다**. 소계(보험별 근로자
  +사업주)는 기획 §7-2 "각 행에 근로자·사업주·소계"에 명시된 값이지 신규 항목이 아니다(§8 대조표).

### 3-1. 결과 카드 공통 골격 (기존 계산기 결과 카드와 동일)

`mt-6 rounded-xl border-l-4 border-brand-accent bg-gradient-to-br from-emerald-50 to-white p-4
transition-colors duration-200 sm:p-6`. Accent 고정(상태별 색 분기 없음 — 보험료·부담 주체에
위험/경고 색 부여 안 함). 계산 후에만 조건부 렌더(`result && (...)`). **`resultRef`로 계산 후 스무스
스크롤**(연봉·전기요금·중도상환 패턴 재사용).

### 3-2. Tier ① — 대표값(근로자 부담 합계) 렌더 (항상)

전기요금/중도상환 히어로(`formatWon` + "원" 작게 분리 + `flex-wrap`) + "참고용" 배지 패턴 결합.

```
근로자 부담 합계  [참고용 예상 보험료]   ← 라벨: text-sm font-medium text-brand-text-secondary
                                         배지: ml-2 rounded bg-brand-border/40 px-1.5 py-0.5
                                         text-xs font-medium text-brand-text-secondary
291,310 원                              ← 히어로: text-4xl sm:text-[2.5rem] font-bold tabular-nums
                                         text-brand-accent, "원"은 text-base font-medium secondary
[월 과세급여 300만원 · 150인 미만]       ← chip pill: mt-1 inline-block rounded-full
                                         bg-brand-border/30 px-2 py-0.5 text-xs font-medium secondary
월 과세대상 급여 300만원 기준, 급여에서   ← 동적 요약(§3-7): mt-2 text-sm text-brand-text-secondary
매달 공제되는 4대 보험료는 약 29만원…
```

- 히어로 컨테이너: `mt-1 flex flex-wrap items-baseline gap-2 tabular-nums`. `flex-wrap`으로 고소득
  구간 값(6~7자리)이 375px 카드 안쪽(약 311px)을 넘을 때 "원" 단위가 줄바꿈으로 흐르도록(가로
  오버플로 0). design-system Result 스케일 "모바일 28px 이하 축소 금지" 준수(36px OK).
- **"참고용 예상 보험료" 배지 = 대표값 옆 항상 노출**(기획 §7-2). 상단 amber 배너(§6-1)와 함께
  결과 지점의 근사 고지를 이중 담당. 배지는 경고 색 아닌 중립 회색(`bg-brand-border/40`).
- **chip**: "월 과세급여 {N}만원 · {규모}" — 대표값이 어떤 급여·규모 기준인지 텍스트로 즉시 노출.
- "약"·"원" 단위 텍스트는 히어로보다 작게(Body) Text Secondary — design-system 단위 규칙.

### 3-3. Tier ② — 근로자/사업주/총 3-way 요약 (항상) — [3열 핵심]

Tier ① 아래 구분선(`mt-4 pt-4 border-t border-brand-border/60`) 다음, **이 계산기의 핵심인 3열 비교**를
합계 수준에서 보여준다.

**데스크톱(≥sm): 3셀 가로 병렬 `grid grid-cols-3 gap-3`.** 각 셀 `flex flex-col`:
```
근로자 부담          사업주 부담 †          총 합계
291,310 원           298,811 원            590,121 원
```
- 셀 라벨(상): `text-xs font-medium text-brand-text-secondary`. 값(하): `text-lg font-semibold
  tabular-nums`. **근로자 부담 셀** 값은 히어로와 연결되도록 `text-brand-accent`(에메랄드),
  사업주·총 합계 값은 `text-brand-text`(Accent 아님 — 컬러 강조는 근로자·히어로에 집중해 위계 유지).
- **사업주 부담 셀 라벨 옆 `†` 마커** + 3셀 아래 각주 1줄 `text-xs text-brand-text-secondary`:
  "† 산재보험(전액 사업주 부담·업종별)은 제외된 금액입니다." → **산재 제외 안내를 사업주 값 바로
  옆**에 배치(기획·마스터 지시: 사업주 열 근처).
- 검산: 근로자 291,310 + 사업주 298,811 = 총 590,121(총 합계). 3값이 서로 검산된다.

**모바일(<sm): 3행 라벨-값 스택**(`flex justify-between py-1.5 text-sm`, 다열 grid 미사용):
```
근로자 부담          291,310 원      ← 값 text-brand-accent
사업주 부담 †         298,811 원
총 합계              590,121 원      ← 라벨/값 font-semibold(합계 강조)
† 산재보험(전액 사업주·업종별) 제외
```

### 3-4. Tier ③ — 4보험 항목별 내역 (항상) — 데스크톱 4열 grid / 모바일 블록 스택

이 계산기의 차별점(회사가 각 보험을 얼마 내는지). **`<table>` 미사용, CSS grid + flex로 구성**하고,
데스크톱과 모바일의 배치를 분리해 375px 가로 오버플로를 0으로 만든다(마스터 지시).

#### 3-4-A. 데스크톱(≥sm) — 4열 grid 표

CSS grid로 표를 만든다(HTML `<table>` 아님). 컨테이너 `grid grid-cols-[1.4fr_1fr_1fr_1fr]`, 각 셀은
`py-1.5 text-sm`, 숫자 셀은 우측 정렬 `text-right tabular-nums`.

```
항목              근로자        사업주        합계        ← 헤더행: text-xs font-medium secondary,
                                                            하단 border-b border-brand-border/60
국민연금 [노사절반]  142,500 원   142,500 원    285,000 원
건강보험 [노사절반]  107,850 원   107,850 원    215,700 원
장기요양 [노사절반]   13,961 원    13,961 원     27,922 원
고용보험 [회사 추가]   26,999 원    34,500 원     61,499 원
────────────────────────────────────────────────────────   ← border-t border-brand-border/60
합계               291,310 원   298,811 원    590,121 원   ← font-semibold
```
- **항목 라벨 옆 뱃지(§3-5)**: 국민연금·건강보험·장기요양 = `노사 절반` 뱃지(근로자=사업주 대칭),
  고용보험 = `회사 추가` 뱃지(사업주 > 근로자). 뱃지: `rounded bg-brand-border/30 px-1 py-0.5
  text-[10px] font-medium text-brand-text-secondary`. 뱃지가 폭을 �(좁을 때) 라벨 아래로 줄바꿈 허용.
- 값 우측 정렬 `tabular-nums`로 세로 자릿수 정렬(대칭 3보험은 근로자·사업주 두 열이 같은 값이라
  정렬 시 시각적으로 "쌍둥이"로 보여 "노사 절반"이 직관 전달됨 — 뱃지와 이중 강화).
- **합계 행 = Tier② 3값과 동일**(291,310 / 298,811 / 590,121). Tier②는 요약 헤드라인, Tier③ 합계
  행은 항목 합의 검산 — 연봉·전기요금이 히어로를 breakdown 소계로 재검산한 것과 동형.

#### 3-4-B. 모바일(<sm) — 보험별 블록 스택 (4열 표 → 세로 재배치, 가로 오버플로 0)

375px에서 4열(항목+근로자+사업주+합계, 값 6자리+원)은 가로가 터지므로 **보험별 블록으로 재배치**한다.
각 보험이 소제목 + 3개 라벨-값 단행(`flex justify-between`)을 가진 작은 블록이 된다.

```
┌ 국민연금  [노사 절반] ────────┐   ← 소제목행: text-sm font-medium text-brand-text +
│  근로자           142,500 원   │      우측 뱃지. 블록: rounded-lg border border-brand-border/50
│  사업주           142,500 원   │      px-3 py-2, 블록 간 gap-2
│  합계             285,000 원   │   ← 합계행 font-semibold, 상단 얇은 구분선
└──────────────────────────────┘
┌ 건강보험  [노사 절반] ────────┐
│  근로자 107,850 / 사업주 107,850 / 합계 215,700 (각 라벨-값 단행)
┌ 장기요양보험  [노사 절반] ────┐
│  근로자 13,961 / 사업주 13,961 / 합계 27,922
┌ 고용보험  [회사 추가부담] ────┐
│  근로자            26,999 원   │
│  사업주            34,500 원   │
│  합계              61,499 원   │
└──────────────────────────────┘
```
- 각 블록 내부 3행: `flex items-center justify-between py-1 text-sm`(라벨 좌 secondary / 값 우
  `tabular-nums`). → 라벨-값 단행이라 **가로 스크롤 구조적으로 발생 불가**(사이트 표준 패턴).
- **모바일에서 Tier③ 하단 합계 블록은 생략 가능**(Tier② 3-way 스택이 바로 위에 이미 3 합계를
  노출하므로 중복 회피). 개발 재량으로 Tier② 유지 시 Tier③ 모바일 합계 블록 미표시 권장.
- 반응형 전환: Tailwind `hidden sm:grid`(데스크톱 4열 표) / `grid gap-2 sm:hidden`(모바일 블록
  스택) 두 마크업을 조건부 렌더. 동일 데이터를 두 레이아웃으로 출력(값은 하나의 계산 결과 소스).

### 3-5. "노사 절반 부담" 의미 전달 요소 (마스터 지시 — 대칭값 단조로움 검토)

국민연금·건강보험·장기요양은 근로자=사업주 동일값이라 표가 "같은 숫자 반복"으로 단조롭게 보일 수
있다. 이를 **정보(제도 이해)로 전환**하는 3중 장치:

1. **뱃지(§3-4)**: 대칭 3보험에 `노사 절반` 뱃지, 고용보험에 `회사 추가` 뱃지. 색만이 아닌 **텍스트
   뱃지**라 "왜 같은 값인지"가 즉시 읽힌다(색 단독 의존 금지 규칙 준수).
2. **안내 1줄(Tier③ 표/블록 위, 항상)**: `flex items-start gap-1 text-xs text-brand-text-secondary`,
   좌측 `ⓘ`(`aria-hidden`) — "국민연금·건강보험·장기요양보험은 회사(사업주)가 근로자와 **같은 금액을
   함께 부담**합니다(노사 절반). 고용보험만 회사가 더 부담합니다." → 단조로움이 "제도 설명"으로 전환.
3. **시각 정렬(§3-4-A)**: 데스크톱 표에서 근로자·사업주 두 열이 같은 값으로 세로 정렬되어 "쌍둥이"
   패턴이 눈에 들어옴 — 뱃지·안내문과 삼중으로 "절반씩"을 전달.

- 문안은 기획 확정 사안(§9). "회사가 절반 낸다"는 사실 서술만, 부추김 표현 금지(§6-3).

### 3-6. 상태별 표시 — 국민연금 상한 clamp / 사업장 규모 변경

- **국민연금 상한 clamp(급여 > 659만)**: 결과는 정상 렌더. 국민연금 근로자·사업주 값이 상한
  (313,025원, 앵커 D)에서 고정되고 건강·고용은 계속 증가한다. 별도 경고 UI 불필요이나, **동적
  요약(§3-7)·interpretation(기획 §2 ②·③단)** 에서 "국민연금은 상한이 있어 더 늘지 않는다"가 자연
  노출된다. (선택) Tier③ 국민연금 행 근처에 `text-xs secondary` 인라인 note "국민연금 상한(월
  659만원) 적용" 1줄 허용 — 경고 아님, 중립 회색. 급여 ≤ 659만이면 미표시.
- **사업장 규모 변경**: 규모 select를 바꿔 재계산하면 **사업주 고용보험·사업주 합계·총 합계·Tier③
  고용보험 사업주 셀만 갱신**되고, 근로자 열·국민연금·건강·장기요양은 불변. §2-5 각주가 이를 뒷받침.
  별도 상태 UI 불필요(정상 재계산).
- **경고 색 미사용**: 위 note·상한·큰 값 모두 중립 회색 안내. 상단 amber 배너와 색 중첩 회피, 공포
  조장 금지(전기요금·중도상환 원칙 승계).

### 3-7. Tier ① 요약 문구 = 기획 §2 1문단 동적 배치

기획 §2 1문단(앵커 B: "월 과세대상 급여 300만원…근로자 291,310원…사업주 298,811원…합치면 590,121원")은
정적 텍스트가 아니라 **사용자 입력값(급여·규모) 기반 동적 생성**이어야 한다(기획 §8-8, 기존 라이브
계산기 전부 동일). 이 문서는 **배치 위치만 확정**: Tier ① chip 아래(`mt-2 text-sm text-brand-text-
secondary`). 연봉·전기요금의 `getResultSummary(...)`처럼 컴포넌트 내부 함수로 입력·결과를 받아
1~2문장 조립. **정적 텍스트 금지.**

- 길이: 모바일 2줄 이내 권장. 금액은 "약 N만 원" 반올림으로 간결하게(상세 숫자는 Tier②·③에 있음).
- 근로자/사업주/총을 문장에 반영("급여에서 매달 공제되는 4대 보험료는 약 {근로자}만 원, 회사도 약
  {사업주}만 원을 함께 부담해 합계 약 {총}만 원…"). 규모 변경 시 사업주·총 금액이 문장에도 반영.
- **CTA 가드**: 보험료를 줄이는 법·상품 가입 부추김 표현 금지(§6-3). 사실 서술만. 정식 카피 톤은
  기획 §2 담당(§9).

## 4. 상태별 결과 카드 (입력전 / 정상 / 에러) — 필수

이 계산기는 중도상환의 "면제"처럼 결과 모드 분기가 없다(입력이 무엇이든 결과 구조는 §3 단일 위계).
급여 크기·규모에 따라 값만 갱신된다(§3-6).

### 4-1. 입력 전 (result null)
- 결과 카드 미렌더. 상단 amber 배너 + 입력 폼만.

### 4-2. 정상 상태 (급여 > 0)
- §3-2 히어로(근로자 합계) + §3-3 3-way 요약 + §3-4 4보험 항목별 내역 + §3-5 노사절반 안내 + 각주.
  앵커 A~D 전부 이 상태(기획 §1-4). 급여 > 659만이면 국민연금 clamp note(§3-6) 부가.

### 4-3. 에러 상태
- §2-4대로 필드별 인라인(`FieldError`, `role="alert"`, `border-brand-warning`), 결과 카드 미렌더.
  결과 영역 빨간 배너 미사용.

## 5. 모바일(375px) 반응형 규칙

### 5-1. 무스크롤 목표 — 입력 2개라 폼은 짧으나 결과 3열 재배치로 카드가 길어 스무스 스크롤 채택

iPhone SE급(375×667, 실사용 높이 약 550~600px) 개략 높이 추정.

| 영역 | 예상 높이 |
|---|---|
| 근사 고지 배너(3줄, §6-1) | 약 88~108px |
| 월 과세대상 급여(라벨+`h-12`+보조문구 2줄) | 약 96px |
| 사업장 규모 select(라벨+`h-12`+보조문구 2~3줄) | 약 104px |
| 계산 버튼(`h-12`) + 개인정보 캡션 | 약 84px |
| 카드 상하 padding | 32px |
| **입력 폼까지 소계** | 배너 포함 약 **420~450px** |
| 결과 카드(히어로+배지+chip+요약 2줄+3-way 3행+노사안내+보험별 4블록[각 약 84px]+각주, padding) | 약 620~700px |

→ 입력 폼(약 430px)은 배너 포함해도 거의 한 화면에 들어오지만(입력 2개), **결과 카드는 4보험 블록
스택(4×84px≈336px) + 3-way + 히어로 때문에 길어** 입력+결과가 한 화면에 다 들어오진 않는다. 절충:
1. **상단 배너 광고 미삽입**(모바일·데스크톱). 최상단은 근사 고지 배너 우선(§6-1).
2. **계산 버튼 클릭 시 결과 영역(`resultRef`)으로 스무스 스크롤**(연봉·전기요금·중도상환 구현 그대로).
   결과 카드가 길어 입력 폼 아래 밀리므로 채택.
3. 결과 카드가 다소 길지만, **대표값(근로자 합계)은 스크롤 직후 즉시 above-the-fold**에 오고,
   3-way 요약이 바로 아래 와 핵심(근로자/사업주/총)을 먼저 전달한다. 보험별 상세는 그 아래로 이어짐.
   (기획팀이 모바일 길이를 더 줄이길 원하면 Tier③를 `<details>` 접이식 기본 접힘으로 전환 가능 — §9.)

### 5-2. 결과 카드 가로 오버플로 0 규칙 (필수 — 3열 표가 가장 큰 리스크)

이 계산기에서 가장 신경 쓸 반응형 리스크는 **근로자/사업주/합계 3열 표가 375px에서 가로로 터지는
것**이다. 원천 차단 규칙:
- **다열 `<table>` 미사용.** 데스크톱 4열은 **CSS grid**(`hidden sm:grid`), 모바일은 **보험별 블록의
  라벨-값 단행**(`grid gap-2 sm:hidden`, 내부 `flex justify-between`)으로 분리 렌더(§3-4). 모바일에는
  다열 grid가 존재하지 않으므로 가로 스크롤 구조적으로 발생 불가.
- **Tier② 3-way**: 데스크톱 `grid-cols-3`, 모바일 3행 라벨-값 스택(§3-3). 모바일에서 3열 병렬 아님.
- **히어로 금액**: 최장 현실값(고소득 6~7자리, 예 `660,251원`대)까지 `text-4xl`(36px) + `flex-wrap`
  으로 "원" 단위 줄바꿈 흐름 → 오버플로 0(§3-2).
- **뱃지·chip**: `flex-wrap` 허용이라 좁으면 줄바꿈, 오버플로 0. 라벨+뱃지가 한 줄 넘으면 뱃지가
  라벨 아래로 내려감(§3-4-A).

### 5-3. 부수 규칙
- 모든 입력·select·버튼 `h-12`(48px) 터치타깃(design-system 4장).
- 입력 2개 전체 폭 스택(§2-1) — 데스크톱·모바일 동일 배치라 2열 어긋남 없음.
- **다크모드 없음**: 기존 계산기 전부 `brand-*` 라이트 토큰만. 이 컴포넌트도 `dark:` 미도입(범위 밖).

## 6. YMYL(낮음~중간) 반영 — 근사 배너 · CTA 금지 · 광고

### 6-1. 상단 근사 고지 배너 (amber `role="note"`, 항상 노출) — 카피 1·2안 + 권장

마스터 확정: "YMYL 낮음~중간, 상단 amber `role="note"` 배너, 항상 노출". **연봉·전기요금·중도상환
컴포넌트의 배너 마크업을 그대로 재사용**(`mb-6 flex items-start gap-2 rounded-lg border border-amber-200
bg-amber-50 px-4 py-3`, `role="note"`, 좌측 `ⓘ` `text-brand-warning aria-hidden`, 본문 `text-sm
text-amber-900`). **광고 유닛 아님**("광고" 캡션 금지). 위치: 컴포넌트 최상단, 계산기 카드 위, 계산
여부와 무관하게 항상 노출.

- **카피(기획 §7-3 초안 다듬기, YMYL 낮음~중간 "안내" 톤 — 최종 문안은 기획 확정):**

  > **1안(권장):** "2026년 요율을 기준으로 계산한 근로자·회사(사업주) 부담 **참고용 예상 4대
  > 보험료**입니다. 산재보험(전액 사업주 부담·업종별)은 제외되며, 회사가 신고한 기준소득월액·
  > 보수월액과 사업장 규모에 따라 실제 공제·납부액과 다를 수 있습니다. 요율은 매년 갱신되며,
  > 정확한 금액은 4대 사회보험 정보연계센터에서 확인해 주세요."

  > **2안(간결, 기획 §7-3 초안):** "2026년 요율 기준 근로자·사업주 부담 예상 보험료입니다.
  > 산재보험(전액 사업주·업종별)은 제외되며, 회사 신고 기준·사업장 규모에 따라 실제와 다를 수
  > 있습니다."

- **디자인팀 권장 = 1안.** 연봉·전기요금 배너(2~3줄)와 동일 지면 예산에서 1안이 YMYL 낮음~중간의
  핵심 면책 요소(참고용 + 2026 요율 + 산재 제외 사유 + 신고 기준/규모 차이 + 매년 갱신 + 공식
  출처[4대 사회보험 정보연계센터])를 다중 커버한다. 2안은 요율 갱신·공식 출처가 빠져 다소 약하다.
  최종 문안은 기획팀 확정 사안이라 "카피 제안"으로 명시(§9 — **기획팀에 되돌릴 카피 확정 항목**).
- **YMYL 톤**: 경고가 아니라 안내. "보험료 줄이는 법", "비과세 늘리면 유리" 류 권유·부추김 금지.

### 6-2. 결과 지점 근사 고지 (다중 고지 정합)
상단 배너(§6-1) + "참고용 예상 보험료" 배지(§3-2) + 산재 제외 각주(§3-3·§3-4 하단) + interpretation
③④⑤단(기획 §2, 산재 제외로 회사 총부담은 더 큼·개정 가능성·신고기준/규모/업종 차이) + FAQ 3·4(기획
§3)로 근사 한계가 다중 고지된다. 결과 카드 내 별도 인라인 노트는 **국민연금 상한 note(§3-6, 선택)** 만
두고 중복 회피. "참고용 예상 보험료" 배지·산재 각주 = 결과 지점 핵심 고지.

### 6-3. CTA 디자인 금지 원칙 (YMYL, 상업적 부추김 금지)
- 계산기 카드 안에는 **"계산하기" 버튼 1개만** 존재. "보험료 절감 상담", "노무·세무 컨설팅 신청",
  "4대 보험 대행 서비스", "내 보험료 낮추는 법" 류 2차 버튼/링크/배너를 결과 카드나 주변에 **절대**
  추가하지 않는다.
- 결과·요약(§3-7)·노사절반 안내(§3-5)에도 보험료 회피·상품 가입을 부추기는 표현 금지. 사실 서술만
  (예: "국민연금은 상한이 있어 더 늘지 않습니다"는 사실 고지, "비과세를 늘려 보험료를 아끼세요"는 금지).
- 부담 주체·상위 항목을 **빨강/경고 색으로 칠하지 않는다**(§3-1) — 공포 조장 = 상업적 부추김 변형.
- 공식 기관(4대 사회보험 정보연계센터·각 공단) 참조가 필요하면(주로 interpretation 본문, 범위 밖)
  **일반 본문 링크 스타일**(밑줄+Primary 인라인)만, 박스형 CTA 버튼 금지(버튼은 계산 버튼 전용 예약).
- "관련 계산기" 카드(공통 템플릿)는 내부 링크로만(기획 §5: 연봉실수령+퇴직금+만나이, override 권장 —
  데이터 소관). 외부 보험·노무·대행 서비스 연결 카드 금지.

### 6-4. 광고 배치 (design-system 5장 + 이 페이지 규칙)
- 상단 배너 광고: 넣지 않음(§5-1, 근사 고지 배너가 그 자리).
- 인터랙티브 클러스터(입력 ↔ 버튼 ↔ 결과 카드) 내부/사이에 광고 미삽입(오인 클릭·흐름 방해).
- 본문 중간/하단 광고: 공통 템플릿 처리(범위 밖). design-system 5장 준수(위아래 margin 24px+, "광고"
  Caption, 결과 카드의 에메랄드 틴트/좌측 강조선·amber 배너와 유사하지 않은 **중립 배경[White/Gray 50
  + 얇은 회색 테두리]**). 보험·노무·재테크 광고 소재가 "보험료 아끼기" 류 문구를 포함하더라도 사이트

## 10. 대안으로 검토했으나 채택하지 않은 안

- **모바일에서도 4열 grid 표 유지(축약)**: 라벨 짧게·"원" 생략으로 375px에 욱여넣는 안. 값이 6자리
  (313,025)까지 가면 3개 값 열이 겹치거나 가로 스크롤 발생 위험 → **모바일은 보험별 블록 스택으로
  재배치**(§3-4-B, 마스터 지시 "세로 재배치, 가로 스크롤 0").
- **4보험 항목별 내역 접이식(`<details>` 기본 접힘)**: 모바일 높이는 줄지만 "회사가 각 보험 얼마
  내는지"라는 **이 계산기의 핵심 가치를 숨김** → 항상 노출(§3-0 결정 3). 높이는 스무스 스크롤로 완화.
- **대칭 3보험(국민연금·건강·장기요양)의 사업주 열 생략하고 "근로자와 동일" 텍스트만**: 표가 간결
  해지나 "회사도 얼마 낸다"를 숫자로 못 보여줘 3열 구조의 취지가 약화됨 → **사업주 열에 실제 금액을
  그대로 노출**하고, 단조로움은 "노사 절반" 뱃지·안내로 전환(§3-5).
- **산재보험을 "업종 평균 추정액"으로 사업주 열에 포함**: 회사 총부담을 더 현실적으로 보이게 하나,
  업종별 편차가 커 검증 불가·오정보 리스크(기획 §1-2·§8-3 보수적 원칙) → **제외 + 각주/배너 고지**
  (§3-3·§3-4·§6-1). "회사 총부담은 이 값 + 업종별 산재"임을 명시.
- **인터랙션 실시간(급여 타이핑 시 즉시 재계산)**: 입력 2개라 가볍지만 amber 배너 YMYL family가
  전부 버튼식이라 일관성 저해 + 급여(민감정보) 부분값 깜빡임 → 버튼식(§0-2, 마스터 확정).

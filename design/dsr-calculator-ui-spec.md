## 0. 이 페이지의 성격 (설계 전제)

1. **YMYL 높음 — 규제선 오인이 최대 리스크.** DSR은 "총부채 원리금 ÷ 연소득"이라는 **비율(%)이고,
   은행권 규제선 40%라는 실재하는 기준선**이 있다. 사용자가 결과 화면을 "대출 승인/거절 판정서"로
   착각하기 가장 쉬운 계산기다. → (1) 상단 amber `role="note"` 배너 **항상 노출**, (2) 규제선 구간
   배지 **바로 옆에 "승인 여부 판단 아님" 캡션 필수**, (3) 히어로 DSR% 숫자 자체는 **상태색을 입히지
   않음**(안전=초록/초과=빨강으로 물들여 "합격/불합격"처럼 보이는 것을 방지, §3-2). 상태는 **별도 배지
   + 위치 게이지**가 색+아이콘+텍스트 3중으로 전달한다(색 단독 의존 금지).
2. **인터랙션 = 버튼식(실시간 아님).** 대출 category YMYL family(대출이자·중도상환)와 동일하게
   `form onSubmit` + **"계산하기"** 버튼 1개. 실시간 계산은 사이트 원칙상 단위변환 전용이다. 소득·
   부채(민감정보) 타이핑 중 부분값 깜빡임을 막고 "확인 후 결과"의 YMYL 리듬을 유지한다.
3. **입력 = 소득 1 + 신규 대출 그룹(금액·이자율·기간·상환방식) + 기존 대출 원리금(선택).** 신규 대출
   4필드는 **대출이자 계산기의 입력 마크업을 거의 그대로 재사용**한다(§2·§6). 결과의 핵심은 **DSR(%)
   대표값 + 40% 규제선 대비 위치**다(§3).
4. **YMYL CTA 절대 금지.** "지금 대출받기", "한도 조회", "대출 상담 신청", "내 DSR 낮추는 법" 류
   상업적/부추김 2차 버튼·문구·색을 계산기 카드 안팎에 절대 넣지 않는다(§6-3). 계산 버튼 1개만 존재.
   DSR은 대출 알선 오인 리스크가 특히 커 애드센스 정책상 가장 엄격히 지킨다.
5. **결과 숫자에 공포 조장 금지.** "초과" 구간이어도 히어로 숫자는 중립색, 배지만 규제선 색을 쓰고,
   문구는 사실 서술("은행권 기준선 40%를 넘습니다")만. "대출 불가", "거절 위험" 류 단정·위협 표현
   금지(§3-3·§6-3).

## 2. 입력 설계 (연소득 1 + 신규 대출 4 + 기존 대출 1 + 계산 버튼)

### 2-0. 그룹핑 원칙 — 3섹션 세로 스택 + 소제목

입력 6개를 **의미 단위 3섹션**으로 묶고 각 섹션에 소제목(`text-sm font-semibold text-brand-text`)을
둔다. 대출이자 계산기(4필드)보다 필드가 많아 그룹 없이 나열하면 사용자가 "무엇을 신규/기존으로
넣는지" 혼동하기 쉽다(특히 기존 대출 원리금은 개념이 낯설다).

| 섹션 | 소제목 | 포함 필드 |
|---|---|---|
| A | (소제목 없음, 최상단) | ① 연소득 |
| B | **신규(대상) 대출** | ② 대출 금액 ③ 연 이자율 ④ 대출 기간(+년/개월) ⑤ 상환 방식 |
| C | **기존 대출 (선택)** | ⑥ 기존 대출 연간 원리금상환액 |

- 섹션 소제목은 `mt-2 border-t border-brand-border/50 pt-4 text-sm font-semibold text-brand-text`로
  얇은 구분선과 함께 둔다(입력 섹션 경계를 시각적으로 분리). A섹션(연소득)은 최상단이라 구분선 없음.
- **2열은 신규 대출의 이자율/기간 한 쌍에만** 적용(대출이자 계산기와 동일, §2-2 ③④). 나머지는 전체 폭.

### 2-1. 공통 스타일 상수 (기존 계산기와 동일 문자열)

- `inputBase` = `"h-12 rounded-lg border px-4 text-base text-brand-text outline-none transition-colors
  focus:border-brand-primary focus:ring-4 focus:ring-brand-primary/15"` (오류 시 `border-brand-warning`,
  평시 `border-brand-border` 토글). 대출이자·중도상환·4대보험 동일 상수.
- 각 필드 컨테이너 `flex flex-col gap-1.5`, 라벨 `text-sm font-medium text-brand-text-secondary`,
  보조문구(Caption) `text-xs text-brand-text-secondary`.
- 우측 단위 접미(`원`/`%`)는 중도상환의 어드먼트 패턴 재사용: `relative` 컨테이너 + input `w-full pr-9`
  + `<span aria-hidden className="pointer-events-none absolute right-4 top-1/2 -translate-y-1/2
  select-none text-sm text-brand-text-secondary">원</span>`.

### 2-2. 필드별 스펙 (순서 고정 ①~⑥ · 기획 확정 시 조정)

| 순서 | 필드 | id | 라벨(표시) | 컨트롤 | 필수 | placeholder / 기본값 | 우측 단위 | 보조문구(Caption) |
|---|---|---|---|---|---|---|---|---|
| ① | 연소득 | `annualIncome` | **"연소득 (원)"** | `type="text"` + `inputMode="numeric"`, 콤마(`formatAmountInput`) | **필수** | `50,000,000` | `원` | "세전 연소득입니다. 상여·수당 포함, 회사가 신고한 소득 기준으로 넣어 주세요." |
| ② | 대출 금액 | `loanAmount` | **"대출 금액 (원)"** | `type="text"` + `inputMode="numeric"`, 콤마 | **필수** | `300,000,000` | `원` | "이번에 새로 받을(또는 대상) 대출의 원금" |
| ③ | 연 이자율 | `interestRate` | **"연 이자율 (%)"** | `type="number"` `step="0.01"` `min="0"` | **필수** | `4.5` | `%` | — (2열 좌) |
| ④ | 대출 기간 | `loanPeriod` | **"대출 기간"** | `type="number"` `step="1"` + **년/개월 토글**(대출이자 패턴) | **필수** | `30` | (토글) | — (2열 우, 하단 년/개월 토글) |
| ⑤ | 상환 방식 | `repaymentType` | **"상환 방식"** | **세그먼트 토글**(대출이자 패턴) [원리금균등 / 원금균등] | 선택 | 기본 **원리금균등** | — | (선택) "원금균등은 초기 상환액이 커 DSR이 더 높게 잡힐 수 있습니다." |
| ⑥ | 기존 대출 연 원리금 | `existingAnnualРmt` | **"기존 대출 연간 원리금상환액 (원)"** | `type="text"` + `inputMode="numeric"`, 콤마 | 선택 | `0` / 빈값=0 | `원` | "이미 갚고 있는 다른 대출의 1년치 원리금 합계입니다. 없으면 비워 두세요." |

- **①②⑥ 금액 = 천단위 콤마 재사용**: 대출이자/중도상환의 `formatAmountInput(raw)`(`[^\d]` 제거 →
  `toLocaleString`)을 그대로 재사용. 음수·문자·소수점이 입력 단계에서 원천 차단(§2-4).
- **③④⑤ = 대출이자 계산기 마크업 그대로**: 연 이자율(`type="number"`), 대출 기간 + **년/개월 컴팩트
  토글**(`role="radiogroup"`, `h-9 flex-1` 버튼), 상환 방식 **세그먼트 토글**(`fieldset` + `sr-only`
  radio, 선택 시 `bg-brand-primary text-white`). `REPAYMENT_TYPE_LABEL`·`RepaymentType` 재사용.

  여러 건 가질 수 있어 계산기 안에서 재계산하면 입력이 폭증하므로, **1년치 합계 금액만 직접 받는다**
  (마스터 요구사항). 비우면 0으로 처리(선택). 기획팀이 "기존 대출도 금액/이자율/기간으로 받아 내부
  계산" 방식을 원하면 회신 시 조정(§9).

### 2-3. 계산 버튼 + 개인정보 캡션

- 버튼: 대출이자·중도상환·4대보험과 동일 `h-12 w-full rounded-lg bg-brand-primary px-5 text-base
  font-semibold text-white transition-colors hover:bg-brand-primary-hover`. 라벨 **"계산하기"**(중립
  문구 — "내 한도 확인", "대출 가능 여부 보기" 등 심사 오인·부추김 문구 절대 금지, §0-4·§6-3).
- 버튼 아래 개인정보 캡션(`mt-2 text-xs text-brand-text-secondary`): "입력하신 소득·대출 정보는
  브라우저 안에서만 계산되며 서버에 저장·전송되지 않습니다." (소득·부채 = 민감정보라 4대보험보다도
  신뢰 고지가 중요).

### 2-4. 입력 검증 (필드별 인라인, `role="alert"`, 기존 계산기와 일관)

대출이자·중도상환의 `FieldError`(필드 하단 인라인) 패턴. **결과 영역 빨간 배너 방식 미사용**, 오류
원인 필드 하단에 `text-xs text-brand-warning` `role="alert"` + 해당 input 테두리 `border-brand-warning`
토글(`aria-invalid`). **경보색은 `brand-warning`(amber)이며 빨강 아님**.

| 검증 | 트리거 | 표기 위치 | 문구 예 |
|---|---|---|---|
| 연소득 미입력/0 | 빈값 또는 숫자화 후 `≤ 0` | `annualIncome` 하단 인라인 + 테두리 | "연소득을 입력해주세요." |
| 대출 금액 미입력/0 | 빈값 또는 `≤ 0` | `loanAmount` 하단 인라인 + 테두리 | "대출 금액을 입력해주세요." |
| 연 이자율 미입력/음수 | 빈값 또는 `< 0` | `interestRate` 하단 | "연 이자율은 0 이상으로 입력해주세요." |
| 대출 기간 미입력/0/비정수 | 빈값 또는 `≤ 0` / 정수 아님 | `loanPeriod` 하단 | "대출 기간을 정수로 입력해주세요." |
| 기존 대출 원리금(선택) | 빈값=정상(0) / 음수·문자 | (콤마 서식이 원천 차단) 발생 안 함 | — |
| 상환 방식 | 항상 기본값 존재 | 발생 안 함 | — |
| 소득 대비 대출 극단값(예 연소득 0 근처인데 대출 큼) | DSR이 매우 큰 값 → **에러 아님**, 정상 렌더 + "초과" 배지 | — | — |

- 에러가 하나라도 있으면 결과 카드는 렌더하지 않는다(대출이자·중도상환과 동일).
- 시각 스타일·배치만 규정하며 정확한 트리거·문구는 개발 재량(기존 계산기 원칙 계승).

## 3. 결과 카드 설계 — DSR% 히어로 + 규제선 배지·위치 게이지 + 내역

### 3-0. 정보구조 (대표값 히어로 + 규제선 상태 + 내역, 전부 항상 노출)

| Tier | 항목 | 노출 | 근거 |
|---|---|---|---|
| **① 대표값** | 예상 DSR `NN.N %` + "참고용" 배지 | 항상 | 사용자 핵심 질문 "내 DSR 몇 %?" |
| ① 규제선 배지 | 안전 / 주의 / 기준선 초과 (색+아이콘+텍스트) + **"승인 여부 판단 아님" 캡션** | 항상 | §3-3 (YMYL 필수) |
| ① chip | 연소득 · 신규대출 · 상환방식·기간 (예 "연소득 5,000만원 · 신규대출 3억 · 원리금균등 30년") | 항상 | 대표값의 맥락 |
| ① 위치 게이지 | 0 → 40%(은행권 기준선) 대비 현재 DSR 위치 마커 | 항상 | §3-4, 규제선 대비 시각 위치 |
| ① 요약 | 동적 1~2줄(연간 총 원리금·연소득·DSR 반영) | 항상 | §3-6 |
| **② 내역** | 신규 대출 연 원리금 / 기존 대출 연 원리금 / 연간 총 원리금(합계) / 연소득 / DSR 계산식 | 항상 | §3-5, 계산 근거·검산 |
| ② 각주 | "40%·50%는 참고 수치, 실제 심사는 스트레스 금리·기관 정책에 따라 다름" | 항상 | §6-2 면책 |

- 결과 카드 공통 골격(기존 계산기와 동일): `mt-6 rounded-xl border-l-4 border-brand-accent
  bg-gradient-to-br from-emerald-50 to-white p-4 transition-colors duration-200 sm:p-6`. **좌측 강조선·
  틴트는 결과 "프레임"일 뿐 상태 판정색이 아니다** — 카드 배경은 DSR 구간과 무관하게 항상 동일(공포
  조장 방지). 계산 후에만 조건부 렌더(`result && (...)`), `resultRef` 스무스 스크롤.

### 3-1. Tier ① — 대표값(예상 DSR %) 렌더 (항상)

```
예상 DSR  [참고용]                       ← 라벨: text-sm font-medium text-brand-text-secondary
                                          배지: ml-2 rounded bg-brand-border/40 px-1.5 py-0.5
                                          text-xs font-medium text-brand-text-secondary
38.4 %                                   ← 히어로: text-4xl sm:text-[2.5rem] font-bold tabular-nums
                                          text-brand-text(중립!), "%"는 text-base font-medium secondary
```
- 히어로 컨테이너: `mt-1 flex flex-wrap items-baseline gap-2 tabular-nums`. **숫자 색 = `text-brand-text`
  (중립 다크). `text-brand-accent`(에메랄드) 미사용.** 이유: DSR%는 비율값이라 에메랄드로 물들이면
  "안전/합격"으로, 빨강이면 "불합격"으로 오독된다. 대표값은 중립색으로 두고 **상태는 배지·게이지가
  전담**한다(§0-1·§0-5). (이는 4대보험·대출이자가 히어로에 에메랄드를 쓴 것과 다른, DSR 고유의
  의도적 결정 — §9에서 마스터 확인 요청.)
- "%"·단위 텍스트는 히어로보다 작게(Body) Text Secondary.
- **"참고용" 배지 항상 노출**(중도상환·4대보험과 동일, 중립 회색 `bg-brand-border/40`).
- DSR% 표기: 소수 1자리(예 `38.4`), 정수면 소수점 생략(`40`) — `formatRatio` 스타일 재사용.

### 3-2. Tier ① chip

`mt-1` 아래 pill: `inline-block rounded-full bg-brand-border/30 px-2 py-0.5 text-xs font-medium
text-brand-text-secondary`. 내용: "연소득 {N}만원 · 신규대출 {억/만} · {상환방식} {기간}" (예:
"연소득 5,000만원 · 신규대출 3억 · 원리금균등 30년"). 대표값이 어떤 입력 기준인지 즉시 노출.

### 3-3. 규제선 구간 배지 + 필수 면책 캡션 (YMYL 높음 핵심)

DSR을 **은행권 기준선(기본 40%)** 대비 3구간으로 분류해 **색 + 아이콘 + 텍스트 3중**으로 표시한다
(색 단독 의존 금지). 배지는 히어로 바로 아래(chip 위 또는 아래, §3-1 순서), **배지 우측/하단에
"승인 여부 판단 아님" 캡션을 반드시 붙인다**(마스터 필수 요구).

**구간 정의(디자인 제안 — 정확한 컷오프는 기획 확정, §9):**

| 구간 | 조건(제안) | 배지 텍스트 | 아이콘 | 색(규제배지 팔레트) |
|---|---|---|---|---|
| 안전 | DSR ≤ 30% | **"안전 구간"** | ✓ | `border-emerald-200 bg-emerald-50 text-emerald-700` |
| 주의 | 30% < DSR ≤ 40% | **"주의 구간"** | ⚠(아이콘 대신 ⓘ 가능) | `border-amber-200 bg-amber-50 text-amber-800` |
| 기준선 초과 | DSR > 40% | **"기준선 초과"** | ! | `border-red-200 bg-red-50 text-red-700` |

- 배지 클래스: `inline-flex items-center gap-1 rounded-full border px-2.5 py-1 text-xs font-semibold`
  + 위 구간별 색. 아이콘은 `aria-hidden`, 텍스트가 의미를 전달(색맹 접근성).
- **필수 면책 캡션(배지 바로 옆·데스크톱 / 배지 아래·모바일)**: `text-xs text-brand-text-secondary`
  — **"이 구간 표시는 참고용이며, 실제 대출 승인·한도 심사 결과를 보장하지 않습니다."** (마스터 필수).
  배지와 항상 한 세트로 렌더(배지만 단독 노출 금지).
- **"기준선 초과" 빨강 사용 = 디자인팀 제안, 마스터 확인 항목(§9).** 사이트 관행상 결과에 빨강
  경보색을 피해 왔으나, DSR 40%는 **실재하는 규제 기준선**이고 "넘었다/안 넘었다"는 사용자가 반드시
  알아야 할 사실 정보다. 따라서 (1) **히어로 숫자는 중립**으로 두어 공포를 최소화하고, (2) **빨강은
  작은 배지에만**, (3) 텍스트는 "기준선 초과"라는 **사실 서술**(≠ "대출 불가"·"거절")로 제한한다.
  마스터가 빨강을 불허하면 **대안**: "초과" 구간도 amber(`border-amber-300 bg-amber-100 text-amber-900`)
  로 하고 아이콘·텍스트("기준선 초과")로 주의와 차등화(§10 대안).

### 3-4. Tier ① 위치 게이지 (규제선 대비 시각 위치) — 중도상환 게이지 패턴 응용

중도상환 `SlidingGaugeView`의 **`role="img"` 텍스트 대체 + 단일 막대 + 마커 + 색 단독 금지** 원칙을
그대로 응용한 **가로 위치 바**. 0%에서 시작해 **40% 기준선 tick**과 **현재 DSR 마커**를 함께 표시한다.

```
        경과 마커(현재 DSR 38.4%)
              ▼
├──────────────────●──────┤        ← 트랙: h-2~h-3 rounded, bg-brand-border/40
0%                40%       (max)   ← 40% 지점에 세로 tick(bg-brand-text) + "40% 은행권 기준선" 라벨
```
- **트랙 스케일**: 0 ~ `max`. `max = 60%`(또는 DSR>60이면 DSR값까지 동적 확장) 제안 — 40% 기준선이
  트랙의 약 2/3 지점에 오도록. 정확한 스케일은 개발 재량(마커·tick가 잘리지 않게 §3-4 정렬 보정
  중도상환 패턴 재사용).
- **채움 세그먼트**: 0→현재 DSR 구간을 구간색(§3-3)의 옅은 톤으로 채움(`bg-emerald-200/50` /
  `bg-amber-200/50` / `bg-red-200/50`). 색 단독 금지라 마커 라벨 텍스트("38.4%")를 함께 표기.
- **40% 기준선 tick**: 트랙 위 40% 위치에 세로선(`w-0.5 bg-brand-text`) + 하단 라벨 "40% 은행권 기준선"
  (`text-[10px] text-brand-text-secondary`). DSR이 40%를 넘으면 마커가 tick 오른쪽에 위치해 "선을
  넘은" 것이 한눈에 보인다.
- **접근성**: 게이지 래퍼 `role="img"` + `aria-label`에 문장 대체("예상 DSR 38.4%로 은행권 기준선
  40% 이내입니다. 이 표시는 승인 여부를 보장하지 않습니다.") — 시각 요소를 스크린리더가 문장으로 낭독.
  막대·마커는 `aria-hidden`. **빨강 단독 의존 금지**(넘침을 마커 위치+텍스트로도 전달).
- 게이지는 **신규 시각 요소**(중도상환 게이지에 이은 두 번째)라 §9에서 design-system 편입 여부 확인.

### 3-5. Tier ② — 내역 (항상 노출, 라벨-값 단행)

Tier ① 아래 구분선(`mt-4 border-t border-brand-border/60 pt-4`) 다음. 전부 `flex items-center
justify-between py-1.5 text-sm`(라벨 좌 secondary / 값 우 `font-semibold tabular-nums text-brand-text`).
`<table>` 미사용.

| 행 | 라벨 | 값 | 강조 |
|---|---|---|---|
| 1 | 신규 대출 연간 원리금상환액 | `18,240,000 원` | 일반 |
| 2 | 기존 대출 연간 원리금상환액 | `960,000 원` (미입력 시 `0 원`) | 일반 |
| 3 | 연간 총 원리금상환액 (합계) | `19,200,000 원` | **bold**, 상단 얇은 구분선 |
| 4 | 연소득 | `50,000,000 원` | 일반 |
| 5 | DSR 계산식 | `19,200,000 ÷ 50,000,000 = 38.4 %` | **bold** (중도상환 "계산식" 행 패턴) |

- 금액은 `formatWon` + " 원". 5행 계산식은 중도상환 `ClauseRow`(값이 문자열인 범용 라벨-값 행) 재사용.
- **각주(면책, `mt-2 text-xs text-brand-text-secondary`)**: "은행권 40%·제2금융권 50%는 참고 수치이며,
  실제 심사는 스트레스(가산) 금리·대출 종류·기관 정책에 따라 달라질 수 있습니다." (§6-2 다중 고지의 일부).
- (선택) 신규 대출 **월 상환액**을 참고로 1행 추가 가능(대출이자 계산 결과 재사용). 단 DSR 핵심
  4행+계산식을 흐리지 않게 개발 재량. 기본은 위 5행만.

### 3-6. Tier ① 동적 요약 (정적 텍스트 금지)

중도상환·4대보험 `getResultSummary(...)`처럼 컴포넌트 내부 함수로 입력·결과를 받아 1~2문장 조립.
배치: chip/게이지 아래(`mt-2 text-sm text-brand-text-secondary`). 예: "연간 총 원리금상환액 약
1,920만원이 연소득 5,000만원의 약 38%로, 은행권 기준선 40% 이내입니다. 실제 대출 심사 결과는 다를 수
있습니다." — **구간별로 문장이 바뀌되(안전/주의/초과), 단정·위협 표현 금지**(§6-3). "초과" 구간 예:
"…약 45%로 은행권 기준선 40%를 넘습니다. 다만 이는 참고용이며 실제 승인 여부를 뜻하지 않습니다."
`formatApprox`(약 N만원) 재사용으로 간결하게. 정확한 카피·톤은 기획 확정(§9).

## 4. 상태별 결과 카드 (입력전 / 정상[안전·주의·초과] / 에러)

### 4-1. 입력 전 (result null)
- 결과 카드 미렌더. 상단 amber 배너 + 입력 폼만.

### 4-2. 정상 상태 (필수 입력 유효) — 3구간 공통 위계
- §3 단일 정보구조(히어로 DSR% + 배지 + 게이지 + chip + 요약 + 내역 5행 + 각주). **구간(안전/주의/
  초과)에 따라 바뀌는 것은 (a) 배지 색·아이콘·텍스트(§3-3), (b) 게이지 채움색·마커 위치(§3-4),
  (c) 동적 요약·게이지 aria 문장(§3-6)뿐.** 히어로 숫자색·카드 프레임·내역 레이아웃은 불변.
- DSR이 매우 큰 값(예 100% 초과)도 정상 렌더(에러 아님). 게이지 트랙 max를 DSR값까지 동적 확장하거나
  마커를 오른쪽 끝(clamp)에 고정 + 텍스트로 실제 값 노출(가로 오버플로 0 유지).

### 4-3. 에러 상태
- §2-4대로 필드별 인라인(`FieldError`, `role="alert"`, `border-brand-warning`), 결과 카드 미렌더.
  결과 영역 빨간 배너 미사용.

## 5. 모바일(375px) 반응형 규칙

### 5-1. 스무스 스크롤 채택 (입력 6개 + 결과로 폼이 길다)
- 입력 6필드(연소득+신규4+기존1)+3섹션 소제목이라 폼이 대출이자보다 길다. **상단 배너 광고 미삽입**
  (최상단은 근사 고지 배너 우선). **계산 버튼 클릭 시 `resultRef`로 스무스 스크롤**(대출이자·중도상환
  구현 그대로). 스크롤 직후 DSR% 히어로 + 배지 + 게이지가 즉시 above-the-fold에 온다.

### 5-2. 가로 오버플로 0 규칙 (필수)
- **다열 `<table>` 미사용.** 결과 내역 5행은 전부 `flex justify-between` 라벨-값 단행 → 다열 grid가
  없어 가로 스크롤 구조적으로 발생 불가.
- **위치 게이지**: 트랙·채움·마커 모두 `%` 폭 기반이라 컨테이너(375px 카드 내부 ≈ 311px)에 맞춰 축소.
  마커/tick 라벨이 양 끝에서 잘리지 않게 중도상환 `markerAlign` 정렬 보정(≤8%→left, ≥92%→right) 재사용.
- **히어로 DSR%**: `38.4 %` 최대 4~5자라 오버플로 없음. `flex flex-wrap`로 안전.
- **규제선 배지 + 면책 캡션**: 모바일은 배지 아래 캡션 줄바꿈(§1-2). 배지 자체 `flex-wrap` 허용.
- **chip**: `flex-wrap`, 길면 줄바꿈.
- **데스크톱 2열(이자율/기간)은 모바일 1열**(전체 폭 스택)로 — 대출이자와 동일하게 `grid grid-cols-2`
  를 쓰되 모바일에서도 2열 유지가 가능하나(대출이자는 유지), DSR은 필드가 많아 **모바일 1열 스택 권장**
  (좁은 화면 가독성). 개발 재량이나 375px에서 이자율/기간 2열이 좁으면 `grid-cols-1 sm:grid-cols-2`.

### 5-3. 부수 규칙
- 모든 입력·select·버튼 `h-12`(48px), 토글 내부 `min-h-11`/`h-9` 터치타깃.
- **다크모드 없음**: 기존 계산기 전부 `brand-*` 라이트 토큰만. `dark:` 미도입.

## 6. YMYL(높음) 반영 — 근사/심사아님 배너 · 다중 면책 · CTA 금지 · 광고

### 6-1. 상단 근사/심사아님 고지 배너 (amber `role="note"`, 항상 노출)

**대출이자·중도상환·4대보험 배너 마크업 그대로 재사용**: `mb-6 flex items-start gap-2 rounded-lg
border border-amber-200 bg-amber-50 px-4 py-3`, `role="note"`, 좌측 `ⓘ` `text-brand-warning
aria-hidden`, 본문 `text-sm text-amber-900`. **광고 유닛 아님**("광고" 캡션 금지). 위치: 컴포넌트
최상단, 계산기 카드 위, 계산 여부와 무관하게 항상 노출.

- **카피(디자인 제안 — 최종 문안 기획 확정, §9):**

  > **1안(권장):** "이 계산기는 **대출 심사·승인 결과가 아닌 참고용 도구**입니다. 실제 DSR은
  > 금융회사가 적용하는 **스트레스(가산) 금리**, 대출 종류(주담대·신용·전세 등), 마이너스통장·

  > 해당 금융기관에 문의해 주세요."

  > **2안(간결):** "대출 심사 결과가 아닌 참고용 예상 DSR입니다. 실제 DSR은 스트레스 금리·대출

### 6-2. 결과 지점 다중 면책 (YMYL 높음 = 고지 밀도 최상)
상단 배너(§6-1) + "참고용" 배지(§3-1) + **규제선 배지 옆 "승인 여부 판단 아님" 캡션(§3-3, 필수)** +
위치 게이지 aria 문장의 면책 문구(§3-4) + 동적 요약 면책 절(§3-6) + 내역 각주(§3-5) + interpretation·
FAQ(기획, 범위 밖)로 **6중 이상 고지**. 이 계산기는 다른 어떤 계산기보다 면책을 촘촘히 배치한다

### 6-3. CTA 디자인 금지 원칙 (YMYL 높음, 상업적 부추김·심사 오인 금지)
- 계산기 카드 안에는 **"계산하기" 버튼 1개만**. "한도 조회", "대출 신청", "대출 상담", "지금 대출받기",
  "내 DSR 낮추는 법", "대환 알아보기" 류 2차 버튼/링크/배너를 결과 카드나 주변에 **절대** 추가 금지.

  위협 표현 금지. **사실 서술만**("기준선 40%를 넘습니다"는 사실, "대출이 거절됩니다"는 금지).
- **히어로 숫자에 빨강·경고색 미부여**(§3-1) — 공포 조장 = 상업적 부추김의 변형. 빨강은 "초과" **작은
  배지에만** 한정, 그마저 마스터 확인 대상(§3-3·§9).
- "관련 계산기" 카드(공통 템플릿)는 내부 링크로만(대출이자·중도상환 권장). 외부 대출·대환·중개 서비스

  Caption, 결과 카드 에메랄드 틴트·amber 배너·규제 배지와 유사하지 않은 **중립 배경**). **대출·대환·

## 8. 개발팀 전달 체크리스트

- 금액 3필드(연소득·대출금액·기존원리금) `formatAmountInput` 재사용(콤마·음수·문자·소수점 차단),
  결과 금액 `formatWon`, DSR% `formatRatio`(소수 1자리).
- 이자율/기간/상환방식 = 대출이자 계산기 마크업·유틸 재사용(신규 정의 최소화).
- **히어로 DSR% 숫자색 = `text-brand-text`(중립), `text-brand-accent` 미사용**(§3-1) — 상태는 배지·
  게이지가 전담.
- **규제선 배지는 반드시 "승인 여부 판단 아님" 면책 캡션과 한 세트로 렌더**(§3-3, 배지 단독 노출 금지).
- 위치 게이지 `role="img"` + `aria-label` 문장(면책 포함), 막대·마커 `aria-hidden`, 색 단독 금지,
  마커/tick 라벨 잘림 방지 정렬 보정(§3-4). DSR>max 시 clamp + 텍스트로 실제 값 노출.
- 필드별 인라인 검증(`FieldError`, `role="alert"`, `border-brand-warning`), 결과 카드 빨간 배너 미사용.
- 동적 요약 `getResultSummary`(§3-6, 구간 분기, 정적 금지, 단정·위협 표현 금지).
- `resultRef` 계산 후 스무스 스크롤(§5-1).
- 결과 내역은 전부 라벨-값 단행(`flex justify-between`), `<table>` 미사용(§5-2).
- 구현 후 375×667 / 390×844 / 1280에서 §5-2 가로 오버플로 0(특히 위치 게이지·배지+캡션 줄바꿈),
  §5-1 스크롤, 3구간(안전/주의/초과) 각각의 배지·게이지·요약 렌더를 디자인팀이 재검수.

## 9. 마스터 확인 필요 / 타팀 이관 (핑퐁)

**기획팀에 되돌릴/확인 요청 (콘텐츠·계산식 소관 — 기획안 미도착 상태라 특히 중요):**
1. **계산식 확정**: DSR = (신규 연 원리금 + 기존 연 원리금) ÷ 연소득 가정. **원금균등의 DSR 연
   원리금 산정 방식**, **스트레스(가산) 금리 반영 여부**, 규제선 컷오프(은행권 40%·제2금융 50%)·구간
   경계(안전 ≤30%/주의 30~40%/초과 >40% = 디자인 제안값)를 기획/개발이 확정 → 확정 시 §2·§3 조정.
2. **입력 필드 최종 목록**: 본 스펙은 마스터 요구(연소득·신규대출 4필드·기존 원리금 선택 직접입력)로
   설계. 기획이 "기존 대출도 금액/이자율/기간으로 받아 내부 계산" 또는 "금융권(1/2금융) 선택 토글
   추가"를 원하면 회신 시 조정(§2-2 ⑥·§3-3).
3. **배너 카피(§6-1) 최종 문안**(1안 권장·2안 간결), 동적 요약(§3-6)·규제선 면책 캡션(§3-3)·내역
   각주(§3-5) 문안 톤 확정(단정·부추김·위협 금지).

**마스터 팀장 확인 필요:**
1. **[핵심] "기준선 초과" 구간 빨강 배지 허용 여부(§3-3).** 사이트 관행은 결과 빨강 경보색 회피였으나,

## 10. 대안으로 검토했으나 채택하지 않은 안

- **히어로 DSR% 숫자를 구간색으로 물들이기(안전=초록/초과=빨강)**: 상태가 강렬히 보이나 "합격/불합격
  판정서"로 오독 → **히어로는 중립, 상태는 배지·게이지 전담**(§3-1·§0-1).
- **"초과" 구간도 amber로(빨강 미사용)**: 사이트 빨강 회피 관행에 부합하나 "주의"와 색이 겹쳐 40%
  선을 넘은 사실이 약하게 전달됨 → 1차 제안은 **빨강 작은 배지 + 중립 히어로 + 사실 서술**, 마스터
  불허 시 **amber-100(주의보다 진한 톤) + 아이콘·텍스트 차등**으로 폴백(§3-3, §9-마스터1).
- **규제선 선택 토글(1금융 40% / 2금융 50%)을 입력에 노출**: 정확도는 오르나 입력이 6→7개로 늘고
  YMYL 오인(기관별 판정) 리스크 → **기본 40% 고정 + 각주로 50% 병기**, 필요 시 기획 회신으로 추가(§9).
- **기존 대출을 금액/이자율/기간으로 받아 내부 재계산**: 정합성은 좋으나 다건 대출 시 입력 폭증 →
  **연간 원리금상환액 직접 입력(선택)**(§2-2 ⑥). 기획 회신 시 전환 가능.
- **결과에 "대출 가능/제한" 판정 문구·아이콘**: 클릭·체류에 유리해 보이나 대출 심사 오인 = 애드센스
  정책·YMYL 최대 위반 → **절대 미채택**. 사실 서술 + 다중 면책만(§6).

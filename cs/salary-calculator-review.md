# 연봉 실수령액 계산기 — CS 콘텐츠 검수 (2026-07-14)

검수자: CS팀 / 대상: `lib/calculators.ts`(salary-net-calculator interpretation·faq), `components/SalaryNetCalculator.tsx`
검증 방법: 대표 예시(연봉 4,000만 / 부양가족 1 / 월 비과세 20만 / 자녀 0)를 실제 `lib/salary.ts` 로직으로 재계산해 대조.
(마스터 주: CS팀은 Write 권한이 없어 마스터가 대신 저장함.)

## 총평
근사 한계 고지는 상단 amber 배너 "항상 노출" + interpretation·FAQ·인라인 노트 3중이라 발견성 양호. "기준소득월액","특별소득공제"는 사용자 노출 텍스트엔 없음(재발 아님). interpretation 첫 문단의 개별 원 단위 예시 수치가 실제 계산기 출력과 어긋나는 것이 유일한 즉시 수정 대상.

## [높음 — 즉시 수정] H-1. interpretation 예시 개별 수치 ≠ 실제 계산기 출력
- 위치: `lib/calculators.ts` L114 interpretation 1문단.
- 문제: 대표 예시 입력의 개별 항목을 "약" 없이 확정 원 단위로 서술하나 실제 엔진 출력과 불일치.

  | 항목 | interpretation | 실제 엔진 | 차이 |
  |---|---|---|---|
  | 국민연금 | 148,810 | 148,817 | +7 |
  | 건강보험 | 112,640 | 112,643 | +3 |
  | 장기요양 | 14,580 | 14,582 | +2 |
  | 고용보험 | 28,200 | 28,199 | −1 |
  | 근로소득세 | 105,930 | 105,888 | −42 |
  | 지방소득세 | 10,590 | 10,588 | −2 |
  | 4대보험 합 | 304,230 | 304,241 | +11 |

  (요약 근사치 "약 291만원/약 3,495만원"은 반올림 후 일치, 문제없음.)
- CS 리스크: 이 입력값은 사이트 대표 예시라 사용자가 그대로 따라 입력할 확률 최고. Tier③에 148,817이 뜨는데 해설은 148,810 → "계산 틀림" 오류 제보 직접 유발.
- 근본 원인: planning §1-4 검산표 라운딩(국민연금 10원 절사 등)이 실제 코드(1원 절사 Math.floor)와 달라, 세 문서(planning 105,935 / interpretation 105,930 / 코드 105,888)가 전부 상이.
- 제안: 개별 항목 수치를 실제 엔진 출력값으로 교체(정확) 또는 "약" 근사 표기로 통일.
- 이관: 개발팀(문구 수치) + 기획팀 공유(§1-4 검산표 라운딩 오차).

## [중간 — 개선 제안]
- M-1. 화면에 급여 입력 "저장 안 됨" 안내 없음(FAQ 6번에만 존재) → 민감 재무정보라 폼 근처 1줄 안내 권장. 개발팀 문구+디자인팀 배치.
- M-2. 부양가족 도움말이 "누구를 넣는지" 미안내, "간이세액표 공제대상 가족 수"는 전문용어. 기획팀 문안.
- M-3. "간이세액표" 전문용어 5회+ 반복인데 최초 풀이 없음(07-13 대출 C-3/C-4 패턴). 노출 1회 괄호 풀이. 기획팀 문안.
- M-4. 모든 고지가 "근로소득세만 근사"로 읽혀 4대보험은 정확하다 오인 소지(실제론 라운딩·기준소득월액 차이 존재, planning §7-2). 기획팀 문안 검토.

# 4대보험료 계산기 (`salary/four-insurance-calculator`) 콘텐츠·계산모델 기획안

- 작성일: 2026-07-14
- 작성: 기획팀
- 배경: `planning/calculator-lineup.md` 로드맵 11순위 "4대보험료 계산기"(/salary)를 라이브 전환하기 위한 콘텐츠·계산모델 초안. `lib/calculators.ts`에 신규 등록될 계산기의 `interpretation`/`faq`/`shortDescription`/`relatedSlugs`를 채우고, 디자인·개발이 서로 다른 입력필드/출력항목을 만들지 않도록 **입력·출력·요율상수·라운딩을 여기서 확정**한다. 직전 완성한 `planning/electricity-bill-calculator-content.md`(#9)·`planning/loan-prepayment-fee-content.md`(#10)와 동일한 § 번호 체계·톤·검증앵커 방식을 따랐다.
- **이 계산기의 위치(정합성 최우선)**: 이 계산기는 **연봉 실수령액 계산기(`salary-net-calculator`)의 보조 페이지**다. "실수령액 말고 4대 보험료만, 그리고 회사가 얼마 내는지까지 보고 싶다"는 파생 검색수요를 흡수한다. 따라서 **근로자 부담분의 요율·산식·라운딩은 `lib/salary.ts`에 이미 확정·검증된 상수를 그대로 재사용**해야 하며, 두 계산기에 같은 월 과세대상급여를 넣으면 근로자 4대 보험료가 **원 단위까지 동일**해야 한다(어긋나면 CS·신뢰도 리스크). 본 기획안의 검증앵커(§1-4)는 `lib/salary.ts`와 동일한 산식·라운딩으로 손계산했다.
- **차별점(핵심)**: 기존 연봉 계산기가 근로자 부담분만 보여주는 것과 달리, 이 계산기는 **근로자 부담 / 사업주(회사) 부담 / 합계**를 함께 보여준다. 4대 사회보험 정보연계센터·사람인 등 대표 4대보험료 계산기가 모두 이 3열 구조라 검색 사용자 기대에 부합한다(§8-2에서 마스터 확정).
- slug/category(확정 대상): category `salary`(확정). **slug 제안 `four-insurance-calculator`** — 라인업/과제 개념 URL은 `/salary/four-insurance`이나, 실제 코드 등록 slug는 기존 `-calculator` 접미 관행(`salary-net-calculator`·`electricity-bill-calculator`·`loan-interest-calculator`)에 맞춰 `four-insurance-calculator`를 제안한다(#9 전기요금과 동일 처리, §8-5 마스터 확인).
- **YMYL 등급: 낮음~중간** — 연봉 계산기(중간)보다 약간 낮다. 세금(간이세액표 근사)이 없고 4대 보험 요율만 다루며, "이미 정해진 요율의 단순 곱셈"이라 근로소득세 근사 같은 정확도 리스크가 없다. 다만 (1) 요율이 **매년 1월 갱신**되어 하드코딩 방치 시 오정보, (2) 사업주 부담(고용보험 규모별)·산재보험(업종별)은 조건에 따라 달라져 "참고용" 고지가 필요하다. 연봉·전기요금과 동일한 3중 고지(결과해석·FAQ·면책) + 4대 사회보험 정보연계센터 링크 병기(E-E-A-T)로 관리한다.

## 1. 계산 모델 (개발팀 구현용 — 2026년 기준, 수치 확정 명세)

### 1-0. 입력 항목 정의 (디자인·개발 공통 확정) — **단순형**

**설계 원칙**: 이 계산기는 연봉 계산기와 달리 세금 파이프라인이 없으므로 **입력을 최소화한 단순형**이 적절하다. 사용자가 "내 월급"으로 바로 아는 값 하나(월 과세대상급여 = 보수월액)를 필수로 받는다.

| 입력 항목 | 타입 | 필수 | 기본값 | 비고 |
|---|---|---|---|---|
| 월 과세대상 급여(보수월액) | number(원, 천단위 콤마) | **필수** | — | 4대 보험료 부과 기준이 되는 **월 소득**. 비과세(식대 등)를 뺀 과세대상 급여. placeholder 예: `3,000,000` |
| 사업장 규모(고용보험 사업주 부담용) | select | 선택 | **150인 미만** | 고용보험 사업주 부담의 고용안정·직업능력개발사업 요율이 규모별로 다름(§1-2). 근로자 부담·다른 3개 보험에는 영향 없음. v1 기본 "150인 미만(0.25%)" |

- **입력을 "월 과세대상급여" 1개로 확정하는 이유(연봉 계산기와의 정합)**: 연봉 계산기의 내부 변수 `monthlyTaxable`(② 월 과세대상급여 T = 월급여총액 − 월비과세)와 **동일한 값**을 직접 입력받는다. 즉 이 계산기의 입력 = 연봉 계산기의 T. 이렇게 하면 (1) 연봉 계산기 사용자가 거기서 본 "월 과세대상급여"를 그대로 넣어 4대 보험료를 재확인할 수 있고, (2) 두 계산기의 근로자 4대 보험료가 원 단위까지 일치한다.
- **연봉 입력 방식(대안)은 채택하지 않음**: "연봉 → ÷12 → 비과세 차감"까지 이 계산기에서 다시 받으면 연봉 계산기와 입력이 중복되고, 비과세·부양가족 등 연봉 계산기의 복잡도를 그대로 끌고 온다. **보조 페이지는 단순하게** 두고, 연봉→실수령액 전체 흐름은 연봉 계산기가 담당한다(관련 계산기로 상호 링크, §5). 단, 사용자가 연봉만 아는 경우를 위해 입력 보조문구에 "연봉을 12로 나눈 뒤 비과세(식대 등)를 뺀 금액"을 안내한다(§7).
- **비과세 처리 명확화**: 국민연금(기준소득월액)·건강보험(보수월액)·고용보험은 모두 **비과세를 제외한 과세대상 급여**를 기준으로 부과한다. 따라서 입력값은 "비과세를 이미 뺀 과세대상 월급여"임을 라벨·보조문구로 분명히 한다(비과세 포함 총급여를 넣으면 보험료가 과대 계산됨).
- **사업장 규모 입력 근거**: 근로자 부담분과 국민연금·건강보험·장기요양·고용보험(실업급여)은 규모와 무관하다. 오직 **고용보험 사업주 부담의 고용안정·직업능력개발사업 요율만** 규모별(0.25~0.85%)로 달라지므로, 이 값만 select로 받는다. 대부분 사용자가 근로자 부담을 보러 오므로 규모 select는 **기본 150인 미만**으로 두고, 근로자만 볼 사용자는 건드리지 않아도 된다.

### 1-1. 계산 파이프라인 (단계별 = 결과 카드에 표시)

```
입력: 월 과세대상급여 T(원), 사업장 규모 size ∈ {150인미만, 150인이상 우선지원, 150~1000인미만, 1000인이상·국가지자체}

STEP 0  국민연금 기준소득월액 P = clamp( floor_to_1000(T), 410,000, 6,590,000 )
          // 천원단위 절사 후 하한 41만·상한 659만으로 캡 (lib/salary.ts와 동일)

STEP 1  국민연금
          근로자 = floor( P × 4.75% )
          사업주 = floor( P × 4.75% )              // 근로자와 대칭(합계 9.5%)

STEP 2  건강보험 (보수월액 = T)
          근로자 = floor( T × 3.595% )
          사업주 = floor( T × 3.595% )              // 근로자와 대칭(합계 7.19%)

STEP 3  장기요양보험 (각자의 건강보험료에 곱)
          근로자 = floor( 건강보험_근로자 × 12.9457% )
          사업주 = floor( 건강보험_사업주 × 12.9457% )

STEP 4  고용보험
          근로자 = floor( T × 0.9% )                        // 실업급여분만
          사업주 = floor( T × (0.9% + 고용안정요율(size)) )  // 실업급여 0.9% + 고용안정·직업능력개발
              고용안정요율: 150인미만 0.25% / 150인이상 우선지원 0.45%
                            150~1000인미만 0.65% / 1000인이상·국가지자체 0.85%
              → v1 기본 150인미만: 사업주 고용보험 = floor( T × 1.15% )

STEP 5  산재보험 = 계산 제외(전액 사업주·업종별 상이) — §1-3, 면책·안내만

STEP 6  합계
          근로자 합계 = 국민연금_근로자 + 건강_근로자 + 장기요양_근로자 + 고용_근로자
          사업주 합계 = 국민연금_사업주 + 건강_사업주 + 장기요양_사업주 + 고용_사업주
          총 합계    = 근로자 합계 + 사업주 합계
```

- **라운딩 정책(lib/salary.ts와 동일 — 필수 준수)**: 각 보험료는 **원 단위 절사(버림, `Math.floor`)**. 국민연금 기준소득월액은 **천원 단위 절사 후 clamp**. 사업주 부담분도 각각 **개별 절사**한다(근로자분을 그대로 복사하지 않고 사업주 base로 다시 floor — 국민연금·건강보험은 근로자와 base가 같아 결과가 같지만, 산식상 각각 floor로 명세).
- **파이프라인 의존성**: 장기요양(STEP 3)은 반드시 STEP 2의 건강보험료(각 부담분)를 base로 곱한다. T에 직접 요율을 곱하는 (B)방식도 수학적으로 같으나(연봉 계산기 §1-2 참조), **근로자/사업주를 분리 표시하므로 "각자의 건강보험료 × 12.9457%"(A방식)가 표시 정합상 명확**하다.
- **건강보험 보수월액 상·하한**: 연봉 계산기 §1-2 주1과 동일하게 v1은 캡 미적용(일반 급여 구간 무영향). 국민연금만 clamp 적용.
- **결과 카드에는** 최소 (근로자 합계) + (사업주 합계) + (총 합계) 3개 대표값 + 4개 보험 항목별 근로자/사업주 내역을 표시. 노출 범위·접이식 여부는 디자인 협의(§7-3).

### 1-2. 요율표 (2026년) — 확정 (근로자분은 lib/salary.ts 상수 재사용)

| 보험 | 근로자 요율 | 사업주 요율 | 부과 기준 | 상·하한 캡 | 출처 |
|---|---|---|---|---|---|
| 국민연금 | **4.75%** | **4.75%** | 기준소득월액(천원절사·clamp) | 하한 **410,000** / 상한 **6,590,000** (2026.7~2027.6) | 국민연금공단(요율 9.5%, 노사 절반) |
| 건강보험 | **3.595%** | **3.595%** | 보수월액(= T) | (v1 캡 미적용, 연봉계산기 준용) | 국민건강보험(직장 7.19%, 노사 절반) |
| 장기요양 | 건강보험료 × **12.9457%** | 건강보험료 × **12.9457%** | 각자의 건강보험료 | (건강 캡 상속) | 보건복지부(장기요양 0.9448%) |
| 고용보험(실업급여) | **0.9%** | **0.9%** | T | 별도 캡 없음 | 고용노동부(실업급여 총 1.8%, 노사 절반) |
| 고용보험(고용안정·직업능력개발) | **없음(0%)** | **0.25%**(150인미만 기본) | T | 규모별 0.25~0.85% | 근로복지공단(사업주 전액 부담) |
| **산재보험** | **없음(0%)** | **업종별 상이(계산 제외)** | (제외) | — | 근로복지공단(전액 사업주·업종별) |

- **근로자분 요율 = `lib/salary.ts` 상수 그대로 재사용**: `NATIONAL_PENSION_RATE=0.0475`, `HEALTH_INSURANCE_RATE=0.03595`, `LONG_TERM_CARE_MULTIPLIER=0.129457`, `EMPLOYMENT_INSURANCE_RATE=0.009`, 국민연금 하한/상한 `410,000`/`6,590,000`, `floorTo1000`+clamp. **개발팀은 이 계산기를 위해 요율을 새로 정의하지 말고 salary.ts 상수를 import**할 것(요율 갱신 시 한 곳만 고치면 두 계산기가 동시에 갱신되어 정합 유지).
- **사업주 대칭분(신규 상수 아님, 재사용)**: 국민연금·건강보험·장기요양·고용보험 실업급여는 근로자와 **요율이 대칭**이라 같은 상수를 재사용한다(base만 사업주 기준). 국민연금·건강보험은 base가 근로자와 동일(P, T)이므로 사업주 보험료 = 근로자 보험료.
- **신규로 추가할 상수 1개(개발 전달)**: 고용보험 사업주의 **고용안정·직업능력개발사업 요율**. `EMPLOYMENT_STABILITY_RATE_UNDER_150 = 0.0025`(150인 미만). 규모 select를 구현하면 `{0.0025, 0.0045, 0.0065, 0.0085}` 맵으로. **2026년 교차확인값**(§참고): 150인미만 0.25% / 150인이상 우선지원 0.45% / 150~1000인미만 0.65% / 1000인이상·국가지자체 0.85%.
- **산재보험은 v1 제외(보수적 원칙)**: 산재보험료율은 **업종별로 크게 다르고(광업 등 고위험 수%~서비스업 0.6%대) 매년 근로복지공단이 업종별로 고시**한다. 전기요금 계산기가 "검증 불가능한 산재 업종 평균값을 임의로 넣지 않고 면책 처리"한 선례(#9)·중도상환 계산기의 보수적 원칙과 동일하게, **정확도 앵커로 검증할 수 없는 산재는 넣지 않고** "산재보험료는 전액 사업주 부담이며 업종별로 달라 이 계산기에 포함하지 않는다"고 결과해석·FAQ·면책에 명시한다(§1-3·§2·§3). 이는 "회사 총부담을 실제보다 약간 낮게 보여주는" 방향의 안전한 단순화다.

### 1-3. 단순화한 부분 (근사·한계 — 결과해석·FAQ·면책에 반영)

1. **산재보험 미포함**: 전액 사업주 부담·업종별 요율이라 v1 제외. → "회사 실제 총부담은 산재보험만큼 이 계산값보다 큼" 고지(해석·FAQ 3).
2. **고용보험 사업주 부담 규모 의존**: 고용안정·직업능력개발 요율이 사업장 규모별(0.25~0.85%)로 달라, 기본 150인 미만(0.25%) 기준. → 규모가 다르면 사업주 고용보험이 달라짐 고지.
3. **건강보험 보수월액 상·하한·연말정산 미반영**: 건강보험은 연 1회 보수총액 신고로 정산(연말정산)되고 보수월액 상·하한이 있으나 v1 미반영(일반 구간 무영향). → 실제 공제액과 소액 차이 가능 고지(FAQ 4).
4. **요율 시점 의존**: 4대 보험 요율은 매년 1월(건강·장기요양 등) 갱신. 조사 시점 요율이며 개정 시 달라짐. → 면책.

## 2. interpretation (결과 해석 텍스트, 5단 구조)

`lib/calculators.ts`의 `interpretation` 필드 초안. 1문단(결과 요약)은 §1-4 앵커B(T=300만, 150인 미만) 기준이며, 실제로는 **사용자 입력값에 따라 동적 생성**되어야 한다(기존 라이브 계산기와 동일 처리, §7 개발 전달).

```
월 과세대상 급여 300만원(150인 미만 사업장 기준)이라면, 근로자가 부담하는 4대 보험료는 국민연금 142,500원, 건강보험 107,850원, 장기요양보험 13,961원, 고용보험 26,999원 등 월 291,310원입니다. 회사(사업주)는 같은 급여에 대해 국민연금 142,500원, 건강보험 107,850원, 장기요양보험 13,961원, 고용보험 34,500원 등 월 298,811원을 부담하며(산재보험 제외), 근로자와 회사 부담을 합치면 월 590,121원이 됩니다.

4대 보험료는 2026년 요율 기준으로 계산합니다. 국민연금은 근로자와 사업주가 기준소득월액의 각 4.75%(합계 9.5%), 건강보험은 보수월액의 각 3.595%(합계 7.19%)를 부담하고, 장기요양보험은 각자의 건강보험료에 12.9457%를 곱해 계산합니다. 고용보험은 실업급여분을 근로자와 사업주가 각 0.9%씩 부담하며, 사업주는 여기에 고용안정·직업능력개발사업 보험료(150인 미만 사업장 0.25%)를 더 냅니다. 국민연금은 기준소득월액에 상한(월 659만원)과 하한(월 41만원)이 있어, 급여가 상한을 넘어도 보험료가 일정 금액에서 더 늘지 않습니다.

급여명세서에서 실제로 공제되는 금액은 근로자 부담분이며, 사업주 부담분은 회사가 별도로 납부합니다. 국민연금 상한 때문에 고소득 구간에서는 급여가 올라도 국민연금료가 늘지 않는 반면, 건강보험과 고용보험은 급여에 비례해 계속 늘어납니다. 이 계산기는 산재보험을 포함하지 않았는데, 산재보험료는 전액 사업주가 부담하고 업종별로 요율이 크게 달라 일률적으로 계산하기 어렵기 때문입니다. 따라서 회사의 실제 총부담은 이 계산값에 업종별 산재보험료가 더해진 금액입니다.

4대 보험 요율은 국민연금공단, 국민건강보험공단, 근로복지공단(고용·산재보험)이 매년 고시하며, 4대 사회보험 정보연계센터(4insure.or.kr)에서 최신 요율과 정확한 보험료를 확인할 수 있습니다. 요율은 매년 1월을 전후로 갱신되므로 계산 시점에 따라 달라질 수 있습니다.

입력하신 급여는 브라우저 내에서만 계산에 사용되며 서버로 전송·저장되지 않습니다. 본 계산 결과는 참고용이며, 실제 보험료는 회사가 신고한 기준소득월액·보수월액과 그 적용 시점, 사업장 규모(고용보험 사업주 부담), 업종(산재보험) 등에 따라 차이가 날 수 있으니 정확한 금액은 4대 사회보험 정보연계센터와 각 공단에서 확인하시기 바랍니다.
```

## 5. 관련 계산기 링크 추천 (자동 결과가 이상적 — override는 "정합 안정용"으로만 권장)

- `getRelatedCalculators()`는 `relatedSlugs` override가 없으면 "같은 카테고리 우선 후 나머지, 배열 순서"(`[...sameCategory, ...rest].slice(0,3)`)로 채운다.
- 이 계산기가 live 전환되면 `salary` 카테고리 live는 `salary-net-calculator`·`severance-pay-calculator`·(신규)four-insurance 3개다. self 제외 시 sameCategory=[**연봉 실수령액, 퇴직금**](배열 순서상 salary-net이 severance보다 앞), rest 첫 번째는 배열 맨 앞 `age-calculator`(만 나이).
- **자동 결과 = 연봉 실수령액 + 퇴직금 + 만 나이** → 과제 요구(연봉 실수령액 + 퇴직금/근속연수 + 만나이 크로스)와 **정확히 일치**하고 라인업 §3-3 원칙(같은 카테고리 2 + 다른 카테고리 1)도 충족한다.
- **override 필요 여부(결론)**: **엄밀히는 불필요**(자동 로직이 이미 이상적 3개를 이상적 순서로 반환). 다만 이 계산기는 **연봉 실수령액 계산기의 보조 페이지**라 부모 페이지(연봉 실수령액)가 **항상 1순위로 노출**되는 것이 중요하다. 향후 `calculators.ts` 배열 순서나 salary 카테고리 계산기가 추가로 바뀌어도 이 우선순위가 흔들리지 않도록 **명시적 override를 권장**한다(퇴직금·근속연수 계산기가 세트 링크로 override를 건 것과 동일 취지의 "정합 안정용"). 권장값:
  ```ts
  relatedSlugs: ["salary-net-calculator", "severance-pay-calculator", "age-calculator"],
  ```
- **역방향 링크는 자동으로 해결됨**(추가 작업 불필요): `salary-net-calculator`는 현재 override가 없어 자동 로직을 쓰는데, four-insurance가 live가 되면 salary-net의 sameCategory=[severance, four-insurance]가 되어 연봉 계산기 관련목록에 **4대보험료 계산기가 자동 노출**된다. 즉 연봉↔4대보험 상호 링크가 코드 수정 없이 성립한다.
- 문장형 앵커 예시(라인업 §3-3): "실수령액까지 보려면 연봉 실수령액 계산기를 이용해보세요." / "퇴직할 때 받을 금액이 궁금하다면 퇴직금 계산기도 확인해보세요." / "정확한 만 나이가 필요하다면 만 나이 계산기를 이용해보세요."

## 7. 디자인팀에 넘길 확정 요약 (화면 설계용 — 목록 불일치 핑퐁 방지)

- **대표값(히어로)**: 근로자 부담 합계 `N원`(가장 궁금한 값). 그 옆/아래 사업주 부담 합계 + 총 합계 병기

- **명시 배지/각주**: "산재보험 제외(전액 사업주·업종별)" 안내를 사업주 열 근처에 각주로. "참고용 예상 보험료" 배지
- 접이식 여부: 연봉 계산기(3-tier)처럼 대표값(근로자 합계) + 3행 요약(근로자합/사업주합/총합) 항상 노출 + 4보험 개별 내역 접힘 검토 가능. 확정은 디자인 스펙 수령 후.

2. **[개발, 우선] §8-6 정합 규칙** — 근로자분 요율·라운딩은 `lib/salary.ts` 상수 재사용(신규 정의 금지). 신규 상수는 고용안정 요율 1건. §1-1 파이프라인·§1-4 앵커(근로자 열은 연봉 계산기와 원 단위 일치)로 회귀 테스트 고정.
3. **[개발] §8-8 동적 요약 + 규모 select**, §8-7 relatedSlugs override(권장).
4. **[디자인] §7 입력 2개/출력 3열(근로자/사업주/합계) 구조 그대로 사용**, §7-3 배너·접이식·산재 제외 각주.
5. 위 사항은 기획팀 단독 결정 범위 밖이거나 타 팀 작업에 직접 영향을 주어 마스터 경유 전달을 요청한다. 카테고리(`salary`)·YMYL 등급(낮음~중간)·근로자분 요율(salary.ts 재사용 확정)은 변경하지 않았다.

### 되돌릴 이슈(핑퐁) — 개발팀 확인 요청 1건
- **연봉 계산기 국민연금 표시값과의 미세 불일치 점검**: 현재 라이브 `salary-net-calculator` interpretation의 예시값은 국민연금 **148,817원**(T=3,133,000 기준, `floor(3,133,000×0.0475)=148,817`)인데, 그 원천이 된 `planning/salary-net-calculator-content.md` §1-4 표에는 **148,810원**으로 적혀 있어 기획 문서와 코드가 7원 어긋나 있다(코드가 정답). 4대보험료 계산기가 같은 요율을 재사용하므로, 이 기회에 **연봉 계산기 기획문서 §1-4의 국민연금 148,810 → 148,817로 정정**해 두 문서·코드의 앵커를 일치시킬 것을 개발/마스터에 요청한다(본 4대보험료 기획 §1-4 앵커는 처음부터 코드와 동일한 floor 방식으로 계산했으므로 불일치 없음).

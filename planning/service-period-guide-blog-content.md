- 유일무이한 정보깊이: **계산기가 내놓는 3가지 값(재직일수 / 근속기간 N년 M개월 D일 / 소수 근속연수)의 차이를 정확히 구분**하는 것. 이 앵글이 이 글의 존재 이유.

- 발행일: 2026-07-17
- 수치: 마스터 실엔진(lib/service-period.ts) 재검산 앵커 표(A~H)만 인용. 표 밖 임의 계산·반올림 없음. 단 '5×365=1,825'는 마스터가 명시적으로 허용한 '틀린 어림값'으로만 대조 사용.
- 카니벌라이제이션 회피: 퇴직금 금액(severance-pay-guide-2026)·연차 일수(annual-leave 글)는 반복 설명 없이 CTA로만 위임. 이 글은 '기간을 어떻게 세는가'까지만.

## §3. 마스터 확인 요청

방향 전환 이슈 없음 — **append 승인**만 필요합니다. 선택지 확인 요청 3건:

1. **title 대안**: 현재 "근속기간 계산법 완전정리 — 재직일수·근속연수 정확히 세는 법 (2026년)"으로 기존 시리즈 패턴("○○ 계산법 완전정리 — …")과 통일. 대안으로 3개념 구분을 전면화한 "근속기간·재직일수·근속연수 차이 완전정리 (2026년)"도 가능. 검색 헤드 키워드('근속기간 계산법')를 앞세우는 현재안을 기본으로 두었습니다 — 교체 원하시면 지시 바랍니다.

2. **tags 선정 근거**: 5개를 '세는 법' 각도의 고볼륨 롱테일로 구성했습니다.
   - "근속기간 계산" / "재직일수 계산": 계산기 직결 헤드 롱테일(검색 의도 = 값 산출).
   - "근속연수 계산법" / "근속기간 산정": '방법·산정 기준'을 찾는 정보성 쿼리 → 이 글 본문(3개념 구분)과 relevance 일치.
   - "재직일수 세는 법": 이 글만의 앵글(+1·양 끝 포함)에 정확히 대응. 후보였던 "입사일 재직일수"보다 본문 내용 일치도가 높다고 판단해 채택 — '입사일 재직일수'가 더 낫다면 교체 가능.

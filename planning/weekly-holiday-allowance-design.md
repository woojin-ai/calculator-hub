## 0. 최저시급 상수 단일 소스 (개발 필수 전제)

- **2026 최저시급 = 10,320원 (확정, 고용노동부 고시)**. 문서 상충(10,320 vs 10,240)은 10,320으로 종결.
- 이 값은 **상수 1곳에서만 관리**한다(예: `lib/` 내 `MINIMUM_WAGE_2026 = 10_320` 및 기준연도 라벨). 화면 3곳(① 시급 힌트, ② 시급 프리필 후보 안내, ③ 하단 출처 병기의 "2026년 최저시급" 표기)이 전부 이 상수를 참조해야 하며, 매년 1월 갱신 시 상수 1곳만 바꾸면 화면 전체가 갱신되도록 한다.
- 화면에는 `2026년 최저시급 10,320원`처럼 **기준연도를 항상 병기**한다(연 단위 갱신 키워드라 연도 없는 숫자는 오정보 리스크).

## 1. 전체 레이아웃 구조 (섹션 배치)

계산기 페이지 표준 템플릿을 따른다(위→아래):

```
[페이지 제목 H1 + 리드 문장]        ← 페이지 셸(기존 계산기 상세 템플릿 공통)
─────────────────────────────
① 상단 amber 근사 고지 배너 (role="note", 항상 노출)
② 입력 폼 카드
    - 그룹 라벨 "시급 · 근로시간"
    - [시급] (전체 폭)
    - [1주 소정근로시간] · [주 근무일수(선택)]  (2열)
    - [월 환산 표시] 세그먼트 토글
    - [주휴수당 계산하기] 버튼
    - 브라우저 내 계산 안내 문구
③ 결과 카드 (계산 후 노출)
    - (정상) Tier① 주휴수당/주  +  Tier② 보조값 행 + 동적 요약
    - (게이트) 주 15h 미만 → 중립 회색 "미발생" 안내 카드
─────────────────────────────
④ 결과 해석 텍스트(interpretation, 500~1000자)   ← calculators.ts
⑤ FAQ 4개                                        ← calculators.ts
⑥ 관련 계산기 3개 링크                            ← 페이지 셸 공통
⑦ 하단 면책 + 공식출처 병기                        ← §5 참조
```

- ②③은 이 컴포넌트가 렌더링. ①은 컴포넌트 최상단(폼 카드 바깥 위). ④~⑦은 페이지 셸/`calculators.ts`가 담당(기존 12개와 동일 구조).
- **광고 배치**: 기존 계산기 상세 페이지의 광고 슬롯 관례를 그대로 따른다 — 결과 카드(③)와 해석 텍스트(④) **사이**, 그리고 관련 계산기(⑥) **아래**에만 배치 가능. 입력 폼 카드 내부·계산 버튼 주변·결과 대표값 바로 옆에는 광고를 넣지 않는다(클릭 유도 오인 방지, 사용자 경험 우선). AdSense 심사 대기 중이므로 실제 광고 유닛은 승인 후 활성화하되, **레이아웃상 광고 자리(placeholder 여백)를 결과-해석 사이에 확보**해 둔다.

## 2. 입력 필드 레이아웃 상세

폼 카드: `rounded-xl border border-brand-border bg-brand-surface p-4 sm:p-6`, 내부 `<form className="flex flex-col gap-4">`. 그룹 라벨은 퇴직금과 동일 패턴 `text-xs font-semibold text-brand-text-secondary`.

### 필드 순서·폭·문구 초안

| # | 필드 | 폭 | 라벨 | placeholder | 힌트 | 필수 |
|---|------|-----|------|-------------|------|------|
| 1 | 시급 | 전체 폭 | `시급 (원)` | `10,320` | `1시간당 세전 시급을 입력하세요. 2026년 최저시급은 10,320원입니다.` | 필수 |
| 2 | 1주 소정근로시간 | 2열 좌 | `1주 소정근로시간 (시간)` | `20` | `근로계약서상 1주 동안 일하기로 정한 시간입니다. 휴게시간·연장근로는 제외합니다.` | 필수 |
| 3 | 주 근무일수 | 2열 우 | `주 근무일수 (선택)` | `5` | `참고용입니다. 주휴수당 금액 계산에는 사용되지 않습니다.` | 선택 |

- **입력 방식**: 시급·소정근로시간·근무일수 모두 `type="text" inputMode="numeric"` + 천단위 콤마 서식(시급은 퇴직금 `formatAmountInput` 재사용, 시간·일수는 콤마 불필요하니 정수 서식). 클래스는 `${INPUT_BASE} border-brand-border`(에러 시 `border-brand-warning`).
- **시급 프리필/힌트**: placeholder에 `10,320`(최저시급)을 노출하되 **자동 입력값은 비워 둔다**(사용자가 자기 시급을 넣도록). 힌트 문구에서 "2026년 최저시급은 10,320원입니다"로 기준값 안내. 최저시급 미만 입력 시 막지는 않되(자유 입력) 힌트로 참고만 제공(별도 경고 배너는 스코프 밖 — 필요 시 §7 핑퐁 참조).
- **주 근무일수의 위상**: 핵심 공식(`(소정근로시간/40)×8×시급`)에 근무일수는 들어가지 않는다. 기획 스펙상 "(선택)"이므로 **금액 계산 미참여**임을 힌트에 명시해 사용자 오해를 차단한다(입력했는데 결과가 안 변한다는 CS 문의 예방). 결과 요약 문구에서 "주 O일 근무" 맥락 표현으로만 활용.
- **에러 처리**: 퇴직금과 동일. 필드 하단 `text-xs text-brand-warning` + `role="alert"`, 입력에 `aria-invalid`. 트리거: 시급 미입력/0 이하, 소정근로시간 미입력/0 이하. 상한 검증(주 168h 초과 등)은 개발 판단(참고: §7).

### 2-2. 40시간 상한 시각 처리

- 공식은 소정근로시간 40h 초과 시 40으로 상한. 사용자가 45h를 넣어도 주휴수당은 40h 기준으로 산출된다. 이때 결과 Tier②에 **"적용 소정근로시간: 40시간(주 40시간 상한 적용)"** 행을 노출해 왜 값이 더 안 오르는지 설명한다(입력 45 → 적용 40 괴리를 명시하지 않으면 오류로 오인).

## 3. 월 환산 토글 UX + 계산 버튼 + 결과 카드

### 3-1. 월 환산 토글

- **형태: 세그먼트 토글**(체크박스 아님). `LoanInterestCalculator` L230~258의 년/개월 토글 패턴을 그대로 계승 — `role="radiogroup"`, 내부 `type="button" role="radio" aria-checked`, 선택 시 `bg-brand-primary text-white` / 비선택 `bg-transparent text-brand-text-secondary`. 컨테이너 `flex items-stretch gap-0.5 rounded-md border border-brand-border bg-white p-0.5`.
- 옵션 2개: **`주 단위만`** / **`월 환산 포함`** (기본값 = `월 환산 포함`. 검색 의도상 "한 달에 얼마?"가 크므로 켠 상태를 기본으로).
- 라벨(위): `결과 표시 범위` (그룹 라벨 `text-xs font-semibold text-brand-text-secondary`).
- **동작**: 토글 상태에 따라 **결과 카드 Tier②의 "월 예상 주휴수당" 행만 표시/숨김**. `주 단위만` 선택 시 월 관련 행(월 주휴수당, 월 환산 시간)을 숨기고 대표값(주휴수당/주)·주휴 포함 실질시급·적용 소정근로시간만 노출. 대표값 자체는 항상 주 단위 주휴수당으로 고정(토글 무관).
- 체크박스가 아닌 세그먼트를 쓰는 이유: 사이트 내 on/off 성격 토글이 모두 세그먼트(년/개월, 단리/복리 등)라 일관성 유지, 그리고 "무엇을 켜는지"가 라벨로 명확히 보임.

### 3-2. 계산 버튼

- 문구: **`주휴수당 계산하기`**
- 스타일: 퇴직금과 동일 `h-12 w-full rounded-lg bg-brand-primary px-5 text-base font-semibold text-white transition-colors hover:bg-brand-primary-hover`.
- 버튼 아래 안내(퇴직금 계승, 문구만 조정): `입력하신 시급·근로시간 정보는 브라우저 안에서만 계산되며 서버에 저장·전송되지 않습니다.` (`text-xs text-brand-text-secondary`)

### 3-3. 결과 카드 — 정상(주 15h 이상)

컨테이너: 퇴직금 정상 결과와 동일 `mt-6 rounded-xl border-l-4 border-brand-accent bg-gradient-to-br from-emerald-50 to-white p-4 sm:p-6`. 계산 후 `resultRef.scrollIntoView`로 스무스 스크롤(폼이 접힘선 아래로 밀리는 것 방지, 퇴직금 관례).

**Tier ① 대표값 — 주 주휴수당**
- 상단 라벨: `1주 주휴수당` (`text-sm font-medium text-brand-text-secondary`)
- 큰 숫자: `text-4xl font-bold tabular-nums text-brand-accent sm:text-[2.5rem]`, `formatWon(주휴수당)` + `원`(`text-base font-medium text-brand-text-secondary`)
- 보조 설명: `세전 기준, 1주 개근 시 지급되는 유급휴일 수당입니다.` (`text-xs text-brand-text-secondary`)

**동적 결과 요약 문구** (`text-sm text-brand-text-secondary`, 정적 텍스트 금지 — 입력값 반영)
- 예시(월 환산 포함, 주 20h·시급 10,320원):
  `시급 10,320원, 1주 소정근로시간 20시간으로 개근할 경우, 1주 주휴수당은 약 41,280원이며, 한 달(4.345주)로 환산하면 약 179,000원입니다. 주휴수당을 포함한 실질 시급은 약 12,384원입니다.`
- 예시(주 단위만): 월 문장 절을 생략하고 실질시급 절만 유지.

**Tier ② 보조값 행** (AmountRow/TextRow, 퇴직금 L378~396 패턴, 계산 흐름 순)
1. `적용 소정근로시간` = `{clamp된 시간}시간` (40h 상한 적용 시 뒤에 `(주 40시간 상한)` 부기) — TextRow
2. `주휴 환산 시간` = `{(시간/40)×8}시간` (소수 1자리) — TextRow
3. `1주 주휴수당` = 대표값과 동일 금액 — AmountRow (요약 확인용, 항상 노출)
4. **`월 예상 주휴수당`** = `주휴수당 × 4.345` — AmountRow **(토글 `월 환산 포함`일 때만)**
5. `주휴 포함 실질 시급` = `((소정근로시간 + 주휴환산시간) × 시급) ÷ 소정근로시간` = `formatWon` — AmountRow
6. (선택 노출) `주휴 포함 월 예상 급여` = `(소정근로시간 + 주휴환산시간) × 4.345 × 시급` — AmountRow **(토글 `월 환산 포함`일 때만)**

> 행 개수: 주 단위만 = 4행 / 월 환산 포함 = 6행. `formatWon` 사용, `tabular-nums`. 금액 행 값 우측 `원`은 AmountRow 컴포넌트가 붙임(퇴직금 AmountRow 그대로 재사용 가능).

### 3-4. 결과 카드 — 게이트(주 15h 미만, 주휴수당 미발생)

퇴직금 "1년 미만" 카드(L400~436)를 **구조 그대로 차용**. 중립 회색 = `mt-6 rounded-xl border border-brand-border bg-brand-surface p-4 sm:p-6`(그라데이션·accent 라인 없음, `ⓘ`는 `text-brand-text-secondary`).

- 제목(`text-base font-semibold text-brand-text`): **`주휴수당이 발생하지 않습니다`**
- 부제(`text-sm text-brand-text-secondary`): `1주 소정근로시간이 15시간 미만입니다.`
- 구분선 후 TextRow: `입력한 소정근로시간` = `{시간}시간 (주 {일수}일)`
- 하단 안내(`text-xs text-brand-text-secondary`, §4 카피):
  `주휴수당은 1주 소정근로시간이 15시간 이상이고 그 주를 개근한 경우에 발생합니다. 15시간 미만인 초단시간 근로자에게는 근로기준법상 주휴수당·유급휴일이 적용되지 않습니다. 여러 사업장에서 일하는 경우 사업장별로 15시간 여부를 판단합니다.`

## 4. 게이트 안내 카드 카피 초안 (중립·비유도)

- 제목: `주휴수당이 발생하지 않습니다`
- 본문: `1주 소정근로시간이 15시간 미만인 초단시간 근로자에게는 근로기준법 제18조 제3항에 따라 주휴수당(유급휴일)이 적용되지 않습니다. 소정근로시간을 15시간 이상으로 다시 확인해 입력해 보세요. 근로시간 산정에 대한 정확한 판단은 고용노동부(1350) 상담을 권장합니다.`
- 톤: 퇴직금 게이트와 동일하게 **중립·정보전달형**. "받을 수 있다/없다"를 단정하되 광고 클릭이나 특정 행동을 유도하지 않는다. 색상은 회색 계열(경고 amber·성공 emerald 아님).

## 5. 상단 amber 배너 + 하단 공식출처 병기

### 5-1. 상단 amber 근사 고지 배너 (컴포넌트 최상단, 항상 노출)

퇴직금 배너 구조 계승(`role="note"`, `mb-6 flex items-start gap-2 rounded-lg border border-amber-200 bg-amber-50 px-4 py-3`, `ⓘ`는 `text-brand-warning`, 본문 `text-sm text-amber-900`). 카피:

> `주휴수당은 근로기준법상 법정 산식으로 계산한 참고용 예상액입니다. 실제 지급액은 근로계약·소정근로시간 산정 방식·개근 여부에 따라 달라질 수 있습니다. 미지급 등 분쟁은 고용노동부(1350)·관할 노동청 상담을 권장합니다.`

### 5-2. 하단 공식출처 병기 (페이지 셸/`calculators.ts` 하단 면책 영역)

기존 계산기 하단 면책 컴포넌트 관례를 따르되, 이 계산기 전용으로 아래 출처를 병기:

## 6. 모바일(375px) 뷰포트 레이아웃 고려사항

- **폼 카드 패딩**: `p-4 sm:p-6` — 375px에서는 `p-4`(16px)가 적용돼 좌우 여백 확보. 필드 세로 간격 `gap-4` 유지.
- **2열 필드(소정근로시간 · 주 근무일수) 처리**: 두 필드 모두 **짧은 숫자 입력(2~3자리)**이라 375px에서도 2열(`grid grid-cols-2 gap-3 sm:gap-4`) 유지가 자연스럽다. 단 **라벨이 2줄로 접힐 수 있음**(`1주 소정근로시간 (시간)`) → 퇴직금 2열처럼 라벨에 `min-h-10`을 주어 좌/우 라벨 높이를 맞추고 입력창 baseline이 어긋나지 않게 한다.
  - 만약 개발 중 375px에서 라벨 2줄+힌트로 세로가 과하게 길어지면, **폴백: 소정근로시간을 전체 폭 단독 행으로, 주 근무일수(선택)를 그 아래 전체 폭**으로 내려도 된다(주 근무일수는 계산 미참여 선택 필드라 우선순위 낮음). 2열 유지가 1순위, 세로 스택은 폴백.
- **세그먼트 토글**: `flex-1` 2등분이라 375px에서 각 버튼 폭 충분(`주 단위만` / `월 환산 포함` 라벨이 한 줄에 들어감). `h-9` 터치 타깃 유지.
- **대표값 숫자**: `text-4xl`(약 36px)가 375px에서 넘칠 수 있으나 주휴수당은 6자리 이하(수만~수십만원)라 `flex-wrap items-baseline`로 `원` 단위가 자연 줄바꿈되게 처리(퇴직금 L359 동일).
- **결과 Tier② 행**: `flex items-center justify-between` 라벨-값 양끝 정렬. 라벨이 긴 행(`주휴 포함 실질 시급`)은 375px에서도 값과 겹치지 않게 라벨 `text-brand-text-secondary`가 필요 시 줄바꿈. `적용 소정근로시간` 부기 `(주 40시간 상한)`이 길면 값 영역이 좁아지므로, 부기는 별도 `text-xs`로 값 아래 배치하거나 괄호를 짧게(`상한 적용`).
- **계산 버튼**: `w-full h-12` 풀폭 유지(엄지 도달 영역).
- **광고 자리**: 결과-해석 사이 광고는 모바일에서 반응형(`display:block`, 폭 100%)으로 삽입되도록 자리만 확보. 결과 카드 바로 아래 첫 스크롤에서 광고가 대표값을 가리지 않도록 결과 카드와 광고 사이 `mt` 여백 확보.

## 7. interpretation / FAQ 방향성 (디자인 권고 — 최종 카피는 기획/개발이 `calculators.ts`에 확정)

### 7-1. interpretation (결과 해석 텍스트, 500~1000자) 방향

한 문단 몰아쓰기 금지, 2~3개 소주제로 스캔 가능하게. 담을 내용:
1. **주휴수당이 무엇인지·왜 받는지**: 1주 소정근로시간 15시간 이상 + 개근 시 유급휴일(1일치 임금)을 별도 지급받는 제도(근로기준법 제55조). "안 준다고 알고 있었는데 받을 수 있다"는 검색 의도에 답.
2. **공식 해설**: `(1주 소정근로시간 ÷ 40) × 8 × 시급`이 나온 이유(주 40시간 만근 시 8시간분, 그에 비례). 40시간 상한의 의미. 초단시간(15h 미만) 제외.
3. **월 환산·실질시급의 의미**: 주휴수당을 더하면 실질 시급이 명목 시급보다 높아진다는 점, 4.345주 환산 근거. 급여 실수령·4대보험과 연결(내부링크 유도).
- 톤: 정보전달·중립. 단정적 법률자문 회피, "상황에 따라 다를 수 있다" 안전장치. 최저시급 10,320원은 §0 상수 흐름과 어긋나지 않게 연도 병기.

### 7-2. FAQ 4개 방향

1. **`주휴수당은 아르바이트도 받을 수 있나요?`** → 고용형태 무관, 1주 15시간 이상+개근이 기준. 초단시간 제외.
2. **`주 15시간 미만이면 정말 못 받나요?`** → 게이트 근거(근로기준법 제18조 제3항) 설명, 여러 사업장 합산 불가(사업장별 판단).
3. **`주휴수당은 최저임금에 포함되나요? 실질 시급이 왜 더 높나요?`** → 주휴수당 별도 지급 개념, 실질시급 산출 의미(계산기 결과와 연결).
4. **`계산한 주휴수당을 못 받으면 어떻게 하나요?`** → 고용노동부(1350)·관할 노동청 진정 안내(면책 톤 유지, 특정 유료서비스 유도 금지).

- FAQ 렌더는 기존 계산기 FAQ 컴포넌트(아코디언/정적) 관례를 그대로 따른다 — 디자인 신규 요소 없음.

## 9. 핑퐁 / 확인요청

**기획팀 확인요청**
1. **주 근무일수(선택) 필드의 존치 여부**: 핵심 공식·월 환산 어디에도 근무일수가 들어가지 않는다(개근 판단은 계산기가 하지 않음). 현재 스펙은 "(선택)참고용"이라 화면에는 넣되 "계산 미참여"를 힌트로 명시하는 안으로 설계했다. **결과에 반영되지 않는 입력을 노출하는 것 자체가 혼란**이라면 이 필드를 제거하는 것도 대안(디자인은 제거 시 더 단순해짐 선호). 기획 판단 요청.
2. **월 환산 근거 4.345주 표기 노출 수준**: interpretation에 "4.345주(365일÷12개월÷7일)" 산출근거를 각주로 노출할지, 아니면 결과 요약에만 "한 달(4.345주)"로 간단 표기할지. 정확성 vs 간결성 트레이드오프 — 기획 카피 확정 시 결정.
3. **주휴 포함 월 예상 급여 행(Tier② 6번) 노출 여부**: 스펙 출력 명세는 "월 예상 주휴수당"까지다. 실질 체류·전환(salary-net 연결)엔 "주휴 포함 월급"이 유용하나 스펙 초과다. 넣을지 기획 확인(넣으면 salary-net-calculator 내부링크 CTA와 자연 연결).

**개발팀 인계 이슈**
4. **입력 상한/검증 규칙**: 소정근로시간 상한(주 최대 근로시간 등)·시급 상한·음수/소수 처리 규칙은 엔진 스펙 사안. 디자인은 40h 초과 시 "적용 40시간" 시각 처리만 규정했다(§2-2). 비현실적 값(예: 소정근로시간 200) 입력 시 에러/클램프 정책은 개발이 정해 회신 바람.
5. **최저시급 미만 시급 입력 시 처리**: 현재 디자인은 자유 입력 + 힌트 안내만(막지 않음). 최저임금 위반 경고 배너를 띄울지는 스코프 밖으로 뒀다. CS에서 "최저임금 미만인데 계산됨" 문의가 돌아오면 §2 힌트를 경고 배너로 승격하는 안을 재설계하겠다.

**마스터 확인**
6. 톤앤매너·카테고리(/salary)·스택 변경 없음. 신규 브랜딩 요소 없이 기존 계산기 컨벤션 100% 계승이라 별도 승인 불요로 판단. 이견 시 회신 바람.

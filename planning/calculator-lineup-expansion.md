## 2. 신규 계산기 후보 #13 ~ #16

각 후보: 제안 slug / 소속 카테고리 / 검색수요·계절성·CPC·경쟁도(정성 등급+근거) / 내부링크 시너지(기존 12개 중 3개) / AdSense·YMYL 리스크 + 필수 면책·공식출처 / 계산 로직 개요.

### #13. 주휴수당 계산기  ★최우선 (3절 참조)
- **slug**: `weekly-holiday-allowance-calculator` — 카테고리 `/salary`
- **검색수요**: **매우 높음**. 알바·시간제 근로자 모집단이 방대하고, 주휴수당은 "받아야 하는데 못 받는" 분쟁성 키워드라 상시 검색. 계절성: **강함(1월)** — 매년 1월 최저임금 개정 시 "2026 주휴수당/시급" 재검색 스파이크(WebSearch 상위에 "2026년 기준" 문구를 단 페이지가 다수 = 연 단위 갱신 수요 확인).
- **CPC**: 낮음~중간(생활정보성이나 채용·급여관리 SaaS·노무 광고주와 겹침). 검색량이 압도적으로 커서 총광고수익(RPM×트래픽) 기여도가 높은 유형.
- **경쟁도**: 높음(알바천국·알바몬·시프티 등 대형 선점). 다만 계산 로직이 단순하고 데모데이·개인 블로그(brunch)도 상위 노출 → 신규 사이트도 UX·결과해설·FAQ로 진입 가능한 몇 안 되는 고검색량 키워드.
- **내부링크 시너지 3개**: salary-net-calculator(시급→월급 실수령 연결), four-insurance-calculator(주휴 포함 월급의 4대보험 공제), service-period-calculator(근속·근로일수 흐름).
- **YMYL/AdSense 리스크**: **낮음~중간(노동법 기반)**. 근로기준법상 요율은 아니나 "주 15시간 이상 개근" 지급조건·최저임금 수치가 매년 바뀜. 필수 면책: "본 계산은 참고용이며 실제 지급액은 근로계약·소정근로시간 산정 방식에 따라 다를 수 있습니다. 미지급 분쟁은 고용노동부(1350)·노동청 상담을 권장합니다." 공식출처 병기: 고용노동부 최저임금 고시, 근로기준법 제55조(주휴).
- **계산 로직 개요**:
  - 입력: 시급(원), 1주 소정근로시간(h), (선택)주 근무일수, (토글)월 환산 여부, (참고 표시)2026 최저시급 10,320원 *추정 — 착수 시 고시 확인*.
  - 핵심 공식: `주휴수당 = (1주 소정근로시간 ÷ 40) × 8 × 시급`, 단 소정근로시간이 40h 초과 시 40으로 상한. 지급조건 게이트: 주 15h 미만이면 "주휴수당 미발생" 안내.
  - 월 환산: `주휴 포함 월급 ≈ (소정근로시간 + 주휴환산시간) × 4.345주 × 시급`.
  - 출력: 주 단위 주휴수당, 월 예상 주휴수당, 주휴 포함 실질 시급.

### #14. 적금·예금 이자 계산기
- **slug**: `savings-interest-calculator` — 카테고리 `/loan`
- **검색수요**: 높음(스테디). 예·적금 만기 수령액 확인은 금리 변동기마다 상시 발생. 계절성: 약함~중간(연말·신년 재테크 시즌 소폭 상승).
- **CPC**: **높음** — 금융/재테크는 애드센스 상위 CPC군(예적금·증권·저축은행 광고주). 기존 /loan(대출) 클러스터가 전부 "빚" 성격인데, "저축(자산)" 성격 계산기를 넣어 금융 광고 인벤토리를 넓힌다.
- **경쟁도**: 높음(은행·핀다) 그러나 ourcalc·calculatorhost 등 소형 계산기 사이트가 다수 상위 → 진입 가능. 은행 계산기는 상품 가입 유도라 "단순 계산만" 수요가 별도 존재.
- **내부링크 시너지 3개**: loan-interest-calculator(이자 개념 짝), loan-prepayment-fee(상환 vs 저축 의사결정), salary-net-calculator(월 저축 여력 산정).
- **YMYL/AdSense 리스크**: **낮음~중간**. 실제로는 결정론적 수학(원리금·이자과세)이라 "조언"이 아니어서 리스크가 대출보다 낮음. 단 금융 카테고리 통념상 면책 권장: "실제 적용 금리·우대조건·과세는 금융기관·상품별로 다릅니다. 예금자보호 한도 등은 금융감독원을 확인하세요." 공식출처: 금융감독원 소비자포털.
- **계산 로직 개요**:
  - 모드 토글: 예금(일시예치) / 적금(매월납입).
  - 입력: 예치금 또는 월납입액, 기간(개월), 연이율(%), 단리/복리 선택, 이자과세(일반 15.4% / 세금우대 9.5% / 비과세).
  - 공식: 예금 단리 이자 `= 원금×r×(개월/12)`; 예금 월복리 `= 원금×(1+r/12)^n − 원금`. 적금 단리 이자 `= 월납입×r×(n(n+1)/2)/12`. 세후 `= 세전이자×(1 − 세율)`.
  - 출력: 세전이자, 이자과세액, 세후 수령액, 실효수익률(%).

### #15. 자동차세 계산기
- **slug**: `car-tax-calculator` — 카테고리 `/life`
- **검색수요**: 중간~높음. 계절성 **매우 강함** — 정기분 6월·12월, **연납 신청 1월** 3개 스파이크(WebSearch 상위 다수가 "연납 할인" 중심 = 계절 검색 확인).
- **CPC**: **중간~높음** — 자동차보험·중고차·주유/정비 광고주와 겹쳐 /life 중 CPC가 높은 편. bmi/electricity/unit-converter로 편중된 /life에 수익형 페이지를 보강.
- **경쟁도**: 중간(위택스 공식 + 카눈·보배드림 등). 배기량·차령·연납 조합이 은근히 번거로워 "간편 계산" 대체 수요 존재.
- **내부링크 시너지 3개**: unit-converter(cc↔배기량 단위 감각), electricity-bill-calculator(생활 고정비·공과금 짝), dday-calculator(연납 신청/납부기한 D-day).
- **YMYL/AdSense 리스크**: **낮음~중간(지방세)**. 세액표·연납 할인율·전기차 정액이 법령으로 매년 바뀜. 필수 면책: "예상 세액이며 실제 부과액은 위택스/지자체 고지서를 확인하세요." 공식출처: 위택스(지방세), 행정안전부 지방세법.
- **계산 로직 개요**:
  - 입력: 차종(비영업용 승용/기타), 배기량(cc) 또는 전기차 여부, 최초등록연도(차령), 납부방식(정기분/연납 시기).
  - 공식(비영업 승용): cc당 세액 `1,000cc 이하 80원 / 1,600cc 이하 140원 / 1,600cc 초과 200원` × 배기량 = 자동차세, + **지방교육세 30%**. 전기차: 정액 13만원(비영업) *추정 — 고시 확인*. **차령 경감**: 3년차부터 매년 5%씩, 최대 50%. **연납 할인**: 1월 최대(검색결과 1월 5.0%·3월 3.8%·6월 2.5%·9월 1.3% — *연납 할인율은 2023년 이후 축소 개정 이력이 있어 반드시 재확인, 추정*).
  - 출력: 연 자동차세(교육세 포함), 연납 시 할인액·납부액, 기분별(6·12월) 납부액.

- **slug**: `acquisition-tax-calculator` — 카테고리 `/loan`(부동산 자금·주택구매 여정 클러스터에 배치, 단 카테고리 적합성은 4절에서 마스터 확인 요청)
- **검색수요**: 높음. 주택 매매·증여 시 필수 확인. 계절성: 약함이나 부동산 정책·거래량 뉴스에 민감(이슈성 스파이크).
- **CPC**: **높음** — 부동산/모기지/법무·세무 광고주로 금융 최상위 CPC군.
- **경쟁도**: 높음(위택스·부동산계산기.com·삼쩜삼). 조건 조합(주택수·조정지역·생애최초)이 복잡해 "정확 계산" 신뢰 확보 시 차별화 여지.
- **내부링크 시너지 3개**: dsr-calculator(주택구매 자금여력), loan-interest-calculator(주담대 상환액), loan-prepayment-fee(대출 상환 계획) — **"집 사기" 원스톱 퍼널**을 완성.
- **YMYL/AdSense 리스크**: **높음(세무·재무 YMYL, 복잡도 상)**. 주택수·조정대상지역·중과·생애최초 감면·농특세 등 변수 많고 정책 개정 잦음. 필수 면책: "예상 세액이며 실제는 주택수 판정·지역·감면요건에 따라 크게 달라집니다. 반드시 위택스·관할 지자체·세무사 확인. 취득일로부터 60일 내 신고·납부." 공식출처: 위택스, 지방세법. **오정보 시 실질 재산 피해 → 4절 갱신 책임 명시 필요.**
- **계산 로직 개요**:
  - 입력: 물건종류(주택/오피스텔/토지/상가), 취득구분(매매/증여/상속), 취득가액, 보유 주택수, 조정대상지역 여부, 전용면적(85㎡ 초과 시 농특세), 생애최초 여부.
  - 공식(주택 매매 1주택): 6억 이하 1% / 6억~9억 슬라이딩 `세율 = (취득가액×2/3억 − 3)/100`(1~3%) / 9억 초과 3%. + 지방교육세(취득세율의 50%×10% = 0.1~0.3%) + 농특세 0.2%(85㎡ 초과). 다주택·조정: 2주택 8%, 3주택 12% *추정 — 개정 확인*.
  - 출력: 취득세, 지방교육세, 농특세, 합계 및 신고기한 안내.

## 3. #13 최우선 지정 — 주휴수당 계산기 (`weekly-holiday-allowance-calculator`, /salary)

**다음 실행에서 디자인+개발에 바로 넘길 1순위로 주휴수당 계산기를 지정한다.**

선정 근거(리스크·검색량·구현난이도·시너지 종합):
1. **검색량 최상위·모집단 최대**: 알바·시간제 근로자는 국내 계산기 수요층 중 가장 광범위. 대형 채용 플랫폼조차 전용 계산기를 상단에 두는 = 총트래픽이 큰 키워드.
2. **구현난이도 최저**: 핵심이 단일 공식(`(소정근로시간/40)×8×시급`) + 조건 게이트뿐. 1차 12개와 같은 클라이언트 순수 연산, 서버·외부데이터 불필요.
3. **리스크 낮음**: 노동법 인접이나 요율표 없이 최저임금 수치 1개만 갱신하면 됨. AdSense 심사 대기 중에도 안전한 저리스크 콘텐츠.
4. **시너지 최강**: 이미 라이브인 salary-net-calculator·four-insurance-calculator·severance-pay-calculator와 "시급→월급→공제→퇴직" 급여 퍼널을 완성 → /salary 카테고리 내부링크 밀도·체류시간 즉시 상승.
5. **계절성 부스트**: 매년 1월 최저임금 개정마다 재검색 스파이크가 구조적으로 발생.

디자인/개발 인계용 최소 스펙: 입력(시급·1주 소정근로시간·주 근무일수·월환산 토글), 출력(주/월 주휴수당·주휴 포함 실질시급), 조건 게이트(주 15h 미만 미발생 안내), 하단 면책+고용노동부/근로기준법 제55조 출처, 관련 계산기 3개(위 시너지). 최저시급 값은 **상수 1곳에서 관리**하고 착수 시 고용노동부 고시로 확정(검색결과 10,320/10,240 상충 → 확인 필요, 추정).

## 4. 다른 팀·마스터 확인 필요 사항 (기획팀 단독 결정 범위 밖)

1. **#16 취득세 계산기의 카테고리 배치**: 세무 성격상 `/life`가 자연스러우나, "집 사기 퍼널"(dsr·loan-interest·prepayment-fee) 내부링크 시너지를 위해 `/loan` 배치를 제안했다. 카테고리 시맨틱과 URL/사이트맵 분류에 영향을 주므로 **마스터 확인 요망**. (URL은 `/calculator/{slug}`라 경로 자체는 안 바뀌고, 카테고리 목록·CollectionPage 소속만 달라짐.)
2. **법정 수치 갱신 책임(개발팀/운영)**: 4개 후보 모두 매년 바뀌는 법정 수치를 포함(최저임금·이자과세율·자동차세액표/연납할인율·취득세율). 특히 #16은 오정보 시 실질 재산 피해 우려. **수치 상수의 단일 소스화 + 연 1회 갱신 체크리스트**를 개발팀 운영 항목으로 등록할지 마스터 판단 필요. 문서 내 2026 수치는 전부 "추정"이며 착수 시 고시 재확인 전제.
3. **프로그래매틱 페이지 확장 없음 확인**: 이번 4개는 1차 unit-converter처럼 롱테일 대량 페이지 생성을 수반하지 않는 단일 페이지 계산기다(사이트맵 규모 급증 없음). 만약 향후 "배기량별/취득가액대별" 랜딩을 대량 생성하는 프로그래매틱 SEO로 확장한다면 그건 별도 구조 결정이므로 그때 재확인.

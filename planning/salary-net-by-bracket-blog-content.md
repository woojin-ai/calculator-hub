## 자가 점검 (타입 유니온 준수)
- section type: 사용한 것은 `paragraph`, `heading`, `list`, `callout`, `calculatorCta` 5종뿐. 유니온 밖 타입 없음. OK
- `callout` variant: `info`(전제 안내 1회) + `warning`(YMYL 면책 1회). "tip" 등 미사용. OK
- `heading`: `text`만 사용, `level` 필드 없음. OK
- `calculatorCta`: `slug`만 사용, `label` 없음(기존 9편과 동일). OK
- `list`: `ordered: false` + `items` 문자열 배열. OK
- 필수 필드(slug/title/description/category/tags/publishedDate/relatedCalculatorSlugs/body) 전부 존재. `updatedDate`는 optional이라 생략. OK
- relatedCalculatorSlugs 3개 모두 라이브(기존 글에서 salary-net-calculator·four-insurance-calculator·severance-pay-calculator slug 사용 확인). OK
- 수치: 마스터 재검산 표 값만 사용. 임의 재계산·다른 연봉 구간 추가 없음. 5,000만원 상세 분해는 마스터가 제공한 세부값 그대로. OK

## 중복 회피 점검 (1편 대비)
- 1편: 공제 항목 정의 + 계산법 + 연봉 4,000만원 단일 예시.
- 2편(이 글): 9개 구간 표 조회 + 구간별 실수령 비율 변화 서술 + 5,000만원 상세. 예시 연봉·앵글이 달라 카니벌라이제이션 최소화. 내부링크는 1편과 다른 조합(four-insurance 추가)으로 차별화.

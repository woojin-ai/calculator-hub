## 1. 계산 모델 (개발팀 구현용 — 수치 확정 명세)

### 1-0. 입력 항목 정의 (디자인·개발 공통 확정)

**최소 스펙 원칙**: 사용자가 실제로 아는 값만 받는다. 과제가 제안한 4개 입력(중도상환금액/수수료율/잔존기간/총기간) 중 **③을 "잔존기간"이 아니라 "대출 실행 후 경과기간"으로 바꾼다**(핵심 결정, §1-0-1 근거). 표준공식의 "잔존일수"는 만기까지가 아니라 **면제기준기간(3년 캡) 내 남은 일수**라, 사용자에게 "잔존기간"을 직접 물으면 30년 주담대에서 "잔존 29년"을 입력하는 치명적 오입력이 발생한다. 대신 "언제 대출받았는지(경과)"를 물으면 명확하고, 잔존은 내부에서 파생한다.

| 입력 항목 | 타입 | 필수 | 기본값 | 비고 |
|---|---|---|---|---|
| 중도상환(조기상환) 금액 | number(원, 천단위 콤마) | **필수** | — | 이번에 미리 갚을 원금. placeholder 예: `100,000,000` |
| 중도상환수수료율 | number(%, 소수 2자리 허용) | **필수** | **0.7**(상수, §1-2) | 본인 대출 약정서/은행 공시값 입력 안내. placeholder 예: `0.65` |
| 대출 실행 후 경과기간 | number(개월) | **필수** | — | 대출받은 뒤 지금까지 지난 개월. placeholder 예: `12` |
| 대출 총 약정기간 | number(개월) | 선택 | **36**(3년) | 비우면 3년 캡 가정. 3년 미만 대출만 실제값 입력. placeholder 예: `36` |

- **입력 검증(확정)**: 모든 값 `≥ 0`, 음수 불가. 수수료율은 상식 범위(예: 0~5%) 밖 입력 시 안내 권장(엣지, 디자인 협의). 대출 총 약정기간은 `≥ 1개월`.
- **입력 4개로 고정**: 단위변환·연봉·전기요금에서 planning/design 목록 불일치 핑퐁이 있었던 전례에 따라, 입력은 위 **4개로 확정**하고 §7 디자인 요약과 1:1 일치시킨다.
- **3년 캡은 입력이 아니라 자동 로직**: 별도 토글을 두지 않고, 분모 = `min(총 약정기간, 36개월)`으로 항상 자동 적용한다(§1-1 STEP 0). "총 약정기간이 3년을 넘으면 자동으로 3년으로 계산됩니다(은행 관행)"를 보조문구로 노출(§7). 토글 UI를 두면 사용자가 캡 개념을 이해해야 해서 오히려 복잡해진다.

#### 1-0-1. 왜 이 입력 구성인가 (근거 기록 — §8에서 최종 승인)

1. **③ = 경과기간(잔존기간 직접입력 아님) 채택** — 표준공식의 "잔존일수"는 **면제기준기간(3년 캡 또는 만기 중 이른 것) 내 남은 일수**다. 사용자에게 "잔존기간"을 직접 받으면 (a) 만기까지 잔존(예: 30년 대출의 29년)과 (b) 3년 면제까지 잔존을 혼동해 **분자가 분모(3년)를 초과하는 오입력**이 구조적으로 발생한다. #9 전기요금이 "정확 입력이 어려운 값을 받지 않는다"는 원칙을 세운 것과 동일하게, **명확한 값(경과기간)만 받고 잔존은 파생**한다. 잔존기간 = 분모 − 경과기간(§1-1 STEP 1).
2. **개월(일 아님) 단위 채택 + 비율 계산** — 표준공식은 일수 기반이나, 분자·분모를 **같은 단위(개월)로 쓰면 비율이 보존**된다(예: 24÷36 = 730일÷1,095일 = 0.667). 사용자가 "몇 개월 전 대출"은 알아도 "며칠"은 모른다. 개월 단위 근사로 소액(수십~수백 원) 오차는 나지만 YMYL-중간에 맞춰 면책 처리(§1-3). 날짜(실행일·상환일) 입력으로 정밀화하는 안은 §8-2 대안으로 남긴다.
3. **총 약정기간 선택·기본 36개월(3년 캡)** — 주택담보대출은 대부분 3년 초과라 분모가 항상 36개월로 고정되므로, 총기간을 비우면 3년 캡을 가정한다. **신용대출 등 3년 미만 만기**만 실제 총기간을 입력받아 분모 = min(총기간, 36)로 처리(3년 미만이면 만기가 면제시점). 이 단순화는 대다수 주담대 사용자의 입력을 하나 줄여준다.
4. **수수료율 사용자 직접 입력(default 0.7%)** — 2025년 개편으로 신규/기존 계약, 은행·상품별 요율이 크게 갈려(§0) 하드코딩하면 오정보다. #9 전기요금이 변동 요율을 상수 분리한 것과 동일하게, **입력값으로 받고 default만 상수로 제공**한다.

### 1-1. 계산 파이프라인 (단계별 중간값 = 결과 카드에 표시)

```
입력:
  A   = 중도상환금액(원)        [≥ 0]
  f   = 수수료율(%)             [≥ 0]
  E_m = 경과기간(개월)          [≥ 0]   // 대출 실행 후 지난 개월
  T_m = 대출 총 약정기간(개월)  [선택, 기본 36, ≥ 1]

STEP 0  면제기준기간(분모) 결정 — 3년 캡
          D_m = min(T_m, 36)                 // 3년 = 36개월 상한(은행 관행)
STEP 1  잔존기간 산출
          R_m = D_m − E_m
          if (R_m ≤ 0)  →  면제 확정: Fee = 0, ratio = 0, isExempt = true → 종료
          // R_m > 0 이면 자동으로 0 < R_m ≤ D_m 범위(E_m ≥ 0 이므로)
STEP 2  잔존비율
          ratio = R_m / D_m                  // 0 < ratio ≤ 1
STEP 3  수수료율 소수화
          f_dec = f / 100
STEP 4  중도상환수수료
          Fee = floor( A × f_dec × ratio )   // 원 미만 절사
                = floor( A × f × R_m / (100 × D_m) )   // 동치, 부동소수 오차 최소화 권장식
```

- **핵심 의존성**: 분자 잔존기간 `R_m`과 분모 `D_m`은 **반드시 같은 3년 캡 창(window)** 안에서 산출한다. 분모만 캡하고 분자를 만기 기준으로 두면 비율이 폭증하는 대표 오류(§1-0-1). 그래서 `R_m = D_m − E_m`으로 묶는다.
- **면제 분기(STEP 1)**: `E_m ≥ D_m`(3년 또는 만기 경과)이면 수수료 0원 확정. 결과 카드는 금액 대신 "중도상환수수료 면제 대상(대출 실행 후 3년 경과 등)" 상태를 노출(§7-2).
- **부동소수 주의**: `24/36` 같은 순환소수를 곱하면 미세 오차가 나므로, 개발팀은 `A × f × R_m / (100 × D_m)` 순서로 정수 곱을 먼저 하고 마지막에 나눗셈 후 `Math.floor` 권장(STEP 4 동치식).
- **결과 카드에는** 최소 (중도상환수수료 Fee) + (적용 수수료율 f%) + (면제기준기간 D_m·잔존기간 R_m·잔존비율 ratio) + (중도상환금액 A) + (면제 여부)를 표시. 노출 정도는 디자인 협의(§7-2).
- `floor(x)` = 원 미만 절사(`Math.floor`). 라운딩 정책은 §8-4에서 확정.

### 1-2. 수수료 상수 (default·캡) — 리서치 확정값 + 확인 필요 표기

**아래는 WebSearch로 조사한 2026년 기준값이다. 블로그·은행·핀테크 교차확인값이며, 수수료율은 은행·상품·약정 시점별로 달라 default는 어디까지나 시작값이다(사용자 입력이 최종). 상수 모듈로 분리해 갱신 가능하게 둘 것(§8-5).**

|---|---|---|---|
| 면제기준기간 캡(분모 상한) | **36개월(3년=1,095일)** | 은행권 표준 관행, 3년 경과 시 면제 | 교차확인(§8-3 확정) |

| (참고) 주택담보대출 요율대 | **약 0.6~0.7%** | 2025.1.13~ 신규계약, 고정 1.4%→0.56~0.65%대·변동 1.2%→0.65%대 인하 | 참고(약정별 상이) |

- **정확성 주의 3건(개발·마스터 필독)**:
  1. **수수료율은 사용자 입력이 최종**이다. default 0.7%는 힌트일 뿐, 결과 해석·면책에서 "본인 약정서/은행 공시값 확인 후 입력"을 강조한다(§2·§3). 대출이자 계산기 YMYL 관리와 동일 톤.
  2. **2025년 인하는 신규계약(2025.1.13~)에만 적용**된다. 기존 대출자는 종전 요율이라, 계산기가 특정 요율을 강제하면 오정보. 그래서 입력 방식.
  3. **3년 캡·면제는 은행권 표준 관행**이나 상품에 따라 예외(일부 정책자금·중도상환수수료 없는 상품)가 있을 수 있어 "일반적" 표현으로 고지(§3).
- **최소 수수료·일할 계산(엣지, v1 제외)**: 일부 은행은 최소 수수료 하한이나 실제 일수(윤년 포함) 기반 정밀 계산을 쓴다. v1은 개월 근사·하한 없음으로 두고 "실제 청구와 다를 수 있음"으로 처리 제안(§8-2 확인).

### 1-3. 단순화한 부분 (근사·한계 — 결과해석·FAQ·면책에 반영)

v1에서 실제 은행 청구 대비 단순화한 항목. 모두 YMYL-중간에 맞춰 고지한다.

1. **개월 단위 근사(일할 미반영)**: 표준공식은 실제 일수 기반이나 v1은 개월 비율로 근사 → 수십~수백 원 오차 가능(FAQ 4·면책).
2. **은행별 계산 방식 차이**: 최소 수수료 하한, 일할 계산, 상품별 캡 예외 등은 미반영 → "은행별로 다를 수 있음" 고지(FAQ 4).
3. **수수료율 시점 의존**: 2025 인하·신규/기존 계약 차이·상품별 상이 → default가 아니라 본인 약정 요율 입력 필요(FAQ 3·면책).
4. **부대비용 미포함**: 인지세·근저당 말소비용 등 갈아타기 총비용은 별개 → 이 계산기는 "중도상환수수료 한 항목"만 산출(FAQ 5·면책).

### 1-4. 검증용 기준값 (QA·마스터가 독립 재현할 앵커) — 기획팀 직접 손계산

과거 CS 이슈(해설 예시 수치 ≠ 엔진 실측) 재발 방지를 위해, 결과해석·FAQ의 모든 예시를 아래 표로 고정한다. **앵커 B를 결과해석 대표 예시로 사용**한다(주택담보대출 갈아타기 = 이 계산기 주 검색의도). 개발팀은 엔진 출력이 이 표와 일치하는지, QA·마스터는 아래 손계산 근거로 독립 재현할 것. **원 미만 절사(floor) 기준**이다.

| # | 중도상환금액 A | 수수료율 f | 경과 E_m | 총기간 T_m | 분모 D_m | 잔존 R_m | 잔존비율 ratio | **중도상환수수료 Fee** | 확인 포인트 |
|---|---|---|---|---|---|---|---|---|---|
| A | 10,000,000원 | 0.5% | 6개월 | 24개월 | 24 | 18 | 18/24 = 75% | **37,500원** | 총기간<3년(캡 미적용), 만기 24개월이 면제시점 |
| **B(대표)** | 100,000,000원 | 0.65% | 12개월 | 360개월 | **36(3년 캡)** | 24 | 24/36 ≈ 66.7% | **433,333원** | 주담대 3년 초과 → 분모 3년 캡, 1년 경과 |
| C | 50,000,000원 | 1.2% | 24개월 | 360개월 | 36(3년 캡) | 12 | 12/36 ≈ 33.3% | **200,000원** | 2년 경과, 잔존 짧아 수수료 급감(슬라이딩) |
| D | 50,000,000원 | 1.2% | 40개월 | 360개월 | 36(3년 캡) | −4 → 0 | 0(면제) | **0원(면제)** | 3년 경과 → 면제 분기(STEP 1) |

**손계산 근거(독립 재현용):**

- **앵커 A** `1천만원·0.5%·경과 6개월·총 24개월`:
  - D_m = min(24, 36) = **24**. R_m = 24 − 6 = **18** (>0). ratio = 18/24 = **0.75**.
  - Fee = floor(10,000,000 × 0.5 × 18 / (100 × 24)) = floor(10,000,000 × 0.005 × 0.75) = floor(50,000 × 0.75) = floor(37,500) = **37,500원**.
- **앵커 B(대표)** `1억원·0.65%·경과 12개월·총 360개월`:
  - D_m = min(360, 36) = **36**(3년 캡). R_m = 36 − 12 = **24** (>0). ratio = 24/36 = **0.66666…**.
  - Fee = floor(100,000,000 × 0.65 × 24 / (100 × 36)) = floor(100,000,000 × 0.0065 × 0.66666…) = floor(650,000 × 0.66666…) = floor(433,333.33…) = **433,333원**.
- **앵커 C** `5천만원·1.2%·경과 24개월·총 360개월`:
  - D_m = min(360, 36) = **36**. R_m = 36 − 24 = **12** (>0). ratio = 12/36 = **0.33333…**.
  - Fee = floor(50,000,000 × 1.2 × 12 / (100 × 36)) = floor(50,000,000 × 0.012 × 0.33333…) = floor(600,000 × 0.33333…) = floor(200,000.0) = **200,000원**.
  - **슬라이딩 효과(해설·FAQ용)**: 같은 대출을 앵커 B(1년 경과)와 비교하면, 경과가 12→24개월로 늘수록 잔존비율이 66.7%→33.3%로 절반이 되어 수수료도 절반 수준으로 줄어드는 구조를 이 대비로 설명한다.
- **앵커 D** `5천만원·1.2%·경과 40개월·총 360개월`:
  - D_m = min(360, 36) = **36**. R_m = 36 − 40 = **−4 ≤ 0** → **면제 분기(STEP 1)**. Fee = **0원**.
  - "대출 실행 후 3년(36개월)이 지나 중도상환수수료가 면제됩니다" 상태 노출.

> **반올림 정책(검증 전제)**: 본 앵커는 **원 미만 절사(floor)** 기준으로 계산했다. 실제 은행은 반올림·일할 계산·최소 수수료 하한 등으로 수십~수백 원 차이가 날 수 있다(§8-4 라운딩 확정 요청). 개발팀은 이 표를 "산식이 맞는지 확인하는 검증용"으로 쓰고, 라운딩 정책 확정 후 회귀 테스트 값으로 고정할 것. **모든 앵커는 3년 캡·개월 근사·원 미만 절사 기준 — 마스터가 다른 라운딩/캡을 확정하면 Fee 재계산 필요.**

## 2. interpretation (결과 해석 텍스트, 5단 구조)

`lib/calculators.ts`의 `interpretation` 필드 초안. 1문단(결과 요약)은 §1-4 앵커B(1억원·0.65%·경과 12개월·주담대) 기준이며, 실제로는 **사용자 입력값에 따라 동적 생성**되어야 한다(기존 라이브 계산기와 동일 처리, §7 개발 전달). 면제(앵커D) 케이스는 별도 문안 분기 필요(§8-6).

```
중도상환금액 1억원, 수수료율 0.65%, 대출 실행 후 12개월이 지난 주택담보대출(총 약정기간 3년 초과)이라면, 면제기준기간은 3년(36개월)으로 적용되어 잔존기간은 24개월, 잔존비율은 24÷36 ≈ 66.7%이며, 예상 중도상환수수료는 약 433,333원입니다.

중도상환수수료는 '중도상환금액 × 수수료율 × (잔존일수 ÷ 대출기간일수)'로 계산하는 슬라이딩(체감) 방식입니다. 대출을 받은 뒤 시간이 지나 갚을수록 잔존비율이 줄어 수수료도 함께 줄어들고, 대출 실행 후 3년이 지나면 면제되는 것이 일반적입니다. 이때 계산식의 '대출기간'은 실제 만기가 30년이라도 은행 관행상 최대 3년(1,095일)으로 제한되므로, 만기가 3년을 넘는 대출은 분모를 3년으로 두고 계산합니다.

수수료율은 대출 종류·상품·약정 시점에 따라 다릅니다. 2025년 1월 이후 새로 체결된 대출은 수수료율이 크게 인하되어 주택담보대출은 대략 0.6~0.7%, 신용대출은 그보다 낮은 경우가 많지만, 그 이전에 받은 대출은 약정 당시 요율이 그대로 적용됩니다. 정확한 요율은 반드시 본인 대출 약정서나 은행에서 확인한 값을 입력하세요.

중도상환수수료율과 상품별 조건은 금융감독원 금융상품 통합비교공시(파인, fine.fss.or.kr)와 은행연합회 소비자포털 공시에서 확인할 수 있습니다.

입력하신 값은 브라우저 내에서만 계산에 사용되며 서버로 전송·저장되지 않습니다. 본 계산 결과는 대출 상담이나 자문이 아닌 단순 참고용 수치이며, 실제 수수료는 은행별 계산 방식(일할 계산·최소 수수료 등)과 약정 조건에 따라 달라질 수 있으니 정확한 금액은 반드시 대출 약정서와 해당 금융기관에서 확인하시기 바랍니다.
```

- 글자 수: 약 740자(공백 포함, 500~800자 충족).
- 5단 구조 매핑: ①결과요약(동적, 앵커B) → ②계산기준/공식·슬라이딩·3년 캡·면제 → ③실생활 팁(수수료율 시점·신규/기존 계약 차이, 본인 약정 요율 입력) → ④공식 출처(파인·은행연합회) → ⑤개인정보(브라우저 내)+면책(상담 아님, 은행별 방식·약정 차이).
- **SEO 키워드 자연 포함**: "중도상환수수료"·"중도상환수수료 계산"·"슬라이딩(체감)"·"3년 면제"·"주택담보대출"·"대출 갈아타기(암시)" 자연 배치.

## 6. 확정한 모델 요약 (개발팀 즉시 참조용)

| 항목 | 확정값 |
|---|---|
| 입력 | 중도상환금액 A(원, 필수) / 수수료율 f(%, 필수, default 0.7) / 경과기간 E_m(개월, 필수) / 총 약정기간 T_m(개월, 선택, 기본 36) |
| 분모(면제기준) | D_m = **min(T_m, 36)** — 3년(36개월) 캡 자동 적용 |
| 잔존기간 | R_m = D_m − E_m (≤0이면 면제) |
| 잔존비율 | ratio = R_m / D_m |
| 중도상환수수료 | **Fee = floor(A × f × R_m / (100 × D_m))** — 원 미만 절사 |
| 면제 분기 | E_m ≥ D_m(3년 또는 만기 경과) → Fee = 0, 면제 상태 노출 |
| 수수료율 | **사용자 입력이 최종**(default 0.7%는 힌트, 상수 분리) |
| 검증 앵커 | A 37,500 · B 433,333(대표) · C 200,000 · D 0(면제) (모두 3년 캡·개월 근사·원 미만 절사 기준) |
| YMYL | 중간. 요율 시점·신규/기존 계약·은행별 방식·개월 근사 면책 |

## 7. 디자인팀에 넘길 확정 요약 (화면 설계용 — 목록 불일치 핑퐁 방지)

단위변환·연봉·전기요금 때 planning/design 목록 불일치가 재발하지 않도록, 아래 입력·출력 목록을 **그대로** 화면 설계에 사용한다. §1-0·§6과 1:1 일치.

### 7-1. 입력 필드 (4개, 순서 고정)

1. **중도상환금액** — number(원, 천단위 콤마), 필수. placeholder `100,000,000`, 보조문구 "이번에 미리 갚을 원금"
2. **중도상환수수료율** — number(%, 소수 2자리), 필수, 기본 0.7. placeholder `0.65`, 보조문구 "본인 대출 약정서·은행 공시 요율을 입력하세요"
3. **대출 실행 후 경과기간** — number(개월), 필수. placeholder `12`, 보조문구 "대출받은 뒤 지금까지 지난 개월 수"
4. **대출 총 약정기간** — number(개월), 선택, 기본 36. placeholder `36`, 보조문구 "3년(36개월)을 넘으면 자동으로 3년으로 계산됩니다"

### 7-2. 출력 값 (결과 카드)

- **대표값(히어로)**: 예상 중도상환수수료 `N원`(천단위 콤마) — "참고용 예상 금액" 배지/각주. **면제 시**: 금액 대신 "중도상환수수료 면제 대상(대출 실행 후 3년 경과 등)" 문구 노출.
- **보조값(계산 근거 3~4행)**: (a) 적용 수수료율 f% / (b) 면제기준기간 D_m개월(3년 캡 여부 표기) / (c) 잔존기간 R_m개월 ÷ D_m개월 = 잔존비율 % / (d) 중도상환금액 A × 수수료율 × 잔존비율 = 수수료
- **슬라이딩 시각화(선택)**: 경과/잔존을 3년(36개월) 막대에 표시하면 "언제 갚아야 수수료가 얼마나 주는지"를 직관적으로 전달(체류시간·재방문 기여). 구현 여부는 디자인 판단.

### 7-3. 디자인 협의 필요(참고)

- 상단 근사·범위 고지 배너(YMYL-중간, "안내" 톤): 대출이자 계산기 배너 선례처럼 amber `role="note"`, 항상 노출. 카피 예: "은행 표준 방식(3년 캡·슬라이딩)에 따른 예상 중도상환수수료입니다. 수수료율·계산 방식은 은행·약정별로 다르며 실제 청구액과 차이가 있을 수 있습니다." (최종 카피는 디자인 스펙 회신 후 기획팀 확정)
- 보조값(a~d) 접이식(`<details>`) 여부: 대표값 + 계산 근거 3~4행 항상 노출 검토. 확정은 디자인 스펙 수령 후.
- **오인 유발 UI 지양(대출이자 계산기 원칙 승계)**: "지금 갈아타기", "대환대출 신청", "최저금리 조회" 등 실제 신청으로 오인될 CTA·배너 금지. 계산 버튼은 "계산하기"류 중립 문구만.

## 8. 마스터 확인 필요 결정사항 (기획팀 단독 결정 범위 밖)

### 8-1. [마스터] slug 확정 — `loan-prepayment-fee` 승인
- 라인업 개념 URL은 `/loan/prepayment-fee`이나 코드 slug는 같은 카테고리 파트너 `loan-interest-calculator`의 `loan-` 접두 관행에 맞춰 **`loan-prepayment-fee`** 제안. 확정 승인 요망(개발 등록 전 확정 필요).

### 8-2. [마스터/개발] 입력필드 확정 — 경과기간 방식 vs 날짜 방식, 엣지 처리
- **기획팀 채택안**: 입력 4개(중도상환금액/수수료율/**경과기간(개월)**/총 약정기간). 과제 제안 ③"잔존기간"을 **경과기간**으로 바꿨다(§1-0-1 1번: 잔존 직접입력은 만기 vs 3년 혼동 오입력 유발).
- **확인 요청**: (a) 경과기간(개월) 방식 승인, (b) 대안 — 대출 실행일·상환일 **날짜 입력**으로 정밀화(#7 근속연수 date 패턴 재사용, 일할 정확)할지, (c) 최소 수수료 하한·일할 계산 등 엣지를 v1에서 계속 제외할지.

### 8-3. [마스터, 우선] 3년 캡 처리 확정
- **기획팀 채택안 = 3년(36개월) 캡 적용을 기본**(분모 = min(총기간, 36), 3년 경과 시 면제). 은행권 표준 관행(§0·§참고).
- **확인 요청**: 3년 캡·면제를 기본 로직으로 확정 승인 요망. 상품별 예외(캡이 다르거나 수수료 없는 상품)는 면책으로 처리. 만약 캡을 사용자 선택(토글)로 노출해야 한다면 §7 입력 UI 변경 필요.

### 8-4. [개발] 수수료율 default값·라운딩 방식 확정
- **default 0.7%**(2025 인하 후 주담대~신용 중간 시작값) 승인 요망 — 사용자 입력이 최종이므로 힌트 성격. 상수 분리(§8-5).
- **라운딩**: §1-4 앵커는 **원 미만 절사(floor)** 기준. 은행 실제는 반올림·일할·최소 수수료로 달라질 수 있음. 최종 라운딩 정책 확정 후 앵커를 회귀 테스트 값으로 고정 요청.

### 8-5. [개발] 수수료 상수 모듈 분리 + 갱신 거버넌스
- 3년 캡(1,095일)·수수료율 default·(참고)요율대를 **상수 모듈로 분리**하고, 제도 개편(요율 인하·폐지 논의 지속) 시 갱신 체크리스트를 운영 워크플로에 추가 제안(전기요금 §8-5·연봉 §7-4와 동일 취지). 중도상환수수료는 정책 민감도가 높아 갱신 주기 관리 필요.

### 8-6. [개발] 결과 요약 동적 생성 + 면제 케이스 분기
- §2 1문단(앵커B 예시)은 정적 텍스트가 아니라 사용자 입력값 기반 **동적 생성**(기존 라이브 계산기와 동일). **면제(잔존 ≤ 0) 케이스는 별도 문안**("대출 실행 후 3년이 지나 중도상환수수료가 면제됩니다")으로 분기 필요. FAQ 개인정보 6번째 문항 추가 여부도 함께 판단.

### 8-7. [마스터/개발] relatedSlugs 확정 + 역방향 링크
- **기획팀 권고**: `loan-prepayment-fee`에 `relatedSlugs = ["loan-interest-calculator", "salary-net-calculator", "age-calculator"]` override(§4). 자동 로직도 대출이자를 1번에 두지만 2·3번 관련성이 약해 override 권고.
- **확인 요청**: (a) override 승인, (b) 대출이자 계산기(`loan-interest-calculator`)에도 `relatedSlugs` 지정해 양방향 순환 순서 고정할지, (c) DSR 계산기(로드맵 #12) 라이브 시 loan 3계산기 재조정 방향.

### 8-8. [디자인] §7-3 배너 카피·접이식·슬라이딩 게이지·면제 상태 표시
- 상단 고지 배너 최종 카피, 보조값 접이식 여부, 3년 슬라이딩 시각화 채택 여부, 면제 케이스 결과 카드 표시 방식은 디자인 스펙 수령 후 기획팀 확정 회신.

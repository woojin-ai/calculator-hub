## 페이지 본문 (사용자 노출용)

### H1
고객센터

### 인트로 (1~2문단)

계산기 허브를 이용해 주셔서 감사합니다. 계산기 이용 중 궁금한 점은 아래 자주 묻는 질문에서 먼저 확인해 보시기 바랍니다. 원하는 답을 찾지 못하셨다면 하단 "질문하기"를 통해 언제든 문의해 주세요.

사이트는 별도의 상담원 채팅이나 자동 응답 시스템 없이, 담당자가 이메일로 직접 확인하고 답변드리는 방식으로 운영되고 있습니다. 문의량에 따라 답변까지 다소 시간이 걸릴 수 있는 점 양해 부탁드립니다.

### 자주 묻는 질문 (FAQ)

**Q1. 계산 결과를 100% 신뢰할 수 있나요?**
A. 계산기 허브의 모든 계산기는 참고용 도구입니다. 입력하신 값을 기준으로 결과를 계산해 보여드리지만, 세율·이자율·건강 기준 등은 시점이나 개인 상황에 따라 달라질 수 있습니다. 법적·재무적·의료적 의사결정을 내리기 전에는 반드시 관련 기관의 공식 안내나 전문가(세무사, 노무사, 금융기관, 의료진 등)의 확인을 거치시기 바랍니다.

**Q2. 계산 결과가 틀린 것 같습니다. 어떻게 알려야 하나요?**
A. 아래 "질문하기"를 통해 (1) 사용하신 계산기 이름, (2) 입력하신 값, (3) 화면에 표시된 결과를 함께 알려주시면 빠르게 확인 후 조치하겠습니다.

**Q3. 회원가입이 필요한가요?**
A. 아니요. 계산기 허브는 회원가입 없이 누구나 무료로 이용하실 수 있습니다.

**Q4. 계산기에 입력한 정보(생년월일, 소득, 대출 정보 등)가 서버에 저장되나요?**
A. 아니요. 계산기에 입력하시는 값은 이용자의 브라우저 안에서만 계산되며 사이트 서버로 전송되거나 저장되지 않습니다. "질문하기"를 통해 문의하실 때 남기시는 이름·이메일·문의 내용만 답변을 위해 이메일로 전달됩니다. 자세한 내용은 개인정보처리방침 페이지에서 확인하실 수 있습니다.

**Q5. 광고가 너무 많이 보이거나 불편한 광고가 있어요.**
A. 이용에 불편을 드려 죄송합니다. 사이트는 Google AdSense를 통해 광고를 게재하고 있으며, 무료 서비스 운영을 위한 재원으로 사용됩니다. 특정 광고에 문제가 있다면 "질문하기"로 알려주시면 확인해 보겠습니다.

**Q6. 지금 이용할 수 있는 계산기는 어떤 것들이 있나요?**
A. 현재 만 나이 계산기, D-Day 계산기, BMI 계산기, 대출이자 계산기 4종을 바로 이용하실 수 있습니다. 급여·생활 카테고리 계산기는 순차적으로 추가되고 있으며, 새로 추가되는 계산기는 각 카테고리 페이지에서 확인하실 수 있습니다.

**Q7. 새로운 계산기를 추가해 줄 수 있나요?**
A. 좋은 제안 감사합니다. "질문하기"로 원하시는 계산기 종류를 알려주시면 검토 후 반영을 고려하겠습니다.

**Q8. 오류나 버그를 신고했는데 언제 반영되나요?**
A. 신고해 주신 내용은 담당자에게 전달되며, 확인 및 수정에는 사안에 따라 시간이 걸릴 수 있습니다. 반영 완료 시 별도로 안내드리지는 않으니, 궁금하시면 다시 문의해 주시면 진행 상황을 확인해 드리겠습니다.

**Q9. 모바일에서도 사용할 수 있나요?**
A. 네, 모바일과 데스크톱 모두에서 이용하실 수 있도록 반응형으로 제공되고 있습니다.

### 질문하기

원하시는 답을 찾지 못하셨다면 아래 내용을 알려주세요.

- **이름(또는 닉네임)**
- **회신 받으실 이메일 주소**
- **문의 내용** (계산 오류를 제보하실 경우 계산기 이름, 입력하신 값, 화면에 표시된 결과를 함께 적어주시면 확인이 훨씬 빨라집니다)

아래 "질문하기" 버튼을 누르면 위 내용을 담아 문의하실 수 있습니다.

[질문하기] → 서버(Resend API) 경유로 담당자 수신함으로 이메일 전송 (2026-07-14 개발팀 갱신: 최초 mailto 기반에서 서버 전송 방식으로 전환, 아래 "다른 팀에 전달 필요 사항" 1번 참고)

## 다른 팀에 전달 필요 사항

1. **[개발팀] "질문하기" 구현 방식 — mailto 기반 (2026-07-14 갱신: 서버 전송 방식으로 전환 완료)**: 최초 작성 시점에는 별도의 문의 폼 서버/CS 인박스 시스템이 없어 `mailto:` 링크(개인 이메일 주소로 직접 열림) 방식으로 구현하는 것을 전제로 작성했었다. 이후 개인 이메일 주소가 사이트/소스에 노출되지 않아야 한다는 방침에 따라, 2026-07-14 개발팀이 `app/api/contact` 서버 라우트(Resend API 경유)로 전환했다. 사용자는 더 이상 메일 앱을 거치지 않고, 폼 제출 시 서버가 담당자 수신함으로 이메일을 발송하며 화면에는 접수 성공/실패 상태만 표시된다.
2. **[개발팀] `/about` 페이지 문의 링크 수정**: 위 배경 설명대로, `planning/about-page-content.md`의 "문의" 섹션(*"주: 아래 다른 팀 전달 사항 1번 참고 — `/contact` 페이지가 아직 없어..."*로 표시된 부분)은 이제 `/contact`가 아니라 `/support`로 링크하면 된다. 별도로 `/about` 문서를 다시 쓸 필요 없이, 개발팀이 페이지 구현 시 `/support`로 연결해 주면 된다.
3. **[디자인팀] 헤더 내비게이션 반영**: `components/SiteHeader.tsx`의 `NAV_ITEMS` 배열 끝에 `{ href: "/support", label: "고객센터" }` 추가가 필요하다(마스터 결정: 급여/대출/날짜/생활/블로그 다음 위치). 코드 수정은 개발팀 몫이나, 배치 순서는 마스터 결정 사항이므로 명시해 둔다.
4. **[디자인팀] FAQ 표시 형태**: FAQ 9개는 아코디언(접고 펼치기) 형태를 권장하나, 단순 리스트 형태도 무방하다 — UI 형태는 디자인팀 판단 영역.
5. **[개발팀] 응답 소요 시간 미고지**: `cs/static-pages-checklist.md`는 `/contact`(현재는 `/support`로 대체) 항목에 "응답 소요 시간 안내(예: 영업일 기준 n일 이내)"를 포함하도록 권고하지만, 실제 운영 인력/SLA가 아직 확정되지 않아 이번 초안에는 구체적 일수를 넣지 않았다(근거 없는 수치를 임의로 넣지 않기 위함). 향후 응답 기준일이 정해지면 인트로 문단에 한 문장 추가를 제안한다.
6. **[개발팀/디자인팀] 푸터 링크 추가 제안**: `/privacy`, `/terms`와 마찬가지로 `/support`도 완성되는 대로 `components/SiteFooter.tsx`에 링크 추가를 제안한다(현재 푸터에는 "사이트 소개"만 있음). 다만 헤더 내비게이션에 이미 "고객센터"가 들어가므로 푸터 추가는 필수는 아니며 최종 판단은 디자인팀에 맡긴다.

- **정확성 리서치 핵심 발견(중요)**: 과제 지침이 예시로 준 **전력산업기반기금 3.7%는 현재 유효 요율이 아니다.** 정부 부담금 정비로 3.7%→**3.2%**(2024.7~2025.6)→**2.7%**(2025.7~)로 단계 인하되어, **2026-07 현재 유효 요율은 2.7%**다(전기신문·정책브리핑 교차확인, §참고). 본 기획안은 **2.7%로 확정**하되, 요율 변동성이 큰 항목이므로 §8-4에서 마스터 최종 확인을 요청한다.

## 1. 계산 모델 (개발팀 구현용 — 수치 확정 명세)

### 1-0. 입력 항목 정의 (디자인·개발 공통 확정)

**최소 스펙 원칙**: 사용자가 가장 쉽게 아는 값(월 사용량 kWh) 하나를 필수로 하고, 계절만 보조로 둔다. 한전 실제 계산기가 "약정/할인/가족구성" 등을 물어 복잡하다는 불만이 라인업 §1(99행)의 대체 수요 근거였으므로, **우리는 반대로 초단순 입력**으로 차별화한다.

| 입력 항목 | 타입 | 필수 | 기본값 | 비고 |
|---|---|---|---|---|
| 월 사용량 | number(kWh) | **필수** | — | 최우선 입력. 검침 사용량(kWh). placeholder 예: `350` |
| 계절 | select | 선택 | **현재월 자동** | `하계(7·8월)` / `기타계절(그 외)`. 기본값 = 오늘이 7·8월이면 하계, 아니면 기타계절. 누진구간 경계가 달라 반드시 반영(§1-2) |

- **입력 검증(확정)**: 월 사용량 `≥ 0`, 음수 불가. 상한은 두지 않되(고사용 세대 존재) 아주 큰 값(예: 10,000kWh 초과) 입력 시 "일반 가정 범위를 벗어남" 안내 권장(엣지, 디자인 협의).
- **입력 2개로 고정**: 단위변환·연봉·퇴직금에서 planning/design 목록 불일치 핑퐁이 있었던 전례에 따라, 입력은 위 **2개로 확정**하고 §7 디자인 요약과 1:1 일치시킨다. 아래 §1-0-1의 3개 결정(저압 고정 / 계절 기본값 / 복지할인 제외)에 따라 저압·복지할인 관련 필드는 v1에 두지 않는다.

#### 1-0-1. 왜 이 입력 구성인가 (단순함 vs 정확도 트레이드오프 — 근거 기록)

과제가 판단을 위임한 3개 결정에 대한 기획팀 채택안과 근거다(최종 승인은 §8 마스터 확인).

1. **주택용 저압 "고정" 채택**(저압/고압 선택 필드 없음) — 단독주택·다세대 등 개별 계약 세대는 대부분 **주택용(저압)**이다. 아파트 등 고압 일괄계약은 요금표가 다르지만(저압보다 단가가 낮음), 관리비에 통합 청구되고 세대별 검침 방식·구간이 관리사무소마다 달라 사용자가 정확히 입력하기 어렵다. #8 퇴직금이 통상임금을 "정확 입력이 어려워 제외"한 것과 동일 논리로, v1은 **저압 고정**하고 "아파트 일괄계약(고압)은 실제 청구와 다를 수 있음"을 면책에 명시한다. 고압 선택은 별도 태스크로 분리(§8-2).
2. **계절 = 현재월 자동 기본값 + 사용자 변경 가능 채택** — 하계(7·8월)와 기타계절은 누진구간 경계가 달라(§1-2) 계절을 반드시 반영해야 한다. 사용자가 대부분 "이번 달 요금"을 보러 오므로 **오늘 날짜의 월로 기본값**을 자동 결정하되(7·8월→하계, 그 외→기타계절), "여름 요금이 얼마나 나올지 미리 보고 싶다"는 예측 수요를 위해 **셀렉트로 변경 가능**하게 둔다. 자동 고정만 하면 6월에 여름 요금을 못 본다.
3. **대가족·생명유지장치 등 복지할인 = 범위 밖(제외) 채택** — 대가족(3자녀/5인 이상)·출산가구·생명유지장치·사회적배려층 할인은 **자격 판정이 필요**하고 대상이 소수라, 넣으면 입력이 복잡해지고 오입력 시 오히려 오정보가 된다. #5 연봉 계산기가 복지·특수공제를 최소화한 판단과 동일하게, v1은 **할인 미반영(정상 요금 기준)**으로 두고 "복지할인 대상은 실제 청구액이 더 적을 수 있음"을 면책에 명시한다(§1-3). 이 단순화는 "실제보다 요금을 높게 보여주는" 방향이라 사용자가 "생각보다 덜 나왔네"가 되는 안전한 오차다.

### 1-1. 계산 파이프라인 (단계별 중간값 = 결과 카드에 표시)

```
입력: 월 사용량 U(kWh), 계절 season ∈ {하계, 기타계절}

STEP 0  누진구간 경계 결정
          하계(7·8월):   b1 = 300, b2 = 450
          기타계절:      b1 = 200, b2 = 400
STEP 1  도달 누진단계 판정 → 기본요금 Base
          U ≤ b1          → 1단계, Base = 910
          b1 < U ≤ b2     → 2단계, Base = 1,600
          U > b2          → 3단계, Base = 7,300
STEP 2  전력량요금 Energy = 구간별 사용량 × 구간단가 합산
          q1 = min(U, b1)                        → × 120.0
          q2 = clamp(U − b1, 0, b2 − b1)         → × 214.6
          q3 = max(U − b2, 0)                    → × 307.3
          Energy = floor( q1×120.0 + q2×214.6 + q3×307.3 )   // 원단위 미만 절사
STEP 3  기후환경요금 Climate = U × 9
STEP 4  연료비조정요금 Fuel  = U × 5
STEP 5  전기요금계 Subtotal = floor( Base + Energy + Climate + Fuel )   // 원단위 미만 절사
STEP 6  부가가치세 VAT       = round( Subtotal × 0.10 )                 // 원단위 4사5입
STEP 7  전력산업기반기금 Fund = floor10( Subtotal × 0.027 )             // 10원 미만 절사, 요율 2.7%
STEP 8  최종 청구요금 Total  = floor10( Subtotal + VAT + Fund )         // 10원 미만 절사
```

- **파이프라인 의존성**: STEP 6·7은 모두 STEP 5의 `Subtotal`(전기요금계)을 base로 곱한다. 부가세·기금을 "전력량요금만"이나 "기본+전력량만"에 곱하지 않도록 주의 — **기후환경·연료비조정을 포함한 전기요금계 전체**에 부과한다.
- **누진 = "도달 단계 기본요금 1회 + 전력량요금은 구간별 분할 단가"**: 흔한 오해가 "3단계면 전량에 307.3원"인데, 실제는 구간을 쪼개 낮은 단가부터 채운다(STEP 2). 기본요금만 도달 최종단계 금액이 1회 부과(STEP 1).
- **결과 카드에는** 최소 (최종 청구요금 Total) + (기본요금 Base) + (전력량요금 Energy) + (기후환경요금 Climate) + (연료비조정요금 Fuel) + (전기요금계 Subtotal) + (부가세 VAT) + (전력기금 Fund) + (적용 누진단계·계절)을 표시. 중간값 노출 정도는 디자인 협의(§7-3).
- `floor10(x)` = 10원 미만 절사(`Math.floor(x/10)*10`). `round`는 4사5입.

- **필수사용량 보장공제(2024.7 폐지)·복지할인**은 v1 범위 밖(§1-0-1·1-3).

### 1-3. 단순화한 부분 (근사·한계 — 결과해석·FAQ·면책에 반영)

v1에서 실제 청구서 대비 단순화한 항목. 모두 YMYL-낮음~중간에 맞춰 고지한다.

1. **복지할인·대가족할인 미반영**: 대가족·출산·생명유지장치·사회적배려층 할인을 넣지 않는다. → **대상자는 실제 청구액이 계산값보다 적을 수 있음** 고지(해석·FAQ 4).
2. **주택용 저압 고정(고압/아파트 일괄계약 미반영)**: 아파트 등 고압 일괄계약 세대는 요금표가 달라 결과가 실제 관리비 청구와 다를 수 있음 고지(FAQ 3).
3. **검침 주기·월할 미반영**: 실제 청구는 검침일 기준(달력 월과 불일치)·일할 계산·전월 미납 등이 섞인다. v1은 "월 사용량 1개월분을 온전히 한 계절 요금표로 계산"한다고 가정. → 검침 주기가 두 계절에 걸치면 실제와 근사임을 고지(FAQ 5).
4. **연료비조정·기후환경·부담금 요율 시점 의존**: 조사 시점 요율이며 분기·연 개정 시 달라짐(§1-2). → "요금표 개정 시 결과가 달라질 수 있음" 면책.

### 1-4. 검증용 기준값 (QA·마스터가 독립 재현할 앵커) — 기획팀 직접 손계산

**손계산 근거(독립 재현용):**

- **앵커 A** `200kWh, 기타계절`:
  - 단계: 200 ≤ 200 → 1단계. Base = 910.
  - Energy: q1=200×120.0=24,000, q2=0, q3=0 → **24,000**.
  - Climate=200×9=1,800. Fuel=200×5=1,000.
  - Subtotal=910+24,000+1,800+1,000=**27,710**.
  - VAT=round(27,710×0.10)=2,771. Fund=floor10(27,710×0.027)=floor10(748.17)=**740**.
  - Total=floor10(27,710+2,771+740)=floor10(31,221)=**31,220원**.
- **앵커 B(대표)** `350kWh, 기타계절`:
  - 단계: 200 < 350 ≤ 400 → 2단계. Base = 1,600.
  - Energy: q1=200×120.0=24,000, q2=(350−200)=150×214.6=32,190, q3=0 → 24,000+32,190=**56,190**.
  - Climate=350×9=3,150. Fuel=350×5=1,750.
  - Subtotal=1,600+56,190+3,150+1,750=**62,690**.
  - VAT=round(62,690×0.10)=6,269. Fund=floor10(62,690×0.027)=floor10(1,692.63)=**1,690**.
  - Total=floor10(62,690+6,269+1,690)=floor10(70,649)=**70,640원**.
- **앵커 C** `500kWh, 하계(7·8월)`:
  - 경계 300/450. 단계: 500 > 450 → 3단계. Base = 7,300.
  - Energy: q1=300×120.0=36,000, q2=(450−300)=150×214.6=32,190, q3=(500−450)=50×307.3=15,365 → 36,000+32,190+15,365=**83,555**.
  - Climate=500×9=4,500. Fuel=500×5=2,500.
  - Subtotal=7,300+83,555+4,500+2,500=**97,855**.
  - VAT=round(97,855×0.10)=round(9,785.5)=9,786. Fund=floor10(97,855×0.027)=floor10(2,642.085)=**2,640**.
  - Total=floor10(97,855+9,786+2,640)=floor10(110,281)=**110,280원**.
  - **하계 완화 효과(해설·FAQ용)**: 같은 500kWh를 기타계절(경계 200/400)로 계산하면 전력량요금이 200×120+200×214.6+100×307.3=**97,650원**(하계 83,555 대비 +14,095원)이 된다. 7·8월 구간 완화로 여름 요금 부담이 줄어드는 구조를 이 대비로 설명한다.
- **앵커 D** `400kWh, 겨울(기타계절)`:
  - 경계 200/400. 단계: 400 ≤ 400 → 2단계(경계값은 하위단계 포함). Base = 1,600.
  - Energy: q1=200×120.0=24,000, q2=(400−200)=200×214.6=42,920, q3=0 → 24,000+42,920=**66,920**.
  - Climate=400×9=3,600. Fuel=400×5=2,000.
  - Subtotal=1,600+66,920+3,600+2,000=**74,120**.
  - VAT=round(74,120×0.10)=7,412. Fund=floor10(74,120×0.027)=floor10(2,001.24)=**2,000**.
  - Total=floor10(74,120+7,412+2,000)=floor10(83,532)=**83,530원**.
  - **비교 참고**: 같은 400kWh라도 하계라면 경계 300/450이라 2단계이되 전력량요금이 300×120+100×214.6=**57,460원**(겨울 66,920 대비 −9,460원). 겨울 400kWh가 여름 400kWh보다 비싸다 — 계절 경계 차이를 보여주는 앵커.

> **반올림 정책(검증 전제)**: 본 앵커는 (전기요금계·전력량요금 = 원 미만 절사) / (부가세 = 원 미만 4사5입) / (전력기금 = 10원 미만 절사) / (최종 청구 = 10원 미만 절사) 기준으로 계산했다. 한전 사이버지점 실제 청구서는 항목별 절사 순서가 미세하게 다를 수 있어 수십 원 차이가 날 수 있다(§8-6 라운딩 확정 요청). 개발팀은 이 표를 "산식이 맞는지 확인하는 검증용"으로 쓰고, 라운딩 정책 확정 후 회귀 테스트 값으로 고정할 것. **모든 앵커는 부담금 2.7% 기준 — 만약 마스터가 3.2%로 확정하면 Fund·Total 재계산 필요.**

## 2. interpretation (결과 해석 텍스트, 5단 구조)

`lib/calculators.ts`의 `interpretation` 필드 초안. 1문단(결과 요약)은 §1-4 앵커B(350kWh, 기타계절) 기준이며, 실제로는 **사용자 입력값에 따라 동적 생성**되어야 한다(기존 라이브 계산기와 동일 처리, §7 개발 전달).

```

이 전기요금 계산기는 한국전력공사 주택용 저압 요금표를 기준으로 계산합니다. 주택용 전기요금은 사용량이 많을수록 높은 단가가 붙는 3단계 누진제로, 기본요금은 도달한 최종 단계 금액이 한 번 부과되고 전력량요금은 각 구간 사용량에 구간별 단가(1단계 120원, 2단계 214.6원, 3단계 307.3원)를 곱해 합산합니다. 7월과 8월(하계)에는 냉방 부담을 줄이기 위해 누진구간 경계가 200·400kWh에서 300·450kWh로 완화됩니다. 여기에 기후환경요금(kWh당 9원)과 연료비조정요금(kWh당 5원)을 더한 전기요금계에 부가가치세 10%와 전력산업기반기금 2.7%가 붙어 최종 청구요금이 됩니다.

정확한 요금과 최신 단가는 한국전력공사 사이버지점(cyber.kepco.co.kr)에서 확인할 수 있으며, 요금표·부담금 요율은 매년 또는 분기마다 개정될 수 있습니다.

입력하신 사용량은 브라우저 내에서만 계산에 사용되며 서버로 전송·저장되지 않습니다. 본 계산 결과는 참고용이며, 대가족·복지할인이나 아파트 일괄계약(고압), 검침 주기 등에 따라 실제 청구액과 차이가 있을 수 있습니다.
```

- 글자 수: 약 750자(공백 포함, 500~1000자 충족).
- 5단 구조 매핑: ①결과요약(동적, 앵커B) → ②계산기준/누진 3단계·기본요금·구간단가·하계완화·부가세·기금 → ③실생활 팁(누진 경계 관리, 하계 구간 활용) → ④공식 출처(한전 사이버지점, 개정 가능성) → ⑤개인정보(브라우저 내)+면책(복지할인·고압·검침주기 차이).

- **등장 수치(1,600 / 56,190 / 3,150 / 1,750 / 62,690 / 6,269 / 1,690 / 70,640)는 §1-4 앵커B와 일치**(임의 수치 금지 원칙). 부담금 2.7%도 본문 명기.

## 4. 관련 계산기 링크 추천 (자동 결과가 이미 이상적 — override 불필요)

- `getRelatedCalculators()`는 `relatedSlugs` override가 없으면 "같은 카테고리 우선 후 나머지, 배열 순서"(`[...sameCategory, ...rest].slice(0,3)`)로 채운다.
- 이 계산기가 live 전환되면 `life` 카테고리 live는 `bmi-calculator`·`unit-converter`·(신규)electricity-bill 3개다. self 제외 시 sameCategory=[**BMI, 단위변환**] 2개, rest는 배열 순서상 맨 앞 `age-calculator`(만 나이)가 첫 번째다.
- **자동 결과 = 단위 변환 + BMI + 만 나이** → 라인업 §3-3 원칙(같은 카테고리 2 + 다른 카테고리 1)을 **정확히 충족**한다. #5 연봉·#8 퇴직금과 달리 **override가 필요 없다**(salary 카테고리는 same-category live가 부족해 override가 필요했지만, life는 이미 2개가 채워짐).
- **기획팀 권고: `relatedSlugs` 미지정(자동 로직 사용).** 굳이 순서를 고정하고 싶다면 아래 문장형 앵커 권장(라인업 §3-3):
  - 단위 변환 계산기: "전기 사용 공간이 궁금하다면 평↔㎡ 단위 변환 계산기도 함께 확인해보세요."
  - BMI 계산기: "생활에 필요한 다른 계산기로 BMI 계산기도 이용해보세요."
  - 만 나이 계산기: "정확한 만 나이가 필요하다면 만 나이 계산기를 이용해보세요."
- (선택) 향후 `/life`에 "가전제품 전기요금(소비전력 W·사용시간 기반) 계산기"가 추가되면 이 전기요금 계산기와 세트 상호링크가 자연스럽다 — 로드맵 확장 아이디어로만 남긴다(§8 별도 제안).

## 7. 디자인팀에 넘길 확정 요약 (화면 설계용 — 목록 불일치 핑퐁 방지)

단위변환·연봉 때 planning/design 목록 불일치가 재발하지 않도록, 아래 입력·출력 목록을 **그대로** 화면 설계에 사용한다. §1-0·§6과 1:1 일치.

### 7-1. 입력 필드 (2개, 순서 고정)

1. **월 사용량** — number(kWh, 정수 권장), 필수. placeholder `350`, 보조문구 "한 달 사용한 전력량(kWh)"
2. **계절** — select, 선택, 기본값=현재월 자동. 옵션 `하계(7·8월)` / `기타계절(그 외)`. 보조문구 "여름(7·8월)은 누진구간이 완화됩니다"

### 7-2. 출력 값 (결과 카드)

- **대표값(히어로)**: 최종 청구요금 `N원`(천단위 콤마) — "참고용 예상 청구액" 배지/각주
- **보조값(항목 분해)**: (a) 기본요금 / (b) 전력량요금 / (c) 기후환경요금 / (d) 연료비조정요금 / (e) 전기요금계 / (f) 부가가치세 / (g) 전력산업기반기금 / (h) 적용 누진단계·계절(예: "기타계절 2단계")
- **누진단계 시각화(선택)**: 사용량이 어느 구간에 있는지 막대/게이지로 표시하면 "누진 경계까지 얼마 남았나"를 직관적으로 전달(체류시간·재방문 기여). 구현 여부는 디자인 판단.

### 7-3. 디자인 협의 필요(참고)

- 상단 근사·범위 고지 배너(YMYL-낮음~중간, "안내" 톤): 연봉/퇴직금 배너 선례처럼 amber `role="note"`, 항상 노출. 카피 예: "한전 주택용 저압 요금표 기준의 예상 청구액입니다. 복지할인·아파트 일괄계약(고압)·검침 주기 등에 따라 실제 청구액과 다를 수 있습니다." (최종 카피는 디자인 스펙 회신 후 기획팀 확정)
- 보조값(a~h) 접이식(`<details>`) 여부: 연봉 계산기(3-tier)처럼 대표값 + 핵심 3행(전기요금계/부가세+기금/최종) 항상 노출 + 세부 접힘 검토 가능. 확정은 디자인 스펙 수령 후.
- 계절 기본값이 자동 설정되면 "현재 7월 기준 하계 요금으로 계산됨" 같은 상태 문구를 계절 셀렉트 옆에 노출 권장(사용자 혼동 방지).
- 절전 상품·특정 요금제 가입 유도 등 상업적 CTA 지양(공공요금 계산기 신뢰도).

## 8. 마스터 확인 필요 결정사항 (기획팀 단독 결정 범위 밖)

### 8-1. [마스터] slug 확정 — `electricity-bill-calculator` 승인
- 라인업 개념 URL은 `/life/electricity-bill`이나 코드 slug 관행(`-calculator` 접미)에 맞춰 **`electricity-bill-calculator`** 제안. 확정 승인 요망(개발 등록 전 확정 필요).

### 8-2. [마스터/개발] 범위 단순화 3건 승인 — 저압 고정 / 슈퍼유저 제외 / 복지할인 제외
- **기획팀 채택안**: (1) 주택용 저압 고정(고압/아파트 일괄계약 미지원), (2) 슈퍼유저(월 1,000kWh 초과) 별도 단가 미구현(3단계 누진까지), (3) 대가족·복지할인 미반영. 근거 §1-0-1·§1-3.
- **확인 요청**: 위 단순화 승인 요망. 만약 고압/아파트 수요를 v1에 넣어야 한다면 고압 요금표 리서치 + 입력 필드(저압/고압 선택) 추가로 별도 태스크 분리가 필요하다.

### 8-3. [마스터/개발, 우선] 요금표 공식 출처 최종 대조
- **§1-2 상수는 블로그·정보사이트 교차확인값**이다. 한전 공식 요금표(home.kepco.co.kr 요금표 PDF, 또는 공공데이터포털 "주택용 전기요금표")로 **기본요금·전력량 단가·누진경계를 최종 대조**한 뒤 상수를 확정할 것. (WebFetch로 한전 공식 페이지 조회 시 구버전(2013) 캐시가 반환되어 현행표 직접 확인이 불발됨 — 개발 착수 전 PDF 원본 대조 필수.)

### 8-4. [마스터, 우선] 전력산업기반기금 요율 — **2.7% 확정** 재확인
- **핵심 정정**: 과제 지침 예시(3.7%)는 현재 유효 요율이 아니다. 3.7%→3.2%(2024.7~2025.6)→**2.7%(2025.7~)**로 인하됨(전기신문·정책브리핑 교차확인). **기획팀 채택안 = 2.7%**, 전 앵커(§1-4)를 2.7%로 계산했다.
- **확인 요청**: 2.7% 확정 승인 요망. 만약 마스터가 별도 근거로 3.2% 등 다른 값을 지정하면 §1-4 Fund·Total을 재계산해야 한다. 요율은 상수 모듈로 분리(§8-5).

### 8-5. [개발] 요금 상수 모듈 분리 + 갱신 거버넌스
- 기본요금·전력량 단가·누진경계·기후환경요금·**연료비조정요금(분기 변동)**·전력기금 요율을 **상수 모듈로 분리**하고, 분기(연료비조정)·연(요금표·기금) 갱신 체크리스트를 운영 워크플로에 추가 제안(연봉 계산기 §7-4와 동일 취지). 특히 연료비조정요금은 분기 경계마다 재고시되므로 갱신 주기가 짧다.

### 8-6. [개발] 원단위/10원단위 라운딩 정책 확정
- §1-4 검증값은 (전기요금계·전력량 = 원미만 절사) / (부가세 = 원미만 4사5입) / (전력기금 = 10원미만 절사) / (최종 = 10원미만 절사) 기준. 한전 사이버지점 실제 청구서와 대조 시 항목별 절사 순서 차이로 수십 원 오차 가능. 최종 라운딩 정책 확정 후 앵커를 회귀 테스트 값으로 고정 요청.

### 8-7. [개발] 결과 요약 문구 동적 생성 + 계절 자동 기본값
- §2 1문단(앵커B 예시)은 정적 텍스트가 아니라 사용자 입력값 기반 **동적 생성**(기존 라이브 계산기와 동일). 계절 미선택 시 서버/클라이언트 현재월로 기본값 결정(7·8월→하계). SSG/ISR 환경에서 "현재월" 판정 위치(빌드 타임 vs 클라이언트)는 개발 판단 — 클라이언트에서 결정 권장(빌드 시점 고정 방지).

### 8-8. [디자인] §7-3 배너 카피·접이식·누진 게이지·계절 상태문구
- 상단 고지 배너 최종 카피, 보조값 접이식 여부, 누진단계 시각화(게이지) 채택 여부, 계절 자동설정 상태문구는 디자인 스펙 수령 후 기획팀 확정 회신.

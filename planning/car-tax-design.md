## 0. 자동차세 법정수치 단일 소스 (개발 필수 전제)

마스터가 웹검색으로 확정한 **2026년 비영업용 승용차 기준** 상수. `lib/interest-tax.ts`·`lib/minimum-wage.ts` 선례처럼 이 수치는 **신규 `lib/car-tax.ts` 1곳에서만 관리**하고 화면·엔진 어디에도 하드코딩하지 않는다. 매년 개정 시 이 파일 1곳만 바꾸면 전 화면이 갱신되도록 한다.

| 구분 | 값 | 비고 |
|------|-----|------|
| cc당 세액 — 1,000cc 이하 | **80원/cc** | 구간 flat(누진 아님). 전체 배기량 × 해당 구간 세율 |
| cc당 세액 — 1,600cc 이하 | **140원/cc** | 상동 |
| cc당 세액 — 1,600cc 초과 | **200원/cc** | 상동 |
| 지방교육세율 | **30% (0.30)** | 차령경감 적용 후 본세 × 30% |
| 차령 경감 시작 | **등록 3년차부터** | 차령 = 2026 − 최초등록연도(연 단위 근사) |
| 차령 경감률 | **매년 5%씩, 최대 50%** | 경감률 = 차령≥3이면 `min(50, (차령−2)×5)%`, 아니면 0% |
| 전기·수소차 본세(비영업 승용) | **정액 100,000원** | 배기량·차령경감 무관. 교육세 30% 별도 |
| 연납(1월 선납) 할인율 | **3% (0.03)** | 2026년 기준. 본세(경감 후)에만 적용, 교육세는 할인 대상 아님 |
| 기준연도 | **2026** | 차령 산정 기준연도 |

```ts
export type CarKind = "combustion" | "eco"; // 일반 승용(내연) / 전기·수소차

export const CAR_TAX_BASE_YEAR = 2026;

/** cc당 세액 구간표 (구간 flat — 전체 배기량 × 해당 구간 세율, 누진 아님). */
export const CC_TAX_BRACKETS = [
  { maxCc: 1000, wonPerCc: 80 },   // 1,000cc 이하 80원/cc
  { maxCc: 1600, wonPerCc: 140 },  // 1,600cc 이하 140원/cc
  { maxCc: Infinity, wonPerCc: 200 }, // 1,600cc 초과 200원/cc
] as const;

export const EDUCATION_TAX_RATE = 0.30;      // 지방교육세 = 본세(경감후) × 30%
export const ECO_CAR_BASE_TAX = 100_000;     // 전기·수소차 본세 정액(비영업 승용)
export const PREPAY_DISCOUNT_RATE = 0.03;    // 1월 연납 할인율(2026)
export const AGE_RELIEF_START_YEAR = 3;      // 등록 3년차부터 경감
export const AGE_RELIEF_PER_YEAR = 0.05;     // 매년 5%씩
export const AGE_RELIEF_MAX = 0.50;          // 최대 50%
```

- 화면에서 이 상수를 참조하는 지점: ① 배기량 힌트의 구간표 안내 ② 결과 Tier② "차령 경감률" 병기 ③ 결과 Tier③ "연납 할인(3%)" 라벨 ④ 하단 출처 병기. 세율/할인율 숫자를 화면 문자열에 직접 박지 말고 상수에서 조합한다(`${(PREPAY_DISCOUNT_RATE*100)}%` 등).
- **범위 제약(v1 면책)**: 비영업용 **승용차**만 다룬다. 영업용·승합·화물·이륜 등 기타 차종은 세율 체계가 달라 v1 범위 밖 → §5 면책·안내 문구로 명시(게이트 카드가 아니라 배너/문구로).

## 1. 개요 · 사용자 목표

- **사용자 검색 의도**: "내 차 자동차세 얼마?", "연납하면 얼마 할인?", "차 오래 타면 세금 줄어드나?" — 배기량·차령·연납 조합이 번거로워 "간편 계산" 대체 수요(lineup §15). 계절성 3스파이크(정기분 6·12월, 연납 신청 1월).
- **사용자 목표**: ① 연 자동차세 총액(교육세 포함)을 한 번에 확인 ② 정기분(6·12월) 회차별 납부액과 1월 연납 시 할인 후 납부액을 비교 ③ 차령 경감이 얼마나 적용됐는지 확인.
- **핵심 인터랙션**: 입력 최소화(차종·배기량·등록연도 3개 이내) → 버튼 → 즉시 3티어 결과. 납부방식 토글 없이 정기분/연납을 **결과에 함께** 보여줘 체류 중 비교가 일어나게 한다(마스터 결정 §2-1).
- 페이지 골격은 1차 12개·#13·#14와 동일한 계산기 상세 템플릿(제목 → 입력폼 → 결과 → interpretation → FAQ → 관련계산기 → 면책). 신규 브랜딩 요소 없음.

### 2-1. 차종 세그먼트 탭 (일반 승용 / 전기·수소차) — 폼 최상단, 탭 성격

- 옵션 2개: **`일반 승용`**(내연, 기본값) / **`전기·수소차`**. 그룹 라벨(legend): `차종`.
- 헬퍼 1줄(`text-xs text-brand-text-secondary`, 고정 노출): `일반 승용(내연기관)은 배기량 기준, 전기·수소차는 정액으로 과세됩니다.`
- **동작**: 전기·수소차 선택 시 (a) **배기량 필드를 숨김**(DOM에서 제거 또는 `hidden`) (b) 등록연도 필드는 유지하되 경감 미적용 안내로 힌트 스위치 (c) **이전 결과 카드 초기화**(차종 바꾸면 값이 어긋나므로 재계산 유도). 등록연도 입력값은 유지.

### 2-2. 차량 정보 필드

| # | 필드 | 폭 | 라벨 | placeholder | 힌트 | 필수 | 노출 조건 |
|---|------|-----|------|-------------|------|------|-----------|
| 1 | 배기량 | 전체 폭 | `배기량 (cc)` | `1,999` | `자동차등록증의 총배기량입니다. 1,000cc 이하 80원, 1,600cc 이하 140원, 1,600cc 초과 200원/cc가 적용됩니다.` | 필수 | **내연일 때만** |

- **입력 방식**: 배기량은 `type="text" inputMode="numeric"` + 천단위 콤마(`formatAmountInput` 재사용, #14 L20~24 그대로). 등록연도는 `type="number" inputMode="numeric" step="1"`(콤마 없이 4자리). 클래스 `${INPUT_BASE} border-brand-border`(에러 시 `border-brand-warning`).
- **방어(마스터 결정 §4)**: 배기량 = 미입력/0 이하/비정수 에러(`배기량을 cc 단위 정수로 입력해주세요.`). 등록연도 = 미입력/범위밖(1900 미만 또는 2026 초과)/비정수 에러(`최초등록연도를 1900~2026 사이로 입력해주세요.`). 필드 하단 `text-xs text-brand-warning` + `role="alert"`, 입력에 `aria-invalid`. 상한 배기량(비현실적 대배기량) 클램프 정책은 엔진 사안(§9).
- **미래연도 방어**: 등록연도 > 2026(기준연도)이면 차령이 음수가 되므로 방어. 2026 등록(차령 0)·2024~2025 등록(차령 1~2)은 경감률 0%로 정상 계산.
- **게이트 카드 없음**(마스터 §4): 유효 입력이면 항상 정상 수치 결과. #13 주휴수당의 "미발생" 회색 게이트 패턴은 차용하지 않는다. 범위밖 차종(영업용/기타)은 게이트가 아니라 §5 배너·문구로 안내.

## 3. 계산 로직 (개발팀 인계용 — 그대로 `lib/car-tax.ts` 엔진으로 이관)

`lib/savings-interest.ts`의 판별 유니온 방어·라운딩 정합 패턴을 계승한다. 내부는 전 정밀도, 최종 표시값만 `Math.round`(원단위). Tier 간 행이 서로 맞아떨어지도록 파생값은 반올림된 표시값에서 도출한다.

### 3-1. 입력 → 결과 산출 순서

```
입력: kind(combustion|eco), cc(내연만), registerYear
─────────────────────────────
[A] 본세(경감 전) baseTaxRaw
    · combustion: cc × (구간 매칭된 wonPerCc)
        - cc ≤ 1000 → 80원/cc
        - 1000 < cc ≤ 1600 → 140원/cc
        - cc > 1600 → 200원/cc
        (구간 flat: 전체 배기량에 해당 구간 단가를 곱함. 누진 아님)
    · eco: ECO_CAR_BASE_TAX (100,000) 고정
[B] 차령·경감률
    · carAge = CAR_TAX_BASE_YEAR(2026) − registerYear
    · reliefRate =
        - combustion & carAge ≥ 3 → min(0.50, (carAge − 2) × 0.05)
        - 그 외(내연 차령<3, 또는 eco) → 0
[C] 본세(경감 후) baseTax = round(baseTaxRaw × (1 − reliefRate))
[D] 지방교육세 educationTax = round(baseTax × 0.30)
[E] 연세액 총액 annualTotal = baseTax + educationTax           ← Tier①
[F] 정기분 회차액 semiAnnual = round(annualTotal / 2)          ← Tier③ (6월·12월 각)
[G] 연납 할인액 prepayDiscount = round(baseTax × 0.03)          ← 본세(경감후)에만
[H] 연납 납부액 prepayTotal = annualTotal − prepayDiscount      ← Tier③
```

- **라운딩 순서 주의**: baseTax를 먼저 반올림(정수화)한 뒤 그 값으로 교육세·할인액·연세액을 파생시켜 Tier 간 합이 정확히 맞게 한다(savings-interest.ts와 동일 원칙). 실제 지자체 부과는 10원 단위 절사 등 세부 라운딩이 있을 수 있으나 v1은 원단위 반올림으로 근사(면책 명시).
- **정기분 합 ≠ 연세액 주의**: `semiAnnual × 2`가 반올림 때문에 annualTotal과 1원 어긋날 수 있음. 표시는 `annualTotal`을 진실값으로 두고 정기분은 `round(annualTotal/2)`로 각 회차 표기(면책 톤 불필요, 흔한 관례). QA 대조 시 annualTotal 기준.
- **엔진 반환 인터페이스(권장)**: `{ ok:true, input, amounts:{ baseTaxRaw, reliefRate, carAge, baseTax, educationTax, annualTotal, semiAnnual, prepayDiscount, prepayTotal } }` / `{ ok:false, error }`. error 종류: `invalid-cc`, `invalid-year`.

### 3-2. 워크드 예시 (QA 대조용 — 대표 케이스)

**케이스 A: 2,000cc · 2020년 등록 · 비영업 승용 내연**

| 단계 | 계산 | 값 |
|------|------|-----|
| [A] 본세(경감 전) | 2000cc > 1600 → 200원/cc × 2000 | **400,000원** |
| [B] 차령 / 경감률 | 2026 − 2020 = 6년 → min(50, (6−2)×5)% | 6년 / **20%** |
| [C] 본세(경감 후) | 400,000 × (1 − 0.20) | **320,000원** |
| [D] 지방교육세 | 320,000 × 0.30 | **96,000원** |
| [E] 연세액 총액(Tier①) | 320,000 + 96,000 | **416,000원** |
| [F] 정기분 각 회차(6·12월) | 416,000 ÷ 2 | **208,000원** |
| [G] 연납 할인액 | 320,000 × 0.03 | **9,600원** |
| [H] 연납 납부액 | 416,000 − 9,600 | **406,400원** |

**케이스 B(검증 보조): 전기·수소차 · 2023년 등록**
- 본세 = 100,000(정액, 경감 없음) / 교육세 = 30,000 / 연세액 = **130,000원** / 정기분 각 65,000원 / 연납 할인 = 100,000 × 0.03 = 3,000 / 연납 납부 = **127,000원**.

**케이스 C(경감 상한 검증): 1,600cc · 2010년 등록 내연**
- 본세(경감 전) = 1600 × 140 = 224,000(1600cc는 "1,600cc 이하" 구간 → 140원/cc). 차령 = 16년 → min(50, 14×5)=50% 상한. 본세(경감 후) = 112,000 / 교육세 = 33,600 / 연세액 = **145,600원**.

> 경계값 주의: **1,600cc 정확히**는 "1,600cc 이하" 구간(140원/cc). 1,601cc부터 200원/cc. `maxCc` 비교는 `cc <= maxCc`로 구현(CC_TAX_BRACKETS 순차 매칭).

### Tier ① 헤드라인 — 연 자동차세 총액(지방교육세 포함)

**동적 요약 문구**(`text-sm text-brand-text-secondary`, 입력값 반영·정적 텍스트 금지)
- 내연 예시(케이스 A): `배기량 2,000cc(1,600cc 초과 200원/cc), 최초등록 2020년(차령 6년, 20% 경감) 기준 연 자동차세는 지방교육세 포함 약 416,000원입니다. 1월 연납 시 9,600원 할인되어 406,400원입니다.`

- 차종·구간·경감률·연납액이 문장에 반영되도록 개발이 조합(하드코딩 금지, §0 상수에서 문자열 조합).

### Tier ② 세부 내역 (AmountRow/TextRow, #14 L78~98 재사용, 계산 흐름 순)

1. `자동차세 본세` = baseTax(차령경감 적용 후) — AmountRow
2. `차령 경감률` = TextRow. 내연 & 경감>0이면 `{경감률}% (차령 {n}년)`, 경감 0이면 `해당 없음 (차령 {n}년)`, 전기차면 `해당 없음 (정액 과세)`

4. `연 자동차세 합계` = annualTotal — AmountRow (Tier①과 동일 금액, 확인용·항상 노출)

### Tier ③ 납부 시나리오 (Tier② 아래, 두 번째 구분선으로 분리)

납부방식 토글 없이 정기분/연납을 **나란히** 보여준다(마스터 §2-1). 소제목(`text-xs font-semibold text-brand-text-secondary mt-4`): `납부 방법별 금액`.

1. `정기분 6월` = semiAnnual — AmountRow
2. `정기분 12월` = semiAnnual — AmountRow

- **Tier① 헤드라인 숫자**: `text-4xl`(약 36px), 자동차세 6자리 이하라 375px 내부(약 343px)에 여유. `flex-wrap items-baseline`으로 `원` 자연 줄바꿈 유지(대배기량 폴백 대비).

- **광고 자리**: 결과-해석 사이 광고는 모바일 반응형(`display:block`, 폭 100%). 결과 카드와 광고 사이 `mt` 여백 확보해 대표값을 가리지 않게.

## 8. interpretation / FAQ 방향성 (디자인 권고 — 최종 카피는 기획/개발이 `calculators.ts`에 확정)

### 8-1. interpretation 초안 (결과 해석 텍스트, 상한 1000자 엄수)

> 자동차세는 소유한 차의 배기량(cc)에 cc당 세액을 곱해 산정하는 지방세입니다. 비영업용 승용차는 1,000cc 이하 80원, 1,600cc 이하 140원, 1,600cc 초과 200원을 배기량 전체에 곱하며(구간별 정액, 누진 아님), 여기에 지방교육세 30%가 더해져 실제 납부액이 됩니다. 전기·수소차는 배기량이 없어 본세가 10만원으로 정액 과세됩니다.
>
> 차를 오래 타면 세금이 줄어듭니다. 최초등록 3년차부터 매년 5%씩, 최대 50%까지 자동차세 본세가 경감되기 때문입니다. 예컨대 2,000cc 차량을 6년째 보유하면 경감률이 20%가 되어, 본세가 40만원에서 32만원으로 낮아집니다. 단 전기·수소차는 정액 과세라 차령 경감이 적용되지 않습니다.
>
> 자동차세는 원래 6월과 12월에 절반씩 나눠 내지만, 1월에 1년치를 미리 내는 연납을 신청하면 할인을 받습니다. 2026년 연납 할인율은 본세 기준 3%이며, 신청 시기가 늦어질수록 이미 지난 기간만큼 할인 폭이 줄어듭니다. 정확한 부과액과 연납 신청은 위택스에서 확인하세요.

(길이: 공백 포함 약 560자 — 상한 1000자 이내 충족. 3개 소주제: ① 세액 산정 구조 ② 차령 경감 ③ 연납. 특정 차종·업체 추천 없음, 세율은 §0 흐름과 일치.)

### 8-2. FAQ 4개 초안

1. **`자동차세는 어떻게 계산되나요?`** → 배기량 × cc당 세액(80/140/200원 구간) + 지방교육세 30%. 구간 정액(누진 아님)이라는 점, 전기·수소차는 정액 10만원임을 병기.
2. **`차가 오래되면 자동차세가 줄어드나요?`** → 등록 3년차부터 매년 5%씩 최대 50% 경감. 차령 = 기준연도(2026) − 최초등록연도로 근사한다는 점, 전기차는 경감 없음.
3. **`자동차세 연납하면 얼마나 할인되나요?`** → 1월 신청 시 2026년 기준 본세의 3% 할인(교육세는 할인 대상 아님). 신청 시기가 늦으면 미경과 기간만큼 할인이 줄어든다는 점(면책 톤).
4. **`정기분은 언제 내나요? 이 계산기는 어떤 차까지 되나요?`** → 정기분은 6월·12월 두 번. 이 계산기는 비영업용 승용차만 지원하며 영업용·승합·화물은 세율 체계가 달라 위택스 확인 권장.

- FAQ 렌더는 기존 계산기 FAQ 컴포넌트 관례 그대로 — 디자인 신규 요소 없음.

### 9-2. 핑퐁 / 확인요청

**개발팀 인계 이슈**
1. **차종 탭 2번째 라벨 375px 실측**: `전기·수소차`(6자)가 375px 세그먼트 절반 폭에서 말줄임 없이 들어가는지 구현 후 실측(§7). 빠듯하면 `전기·수소`(5자)로 축약 폴백 — 검수 때 디자인 재확인.
2. **배기량 상한/엣지 정책**: 비현실적 대배기량(예: 99,999cc)·0·소수 입력 클램프/에러 규칙은 엔진 스펙. 디자인은 필드 에러(§2-2)·미래연도 방어만 규정했다. 상한 정책 정해 회신 바람.
3. **정기분 반올림 1원 오차 처리**: `round(annualTotal/2) × 2 ≠ annualTotal` 가능성(§3-1). annualTotal을 진실값으로 두는 안으로 설계했으나, 지자체 실무처럼 정기분을 먼저 확정하고 합산하는 방식을 원하면 회신(표기 라벨 영향).

**기획팀 확인요청**
4. **연납 할인 기준(본세만 vs 총액)**: 마스터 결정대로 할인액 = 본세(경감후) × 3%(교육세 제외, 보수적)로 설계했다. 일부 계산기는 총액 기준으로 표기하기도 하나 마스터 결정을 우선 반영. 이견 시 회신.
5. **interpretation/FAQ 최종 카피**: §8은 디자인 초안(560자). `calculators.ts` 확정 시 세율·연도·차령 근사 면책 톤이 §0·§5와 일관되게 유지되는지만 확인 바람.

**마스터 확인**
6. 톤앤매너·카테고리(/life)·스택 변경 없음. 신규 브랜딩 요소 없이 기존 계산기 컨벤션 100% 계승이라 별도 승인 불요로 판단. 다만 §0 법정수치(cc구간 80/140/200·교육세 30%·차령경감 5%씩 최대 50%·전기차 10만원·연납 3%·기준연도 2026)를 마스터 웹검색 근거로 상수화하므로, **v1 범위를 "비영업용 승용차"로 한정**하고 영업용/기타차종을 면책 처리하는 결정(§0·§5)에 이견 없는지만 확인 부탁드림.

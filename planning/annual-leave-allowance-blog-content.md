## 마스터확인 N건 (개발·QA 교차검증 포인트) — 5건

1. **[QA] 총액 반올림 정밀도 일치** — 대표예시 본문에 "1,148,330(표시일급×10)이 아니라 1,148,325(정밀 총액)"임을 명시했다. 라이브 계산기가 중간값 미반올림·최종 1회 반올림으로 구현되었는지(#14/#15에서 지적됐던 문서-라이브 불일치) 재확인 필요. 계산기 기획안 "반올림 주의" 항목과 동일 이슈.

2. **[개발] append 위치·문법** — 이 객체는 `blogPosts` 배열의 마지막 원소(`car-tax-guide-2026`) 뒤에 콤마로 이어 붙이고, 기존 배열 닫는 `];`는 그대로 둔다. 앞 원소 끝의 `},` 유지 확인.

3. **[개발] BlogSection 타입 준수** — live 유니온(`heading`/`paragraph`/`list`/`callout`(info|warning)/`calculatorCta`)만 사용. 태스크 지시서에 있던 `tip` variant·heading level 필드는 **실제 코드 타입에 없어 미사용**. 타입 에러 없어야 함.

4. **[개발] slug live 여부** — `relatedCalculatorSlugs` 및 CTA 3종(`annual-leave-allowance-calculator`, `salary-net-calculator`, `four-insurance-calculator`) 전부 `calculators.ts`에 존재하는지 재확인. 존재하지 않는 slug 참조 시 내부링크·`getBlogPostsForCalculator` 역링크 깨짐. 특히 주CTA `annual-leave-allowance-calculator`가 역링크로 계산기 상세에 노출되는지 확인(0편→1편 커버 목적).

5. **[기획/승인] YMYL 면책 충분성** — 노무/임금 = 중간 YMYL. 하단 warning에 (참고용·법적효력 없음 / 통상임금 범위 회사규정·판례 의존 / 사용촉진 소멸 / 5인 미만 예외 / 1350·노무사) 전부 포함, 단정표현("무조건 받는다") 회피. 주휴수당 글 수준의 면책 톤 유지. 광고 배치는 승인 후 정책대로.

## 근거 메모

### 1. 주제·카테고리 선정 이유
- BMI는 키·몸무게만 있으면 누구나 궁금해하는 생활 밀착 건강 지표로 검색 수요가 꾸준하며, 이미 라이브인 `bmi-calculator`(life)의 유입 콘텐츠로 직결된다. 계산기와 동일한 `life` 카테고리로 묶어 카테고리 내부링크·색인 일관성을 유지했다.

### 2. relatedCalculatorSlugs 2개 live 확인 방법
- `lib/calculators.ts`를 Read해 확인함:
  - `bmi-calculator` (line 282~): `category: "life"`, `status: "live"`.
  - `unit-converter` (line 318~): `category: "life"`, `status: "live"`.
- 둘 다 live·life 카테고리이며, calculatorCta는 이 두 slug만 사용(대표 = bmi-calculator, 2차 = unit-converter). unit-converter는 해외 신장(피트·인치)·체중(파운드) → cm·kg 변환 연계로 자연스럽게 링크했다.

### 3. 사용한 검증수치 (마스터 확정값만 인용)
- 공식: BMI = 몸무게(kg) ÷ 키(m)². 표시값 소수 둘째 자리 반올림, 구간 판정은 rawBmi 기준.
- 대표 예시: 170cm·70kg → 24.22 / 과체중.
- 160cm·55kg → 21.48 / 정상체중.
- 175cm·80kg → 26.12 / 비만(1단계).
- 165cm·50kg → 18.37 / 저체중.
- 180cm·100kg → 30.86 / 비만(2단계).
- 172cm·68kg → rawBmi 22.9854 → 표시 22.99 / 정상체중 (rawBmi 판정 강점 예시).

- **가드레일 선언: 위 목록에 없는 키/몸무게 조합이나 새로 계산한 BMI 수치는 본문에 일절 넣지 않았음. WHO 예시로 언급한 "BMI 27"은 구간 경계 설명용 정성 서술이며 특정 키·몸무게 조합을 재계산해 만든 값이 아님(단순히 25~30 사이 임의 정수로 국내/WHO 판정 차이를 설명).** 만약 이 표현도 가드레일 밖으로 간주된다면 아래 확인 요청 참조.

### 4. 마스터 확인 필요 사항
1. **YMYL 발행 적정성(건강·최대강도)**: 건강 지표 주제라 애드센스 YMYL 리스크가 있음. 본문은 단정 표현("무조건 살 빼야"·"건강하다")을 피하고 완화 어조·의료진 상담 안내를 유지했으며, 맨 끝 warning callout에 최대강도 면책(진단 아님·한계·청소년/임산부 예외·공신력 자료 안내)을 넣었다. 그래도 발행 전 최종 검수·발행 여부는 마스터 승인 필요.
2. **WHO 설명의 "BMI 27" 표현**: 특정 키/몸무게에서 재계산한 값이 아니라 국내(≥25 비만)와 WHO(≥30 비만) 판정이 갈리는 지점을 보여주기 위한 25~30 사이 예시 정수임. 가드레일("목록 밖 수치 금지")을 엄격 적용해 이 표현도 빼길 원하면 해당 문장을 정성 서술("같은 값이라도 국내 기준은 비만, WHO 기준은 과체중이 될 수 있다")로 교체 가능 — 마스터 판단 요청.
3. **타입 유니온 준수**: callout은 `info`/`warning`만 사용(중간 info 1개 + 말미 warning 1개), heading에 level 필드 없음, calculatorCta는 bmi-calculator/unit-converter 2개 slug만 사용. 신규 필드·variant 없음. 개발팀 append 시 빌드 영향 없을 것으로 판단하나 최종 타입 체크는 개발팀 확인.
4. **calculatorCta label 사용 여부**: label은 선택 필드로, 기존 최근 글(car-tax)은 label 없이 slug만 넣는 패턴을 쓴다. 본 안은 CTA 문구를 명확히 하려고 label을 넣었으니, 사이트 표준을 "label 없이 slug만"으로 통일하려면 두 CTA의 label 필드를 삭제하면 된다(기능·타입 영향 없음).

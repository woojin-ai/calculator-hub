## 리서치 방법 및 한계 명시
네이버 데이터랩·네이버 키워드 도구의 정확한 월간 검색량·CPC 수치는 API 접근 없이는 실시간 확인이 불가능했다(WebSearch로 확인 시도했으나 구체적 수치 없음, 출처 하단 표기). 아래 검색량/CPC/경쟁도 등급은 (1) 상위 노출 사이트의 규모·수 (2) 계절성·뉴스 이슈 여부 (3) 금융/광고 카테고리 일반 CPC 시세에 대한 업계 통념을 근거로 한 **정성적 추정**이다. 실제 착수 전 네이버 검색광고 키워드도구 또는 Google Keyword Planner로 수치 검증을 권장하며, 이는 개발 착수를 막을 사안은 아니라 병행 가능.

참고한 웹 검색 출처:

## 1. 카테고리별 1차 출시 계산기 (총 12개, 카테고리당 3개)

각 항목: `slug 제안` / 검색량·CPC·경쟁도 판단 근거 / 카테고리 내 우선순위 / 애드센스 승인 리스크

### /salary (연봉·급여)

**1순위 — 연봉 실수령액 계산기**
- slug: `/salary/take-home-pay`
- 검색량: 매우 높음(연중 스테디, 1~2월 연봉협상 시즌 급증). CPC: 중~높음(보험·재테크·이직 광고주 타겟 키워드와 겹침). 경쟁도: 매우 높음 — 사람인·잡코리아·원티드 등 대형 채용 플랫폼이 이미 선점, 상위 노출 난이도 높음.
- 판단 근거: 검색 결과 상위에 사람인·잡코리아·데모데이 등 다수 경쟁 사이트 확인, 이미 레드오션이나 트래픽 총량 자체가 커서 반드시 보유해야 하는 필수 계산기.
- 애드센스 리스크: **중간(YMYL 인접)**. 세후 급여액은 세율·4대보험 요율 변경에 따라 매년 갱신 필요. "본 계산 결과는 참고용이며 실제 급여와 차이가 있을 수 있다"는 면책 문구 필수. 세율 하드코딩 후 방치 시 오정보 리스크.

**2순위 — 퇴직금 계산기**
- slug: `/salary/severance-pay`
- 검색량: 높음(이직·퇴사 시즌 계절성 강함). CPC: 중~높음. 경쟁도: 높음 — 고용노동부 공식 계산기(권위 도메인)가 존재해 검색 결과 상단을 차지하는 경우가 많음.
- 판단 근거: 고용노동부(moel.go.kr) 공식 도구와 경쟁해야 하므로 신규 사이트가 앞지르기 어려움. 다만 UX 개선(모바일 친화, 결과 해설 제공)으로 차별화 여지 있음.
- 애드센스 리스크: **중간(YMYL)**. 근로기준법 기반 계산이므로 법적 정확성 필요, 법 개정 시 업데이트 안 하면 오정보. "법률 자문 아님, 정확한 금액은 고용노동부/노무사 확인 권장" 문구 필수.

**3순위 — 4대보험료(국민연금·건강보험 등) 계산기**
- slug: `/salary/four-insurance`
- 검색량: 중간(매년 1월 요율 개정 시 스파이크). CPC: 중간. 경쟁도: 중간 — 대형 채용 사이트가 별도 페이지로 다루지 않는 경우가 많아 틈새 여지 있음.
- 판단 근거: 연봉 실수령액 계산기의 보조/세부 페이지로 내부링크 시너지 가능, 독립 키워드로도 검색 유입.
- 애드센스 리스크: **낮음~중간**. 매년 요율 갱신 관리 부담, 사회보험 정보라 부정확 시 신뢰도 하락.

### /loan (대출·이자)

**1순위 — 대출이자 계산기 (원리금균등/원금균등상환)**
- slug: `/loan/interest-calculator`
- 검색량: 매우 높음(부동산·대출 뉴스 연동 상시 검색). CPC: **매우 높음** — 금융/대출 카테고리는 애드센스 CPC 최상위권 중 하나(대출·보험 광고주 입찰 경쟁 치열).
- 판단 근거: 은행 앱, 대출 비교 플랫폼이 강세지만, "단순 계산만 원하는" 수요는 별도로 존재(은행 앱은 가입/실명인증 요구해 이탈 발생 → 계산기 단독 사이트 수요 있음).
- 애드센스 리스크: **높음(YMYL)**. 대출 관련 계산은 실제 금융 의사결정에 영향. "대출 상담/자문이 아닌 단순 참고용 계산 도구"임을 명확히 고지, 실제 금리·상환조건은 금융기관 확인 유도 문구 필수. 광고 배치 시 "지금 대출 신청" 류 오인 유발 디자인 지양.

**2순위 — DSR(총부채원리금상환비율) 계산기**
- slug: `/loan/dsr-calculator`
- 검색량: 중~높음(정부 대출 규제 발표 시 급증하는 이슈성 키워드, 뉴스성 트래픽). CPC: 높음. 경쟁도: 중간 — 대출이자 계산기보다 경쟁 사이트 수 적음.
- 판단 근거: 규제 정책 변경 뉴스와 함께 검색 스파이크 발생, 블로그(`/blog`) 자동생성 콘텐츠와 결합 시 트래픽 폭 확대 가능.
- 애드센스 리스크: **높음(YMYL)**. 대출 심사 기준(정책)에 따라 결과가 달라져 정책 변경 시 즉시 갱신 필요. "은행별 세부 기준 상이, 실제 심사 결과 아님" 명시 필수.

**3순위 — 중도상환수수료 계산기**
- slug: `/loan/prepayment-fee`
- 검색량: 중간(대출 상환·갈아타기 시 검색). CPC: 높음. 경쟁도: 낮음~중간 — 틈새 키워드로 대형 사이트가 세분화해 다루지 않는 경우 많음.
- 판단 근거: 대출이자 계산기 이용자의 다음 액션(중도상환 고민) 동선과 자연스럽게 연결, 관련 계산기 링크로 체류시간 확대 기대.
- 애드센스 리스크: **중간(YMYL)**. 금융기관별 수수료율 상이하므로 예시 수치임을 명확히 고지.

### /date (날짜·나이)

**1순위 — 만 나이 계산기 (개발팀 오늘 구현 최우선 추천 대상)**
- slug: `/date/age-calculator`
- 검색량: **매우 높음** — 2023년 만 나이 통일법 시행 이후 스테디한 고빈도 검색어(연초 특히 급증). CPC: 낮음~중간(생활정보성 키워드라 대출/금융 대비 CPC는 낮으나 검색량이 압도적으로 커서 총광고수익(RPM×트래픽) 기여도 높음). 경쟁도: 높음(포털 자체 계산기 제공)이나, 계산 로직이 단순(생년월일 입력 → 날짜 연산)해 신규 사이트도 UX·부가정보(만 나이/연 나이/띠 등)로 상위 노출 사례 다수.
- 판단 근거: 검색 결과에서 superkts.com 등 개인·중소 사이트도 상위 노출되는 것을 확인 — 대형 자본 없이도 진입 가능한 몇 안 되는 고검색량 키워드.
- 애드센스 리스크: **매우 낮음**. 개인정보 저장 없이 클라이언트 단 계산만 하면 되고, 의료·금융·법률 조언 요소가 없어 YMYL 해당 없음. **애드센스 승인 심사용 초기 콘텐츠로도 가장 안전.**
- **개발팀 참고: 구현 난이도 최저(순수 날짜 연산 로직), 리스크 최저, 검색량 최고 — 오늘 1개 구현 대상으로 최적.**

**2순위 — D-day 계산기**
- slug: `/date/d-day`
- 검색량: 높음(시험일·결혼기념일·수능 D-day 등 검색 상시 발생, 수능 시즌 등 계절성 스파이크). CPC: 낮음. 경쟁도: 높으나 구현 난이도 매우 낮음.
- 판단 근거: 만 나이 계산기와 유사한 성격(순수 유틸리티), 결과 페이지에서 "관련 계산기"로 만 나이 계산기와 상호 링크 시너지 큼.
- 애드센스 리스크: **없음**.

**3순위 — 근속연수·근무일수 계산기**
- slug: `/date/service-period`
- 검색량: 중간(연차·퇴직금 계산 전 단계로 검색). CPC: 낮음~중간. 경쟁도: 중간.
- 판단 근거: `/salary/severance-pay`(퇴직금 계산기) 이용 전 단계 계산으로 자연스러운 내부링크 도출, 카테고리 간(day→salary) 크로스 트래픽 유도 목적.
- 애드센스 리스크: **낮음**. 단, 연차 일수는 근로기준법 기준값이 있어 완전 무관하지는 않음 — 참고용 명시 권장.

### /life (생활)

**1순위 — BMI(체질량지수) 계산기**
- slug: `/life/bmi`
- 검색량: 높음(다이어트·건강검진 시즌 스파이크). CPC: 중간(건강기능식품·다이어트 광고 타겟과 겹쳐 낮지 않음). 경쟁도: 높음(공식이 표준화되어 있어 구현은 쉬우나 포화 키워드).
- 판단 근거: 공식이 단순(체중/신장²)해 즉시 구현 가능, 결과 해석 텍스트·FAQ로 콘텐츠 두께를 만들기 좋은 주제.
- 애드센스 리스크: **중간(YMYL, 건강 관련)**. "의료적 진단이 아니며 체성분·근육량 등을 반영하지 않는 참고 지표"라는 면책 문구 필수, 과체중/저체중 결과에 "의료진 상담 권장" 문구 권장. 다이어트/시술 관련 과장 문구 절대 금지(애드센스 건강 콘텐츠 정책 위반 소지).

**2순위 — 전기요금(누진세) 계산기**
- slug: `/life/electricity-bill`
- 검색량: 계절성 매우 강함(여름철 폭증, 누진세 이슈 뉴스와 연동). CPC: 중간. 경쟁도: 중간 — 한전 공식 계산기가 있으나 UX가 불편하다는 사용자 불만이 많아 대체 수요 존재.
- 판단 근거: 여름 시즌 트래픽 스파이크로 광고 노출 극대화 가능, 단 계절 편중이 커서 연중 안정적 트래픽원은 아님(우선순위 2위인 이유).
- 애드센스 리스크: **낮음~중간**. 요금제·누진구간이 매년 바뀔 수 있어 갱신 관리 필요, "실제 청구액과 차이가 있을 수 있음" 명시.

**3순위 — 단위 변환 계산기 (평↔㎡, 인치↔cm 등 통합형)**
- slug: `/life/unit-converter`
- 검색량: 꾸준함(부동산 면적 환산 수요 특히 높음). CPC: 낮음. 경쟁도: 낮음~중간. 구현 난이도: 매우 낮음(단순 곱셈 변환).
- 판단 근거: 리스크 없이 페이지 수를 빠르게 늘릴 수 있는 프로그래매틱 SEO 소재(평형별, 단위별 롱테일 페이지 대량 생성 가능 — 개발팀과 협의 필요 사안이므로 큰 구조 변경은 별도 확인 요망).
- 애드센스 리스크: **없음**.

**중요**: 애드센스 최초 승인 신청 전에는 YMYL 리스크가 있는 대출/퇴직금 계산기보다 만 나이·D-day·BMI 등 저리스크 페이지를 먼저 채워 "얇은 콘텐츠(thin content)" 지적을 피하는 것을 권장. 정책/재무 계산기는 승인 이후 순차 추가하는 편이 안전 — 이 순서 변경이 필요하다고 판단되면 승인 전략 자체는 큰 방향 결정이므로 마스터 팀장 확인 요망.

## 3. 계산기 페이지 공통 템플릿 — 텍스트 구성 가이드

### 3-1. 결과 해석 텍스트 (500~1000자) 섹션 구조

모든 계산기 페이지에 공통 적용할 5단 구조:

1. **결과 요약 한 줄** (계산기 UI 바로 아래, 1~2문장): 사용자가 입력한 값 기준 결과를 자연어로 풀어 설명. 예: "만 30세, 연 나이 31세입니다."
2. **계산 기준/공식 설명** (150~250자): 어떤 공식·법령·요율 기준으로 계산되었는지 명시. SEO 키워드(계산기명 반복) 자연스럽게 포함. 예: 퇴직금 계산기 → "근로기준법 제34조에 따라 평균임금×30일×(재직일수/365)로 산정됩니다."
3. **실생활 적용 팁/주의사항** (150~300자): 사용자가 이 결과를 어떻게 활용하면 좋은지, 흔한 오해나 예외 케이스 설명. 체류시간을 늘리는 핵심 구간.
4. **관련 제도/공식 출처 안내** (해당 시, 100~200자): YMYL 성격 계산기(퇴직금/대출/DSR/BMI/4대보험)는 정부기관·공식 출처 링크(예: 고용노동부, 국세청, 금융감독원) 병기 — E-E-A-T 신뢰도 확보 및 애드센스 심사 리스크 완화 목적.
5. **면책 문구** (50~100자, 전 계산기 공통 하단 고정): "본 계산 결과는 참고용이며 법적/의료적/재무적 효력이 없습니다. 정확한 사항은 관련 기관 또는 전문가와 상담하시기 바랍니다." — YMYL 등급이 "중간" 이상인 계산기는 필수, "없음/낮음" 등급도 권장 삽입(사이트 전체 신뢰도 통일성).

총 글자수 500~1000자 배분 예시: 1)50자 + 2)200자 + 3)250자 + 4)150자 + 5)80자 ≈ 730자.

### 3-2. FAQ 구성 — 계산기 유형 공통 질문 패턴 (3~5개)

모든 계산기에 아래 5가지 유형 중 계산기 성격에 맞는 3~5개를 선택해 구성:

1. **계산 기준 질문형**: "OO 계산기는 어떤 기준으로 계산되나요?" (공식/법령/요율 재설명, 본문과 중복되어도 무방 — FAQ 스니펫이 검색결과에 노출되는 경우가 많아 SEO 효과 큼)
2. **오차/예외 상황형**: "결과가 실제와 다를 수 있는 경우는?" (예: 연봉 계산기 → 성과급/비과세 항목 별도 처리 여부)
3. **관련 제도·용어 설명형**: "만 나이와 연 나이의 차이는?", "DSR과 DTI의 차이는?" — 초심자 대상 용어 설명, 체류시간·연관 검색어 유입 확대
4. **타 계산기와의 차이·연계형**: "퇴직금 계산기와 실수령액 계산기는 어떻게 다른가요?" — 관련 계산기 링크 섹션과 자연스럽게 연결, 내부 체류 페이지 수 증가
5. **개인정보·데이터 처리형**: "입력한 정보가 저장되나요?" — 신뢰도 확보용, 특히 급여·대출 등 민감정보 입력 계산기에는 필수 포함("모든 계산은 브라우저 내에서만 처리되며 서버에 저장되지 않습니다" 등 개발팀 확인 후 정확한 문구 기재 요망)

### 3-3. 관련 계산기 링크 배치 원칙

- **위치**: 결과 해석 텍스트와 FAQ 사이, 또는 FAQ 하단 — "함께 보면 좋은 계산기" 카드형 섹션으로 고정 배치(전 페이지 동일 위치 유지, 사용자 학습 용이성).
- **개수**: 3개 고정 권장(너무 많으면 클릭 분산으로 광고 뷰 체류 방해, 너무 적으면 내부링크 효과 저하).
- **구성 원칙**: 같은 카테고리 내 2개 + 다른 카테고리 1개(카테고리 간 크로스 트래픽 유도, 예: 퇴직금 계산기 → 근속연수 계산기(같은 흐름), 연봉 실수령액 계산기(같은 카테고리), 만 나이 계산기(타 카테고리, 일반 트래픽 유입 소스)).
- **앵커 텍스트**: "계산기 바로가기" 같은 범용 텍스트 대신 계산기명을 포함한 자연스러운 문장형 앵커 사용(예: "퇴직금과 함께 근속연수도 확인해보세요") — SEO 앵커 다양성 확보.
- **광고 배치와의 간섭 방지**: 관련 계산기 카드 섹션 바로 위/아래에 광고 유닛을 배치하는 것은 실수 클릭 유발 소지가 있어 지양(애드센스 정책상 콘텐츠와 명확히 분리된 광고 배치 권장) — 이 부분은 실제 광고 유닛 배치 시 디자인팀과 협의 필요.

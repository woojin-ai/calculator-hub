## 0. 계산 모델 확인 (개발 필수 전제 — 마스터 확정, 변경 금지)

화면 문구·결과 라벨이 아래 모델과 1:1로 대응해야 한다.

```
시간당 통상임금 = 월 통상임금 ÷ 209
1일 통상임금   = 시간당 통상임금 × 8
연차수당 총액   = 1일 통상임금 × 미사용 연차일수     ← 원 단위 반올림(최종 표시값만)
```

- **209** = 월 소정근로시간 나눗수. **8** = 1일 소정근로시간. 두 값 모두 **상수 1곳(`lib/annual-leave-allowance.ts`)에서만** 관리하고 화면은 그 상수를 참조한다(하드코딩 산재 금지).
- 보조 — 근속연수 기반 **발생 연차일수**(선택 입력 시에만):
  - 1년 미만: 1개월 개근당 1일, **최대 11일**
  - n년(≥1): `15 + floor((n − 1) ÷ 2)`, **최대 25일**
- **라운딩 정책(주휴수당 lib 원칙 계승)**: 내부 계산은 전 정밀도 유지(`hourlyRaw = 월통상임금/209`, `dailyRaw = hourlyRaw × 8`), **최종 표시값만** `Math.round`(원 단위 반올림). 총액도 `Math.round(dailyRaw × 미사용일수)`. 화면에 보여주는 "시간당/1일 통상임금"은 표시용으로만 반올림하고, 총액 계산에는 반올림 전 값을 쓴다(§6 인계 이슈 A 참조 — 이 부분은 확인 필요).

## 1. 전체 레이아웃 구조 (섹션 배치)

계산기 페이지 표준 템플릿을 따른다(위→아래):

```
[페이지 제목 H1 + 리드 문장]          ← 페이지 셸(계산기 상세 공통)

    - [미사용 연차일수] · [근속연수(선택)]  (2열)

    - Tier② 보조값 행(시간당/1일 통상임금, 미사용일수)
    - (선택) 발생 연차일수 안내 블록 — 근속연수 입력 시에만

⑤ FAQ 4개                            ← calculators.ts
```

## 2. 입력 필드 레이아웃 상세

폼 카드: `rounded-xl border border-brand-border bg-brand-surface p-4 sm:p-6`, 내부 `<form className="flex flex-col gap-4">`. 그룹 라벨은 선례와 동일 `text-xs font-semibold text-brand-text-secondary`.

### 필드 순서·폭·문구 초안

| # | 필드 | 폭 | 라벨 | 단위 | placeholder | 힌트 | 필수 |
|---|------|-----|------|------|-------------|------|------|
| 1 | 월 통상임금 | 전체 폭 | `월 통상임금 (원)` | 원 | `2,500,000` | `기본급 + 고정수당 등 매월 고정적으로 지급되는 통상임금을 입력하세요. 식대·교통비 등 실비변상적 금품은 통상임금에서 제외될 수 있습니다.` | 필수 |
| 2 | 미사용 연차일수 | 2열 좌 | `미사용 연차일수 (일)` | 일 | `5` | `아직 사용하지 않은 연차 일수입니다. 반차는 0.5일 단위로 입력할 수 있습니다.` | 필수 |
| 3 | 근속연수 | 2열 우 | `근속연수 (년, 선택)` | 년 | `3` | `입력하면 발생 연차일수(법정 기준)를 함께 안내합니다. 연차수당 금액 계산에는 사용되지 않습니다.` | 선택 |

- **입력 방식**
  - **월 통상임금**: `type="text" inputMode="numeric"` + 천단위 콤마 서식(주휴수당 `formatAmountInput` 재사용). 클래스 `${INPUT_BASE} border-brand-border`(에러 시 `border-brand-warning`).
  - **미사용 연차일수**: `type="text" inputMode="decimal"` + 소수 1개 허용 서식(주휴수당 `sanitizeDecimalInput` 재사용 — **0.5 반차 입력 필수 지원**). 콤마 불필요(정수·소수만).
  - **근속연수(선택)**: `type="text" inputMode="decimal"`. 소수 허용(§6 인계 이슈 B — 1년 미만/개월 처리 확인 필요). 비워 두면 발생일수 블록을 숨김.
- **근속연수 필드의 위상**: 핵심 공식(`1일 통상임금 × 미사용일수`)에 근속연수는 **들어가지 않는다**. "(선택)"·"금액 계산에는 사용되지 않습니다"를 라벨·힌트에 명시해 "입력했는데 금액이 안 변한다"는 CS 문의를 예방한다(주휴수당 §9 이슈1과 동일 방어 전략).
- **에러 처리**: 선례와 동일. 필드 하단 `text-xs text-brand-warning` + `role="alert"`, 입력에 `aria-invalid`.

### 2-2. 검증 규칙 (컴포넌트 1차 방어 → lib 판별 유니온 2차 방어)

| 필드 | 규칙 | 오류 문구(초안) |
|------|------|-----------------|
| 월 통상임금 | 필수 · 유한 · **> 0** | `월 통상임금을 올바르게 입력해주세요.` |
| 미사용 연차일수 | 필수 · 유한 · **≥ 0** (0 허용) | `미사용 연차일수를 0 이상으로 입력해주세요.` |
| 근속연수(선택) | 입력 시에만 · 유한 · **≥ 0** | `근속연수는 0 이상으로 입력해주세요.` |

- **미사용일수 0**은 오류가 아니다 → 계산은 성립하되 총액이 **0원**. §4 엣지케이스에서 중립 안내 처리(게이트/에러 아님).
- 상한(비현실적 값, 예: 미사용일수 366, 월 통상임금 10억)에 대한 클램프/경고 정책은 엔진 사안 → §6 인계 이슈 C로 개발에 회신 요청. 디자인은 상한을 강제하지 않고 자유 입력을 전제로 둔다.
- 근속연수는 **선택**이므로 비어 있으면 검증 대상에서 제외(값이 있을 때만 ≥0 검사). 비어 있어도 총액 계산·결과 노출은 정상 진행.

## 3. 계산 버튼 + 결과 카드

### 3-1. 계산 버튼

- 문구: **`연차수당 계산하기`**
- 스타일: 선례 동일 `h-12 w-full rounded-lg bg-brand-primary px-5 text-base font-semibold text-white transition-colors hover:bg-brand-primary-hover`.
- 버튼 아래 안내(선례 계승, 문구만 조정): `입력하신 통상임금·연차 정보는 브라우저 안에서만 계산되며 서버에 저장·전송되지 않습니다.` (`text-xs text-brand-text-secondary`)
- 계산 후 `resultRef.scrollIntoView({ behavior: "smooth", block: "nearest" })` — 모바일에서 결과를 놓치지 않도록(선례 관례).

### 3-2. 결과 카드 — 정상(미사용일수 > 0)

컨테이너: 선례 정상 결과와 동일 `mt-6 rounded-xl border-l-4 border-brand-accent bg-gradient-to-br from-emerald-50 to-white p-4 sm:p-6`.

**Tier ① 대표값 — 연차수당 총액**
- 상단 라벨: `미사용 연차수당` (`text-sm font-medium text-brand-text-secondary`)
- 큰 숫자: `text-4xl font-bold tabular-nums text-brand-accent sm:text-[2.5rem]`, `formatWon(총액)` + `원`(`text-base font-medium text-brand-text-secondary`), `flex flex-wrap items-baseline gap-2`(자리수 많을 때 `원` 자연 줄바꿈)
- 보조 설명: `세전 기준, 미사용 연차 {N}일에 대한 수당입니다.` (`text-xs text-brand-text-secondary`, {N}은 입력값 반영)

**동적 결과 요약 문구** (`text-sm text-brand-text-secondary`, 정적 텍스트 금지 — 입력값 반영)
- 예시(월 통상임금 2,500,000원 · 미사용 5일):
  `월 통상임금 2,500,000원 기준 시간당 통상임금은 약 11,962원, 1일 통상임금은 약 95,694원입니다. 미사용 연차 5일에 대한 연차수당은 약 478,469원입니다.`
- 소수 연차(예 미사용 3.5일)도 문구에 그대로 반영: `미사용 연차 3.5일 ...`.

**Tier ② 보조값 행** (AmountRow/TextRow — 선례 컴포넌트 그대로 재사용, 계산 흐름 순)
1. `시간당 통상임금` = `formatWon(round(월통상임금/209))` — AmountRow  → `... 원`
2. `1일 통상임금` = `formatWon(round(시간당통상임금raw × 8))` — AmountRow
3. `미사용 연차일수` = `{N}일` — TextRow (반차 시 `3.5일`)
4. `연차수당 총액` = 대표값과 동일 — AmountRow (요약 확인용, 항상 노출)

> `formatWon` 사용, `tabular-nums`. 금액 행 우측 `원`은 AmountRow가 붙임(선례 AmountRow 그대로 재사용).

### 3-3. (선택) 발생 연차일수 안내 블록 — 근속연수 입력 시에만

- **근속연수 미입력 시 이 블록은 렌더하지 않는다**(숨김). 입력 시에만 Tier② 아래 구분선(`border-t border-brand-border/60 pt-4 mt-4`) 후 노출.
- 블록 소제목(`text-sm font-medium text-brand-text-secondary`): `근속 {y}년 기준 발생 연차`
- TextRow: `발생 연차일수` = `{발생일수}일`
- 보조 설명(`text-xs text-brand-text-secondary`): 근속에 따른 법정 발생 기준 요약. 예:
  - 근속 ≥1년: `근속 3년은 근로기준법상 연 16일의 연차가 발생합니다(1년 15일 + 2년마다 1일 가산, 최대 25일).`
  - 근속 <1년: `1년 미만 근로자는 1개월 개근 시 1일씩, 최대 11일의 연차가 발생합니다.`
- **주의**: 이 블록은 "발생" 일수 안내일 뿐 **미사용일수와 무관**하다. "발생일수 = 미사용일수 = 수당"으로 오해하지 않도록, 블록 하단에 한 줄: `발생 연차일수는 참고용이며, 위 연차수당은 실제 미사용 일수 기준으로 계산됩니다.`

### 3-4. 결과 카드 — 미사용일수 0 (중립 안내, 게이트 아님)

- 미사용일수가 0이면 총액은 0원이 맞다. 대표값을 `0원`으로 크게 띄우기보다는 **중립 회색 안내 카드**로 처리해 "계산이 안 됐나?" 오해를 줄인다.
- 컨테이너: 주휴수당 게이트 카드 구조 차용 `mt-6 rounded-xl border border-brand-border bg-brand-surface p-4 sm:p-6`(그라데이션·accent 라인 없음, `ⓘ`는 `text-brand-text-secondary`).
- 제목(`text-base font-semibold text-brand-text`): `지급할 연차수당이 없습니다`
- 부제(`text-sm text-brand-text-secondary`): `미사용 연차일수가 0일이라 연차수당이 발생하지 않습니다.`
- 구분선 후 참고로 TextRow 노출: `시간당 통상임금`, `1일 통상임금` (통상임금은 입력됐으므로 계산값 안내 — 연차가 생기면 1일당 얼마인지 참고 가능)
- 하단 안내(`text-xs`): `미사용 연차가 있으면 위에 일수를 입력해 다시 계산해 보세요. 반차는 0.5일로 입력할 수 있습니다.`
- 근속연수를 함께 입력했다면 §3-3 발생 연차 블록은 이 중립 카드 안에서도 동일하게 노출(발생일수 안내는 미사용 0과 무관하게 유용).

- `ResultState` 판별 유니온 권장(주휴수당 패턴):
  - `{ kind: "paid"; info; amounts; leaveInfo? }` — 미사용일수 > 0
  - `{ kind: "zero"; info; leaveInfo? }` — 미사용일수 = 0
  - `null` — 미계산
  - `leaveInfo`는 근속연수 입력 시에만 채움(발생일수). 미입력이면 `undefined` → §3-3 블록 숨김.
- lib `calculateAnnualLeaveAllowance`는 `{ ok: false; error }` 방어 브랜치 포함(주휴수당 `Outcome` 스타일). 컴포넌트 1차 검증 통과 시 도달하지 않는 이중 방어.

## 5. 접근성 · 모바일(375px) 확인 포인트

- **라벨-입력 연결**: 모든 `<input>`에 `id` + `<label htmlFor>`. 에러 시 `aria-invalid`, 오류 문구 `role="alert"`.
- **포커스 링**: 입력은 `INPUT_BASE`에 포함된 `focus:ring-4 focus:ring-brand-primary/15`. 버튼 등 커스텀 인터랙션이 생기면 `FOCUS_RING_*`(`@/lib/focusRing`) 사용. 본 계산기는 세그먼트 토글이 없어 추가 radiogroup 접근성 요구는 없음(선례 대비 단순).

- **2열 필드(미사용일수 · 근속연수)**: 둘 다 짧은 숫자라 375px에서도 `grid grid-cols-2 gap-3 sm:gap-4` 유지 자연스러움. 단 라벨 `근속연수 (년, 선택)`이 2줄로 접힐 수 있음 → 좌/우 라벨에 `min-h-10`을 주어 입력창 baseline 정렬(주휴수당 §6 관례). 세로가 과하게 길어지면 **폴백: 근속연수를 전체 폭 단독 행으로 하단 배치**(선택·계산 미참여라 우선순위 낮음). 2열 1순위, 세로 스택 폴백.
- **대표값 숫자**: `text-4xl`가 375px에서 넘칠 수 있으나 연차수당은 통상 수십만~수백만원(6~7자리). `flex-wrap items-baseline`로 `원` 자연 줄바꿈. 7자리(백만 단위) 값도 한 줄에 들어가는지 375px 프리뷰에서 확인.
- **결과 Tier② 행**: `flex items-center justify-between` 양끝 정렬. 긴 라벨(`시간당 통상임금`)이 값과 겹치지 않는지 375px 확인.
- **발생 연차 블록**: 근속연수 입력 시에만 추가 노출되어 결과 카드가 길어짐 → 모바일 스크롤 부담 점검. 소제목+1행+설명 1~2줄로 컴팩트 유지.

## 6. 개발팀 인계 시 주의사항

- **lib 분리 필수**: 계산은 전부 `lib/annual-leave-allowance.ts`에. 컴포넌트는 입력 수집·1차 검증·표시만. 주휴수당 lib의 `Input/Info/Amounts/Error/Outcome` 판별 유니온 구조를 그대로 계승.
- **상수 단일 소스**: `209`, `8`, 발생일수 캡(`11`, `15`, `25`), 2년 가산 주기는 **lib 상수 1곳**에서만 선언(예 `MONTHLY_STANDARD_HOURS = 209`, `DAILY_WORK_HOURS = 8`). 컴포넌트·문구에 숫자 하드코딩 금지.
- **라운딩은 최종 표시값만**: 내부는 `hourlyRaw = 월통상임금/209`, `dailyRaw = hourlyRaw × 8` 전 정밀도 유지. 표시용 시간당/1일 통상임금은 표시 시점 `Math.round`. 총액 = `Math.round(dailyRaw × 미사용일수)`.
  - **(이슈 A) 확인 필요**: 마스터 확정 문구가 "**1일 통상임금 × 미사용 연차일수**(원 단위 반올림)"인데, 여기서 "1일 통상임금"이 (a)반올림 전 `dailyRaw`인지 (b)화면에 표시되는 반올림된 1일 통상임금인지에 따라 총액이 몇 원 달라질 수 있다. 일부 실무 계산기는 **1일 통상임금을 원 단위로 먼저 반올림한 뒤 일수를 곱한다**(표시값과 총액이 정확히 일치해 사용자 신뢰↑). 디자인 권고 = 표시 일관성 위해 **(b) 방식**(1일 통상임금 원단위 반올림 후 × 일수)을 제안하나, 이는 "라운딩은 최종 표시값만" 원칙과 상충하므로 **개발/기획 확정 요청**. 어느 쪽이든 화면의 `1일 통상임금 × 미사용일수 = 총액`이 산술적으로 맞아떨어져야 한다(사용자가 암산 검산 시 어긋나면 CS 문의 발생).
- **(이슈 B) 근속연수 단위 vs 1년 미만 개월 처리**: 입력 단위는 `년`인데 발생일수 모델의 1년 미만 브랜치는 `개월`(1개월당 1일) 기준이다. 입력이 년(소수 허용)일 때 <1년 구간을 어떻게 개월로 환산할지(예 `floor(년 × 12)` 개월, 최대 11) 개발이 정해 회신 바람. 디자인 제안: 년 소수 입력 허용 → lib에서 `years >= 1 → 15+floor((years-1)/2)` cap25, `0 < years < 1 → min(floor(years*12), 11)`, `years == 0 → 0`. 발생일수 블록 설명 문구도 이 처리에 맞춰 확정.
- **(이슈 C) 상한/방어 정책**: 미사용일수·통상임금·근속연수 상한(비현실적 값 클램프 or 경고)은 엔진 사안. 디자인은 자유 입력 전제. 필요 시 상한 규칙 회신 요청(회신 오면 힌트/에러 문구 반영해 재설계).
- **`calculators.ts` 등록**: slug·카테고리(/salary)·관련계산기 3개(권장: `severance-pay-calculator`, `salary-net-calculator`, `weekly-holiday-allowance`)·interpretation·FAQ는 기획/개발이 확정. 디자인은 결과 라벨·요약 문구 초안만 제공(위 §3).
- **재사용 헬퍼**: `formatAmountInput`·`sanitizeDecimalInput`·`AmountRow`·`TextRow`는 주휴수당 컴포넌트에서 사실상 동일하게 재사용 가능(중복 선언되어 있으니, 공통 유틸로 뽑을지는 개발 재량 — 스코프 밖).

## 9. 마스터에게 보고할 이슈

1. **(개발 회신 필요) 총액 라운딩 방식 — 이슈 A**: "1일 통상임금 × 미사용일수(원 단위 반올림)"에서 1일 통상임금을 **반올림 전 값**으로 곱할지(현행 "라운딩은 표시값만" 원칙) **원단위 반올림 후** 곱할지에 따라 총액이 몇 원 달라진다. 디자인은 사용자 암산 검산이 맞아떨어지도록 **표시된 1일 통상임금 × 일수 = 총액**이 일치하는 방식(반올림 후 곱)을 권고하나, 원칙과 상충하므로 마스터/개발이 확정해 주기 바람. 확정값에 맞춰 요약 문구를 조정하겠다.

2. **(개발 회신 필요) 근속연수 입력 단위 — 이슈 B**: 입력 단위가 `년`인데 발생일수 모델의 1년 미만 브랜치는 `개월`(개근 1개월당 1일) 기준이라 단위 불일치가 있다. 년 소수 입력을 개월로 환산(`floor(년×12)` cap 11)하는 안을 제안하나, 확정 필요. 확정 전까지 발생일수 블록 설명 문구를 잠정 초안으로 둔다.

3. **(기획 확인) 근속연수 필드 존치 여부**: 이 필드는 연차수당 **금액 계산에 미참여**하는 순수 참고 안내다(주휴수당 "주 근무일수(선택)"와 동일한 성격). 반영되지 않는 입력 노출이 혼란을 준다면 제거도 대안(디자인은 제거 시 더 단순해짐 선호). 다만 "내 연차가 며칠 생기는지" 검색 의도가 커 체류·전환에는 유용하므로 **존치 + 계산 미참여 명시**로 설계했다. 기획 판단 요청.

4. **(개발 회신 필요) 상한/방어 정책 — 이슈 C**: 미사용일수·통상임금·근속연수의 비현실적 상한값 클램프/경고 정책 미정. 디자인은 자유 입력 전제. 회신 오면 힌트/에러로 반영.

5. **(마스터 확인) 톤앤매너·카테고리·스택 변경 없음**: 기존 계산기 컨벤션 100% 계승(신규 브랜딩 요소 없음)이라 별도 승인 불요로 판단. 이견 시 회신 바람.

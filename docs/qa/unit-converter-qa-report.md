## 3. 엣지케이스 입력

**PASS.**

- **빈 입력**: `parseNumericInput("")→null`, `formatResultValue(null)→"—"`. 에러 배너 없이 조용히
  처리(design §8 요구사항과 일치).
- **0**: `formatResultValue(0)→"0"` (정상, 포맷 함수의 특수분기로 "0.00" 같은 어색한 표기 방지).
- **NaN/Infinity**: 둘 다 `"—"`로 안전 처리, 화면에 `NaN`/`Infinity` 문자열이 새는 경우 없음.
- **음수 허용 여부(길이/무게/면적/부피)**: 코드에 음수 차단 로직이 전혀 없음을 확인(HTML `min`
  속성도 없고, `type="text"`라 브라우저 자체 검증도 없음). 실제 계산도 정상 수행됨:
  - `-5cm → inch = -1.97` (부호 유지, 정상 계산)
  - `-10kg → lb = -22.05`
  - `-3㎡ → 평 = -0.9075`
  - `-1L → gallon = -0.264`

  이는 design §8("물리적으로 무의미하지만 수식은 성립하므로 차단하지 않고 그대로 계산")에서 명시적
  으로 승인한 설계이며, 실제 구현이 그 설계를 정확히 따른다.
- **매우 크고 작은 값(표시 포맷)**:
  - `1mm → km = 0.000001`(초소형 유효숫자 분기, design §5-2 예시 "0.00 아님"과 정확히 일치)
  - `1mg → t = 1e-9` → 표시 `1 × 10⁻⁹`(초소형이 지수표기로 갈 정도로 작을 때 자동 과학표기 폴백,
    깨진 문자열 없이 정상)
  - `1e15 m → mm = 1e18` → 표시 `1 × 10¹⁸`(초대형 폴백, `abs ≥ 1e12` 경계 정확히 동작:
    `999,999,999,999`는 콤마 표기, `1,000,000,000,000`은 즉시 과학표기로 전환됨을 개별 확인)
  - 음수 극단값(`-1e15`, `-1mg→t`)도 부호가 유지된 채 과학표기로 정상 표시(`-1 × 10¹⁸`,
    `-1 × 10⁻⁹`), 깨짐 없음.
- **절대영도 미만 온도 안내 문구**: 코드상 `belowAbsoluteZero` 조건이 참일 때
  `"물리적으로 불가능한 온도입니다 (절대영도 −273.15℃ 미만)."` 문구가 렌더되는 조건부 JSX를 확인.
  단, 이 상태는 기본 진입 상태(면적 카테고리)에서는 트리거되지 않으므로 curl로 뜨는 SSR 기본 HTML
  에는 나타나지 않는다 — **실제 사용자 입력에 따른 렌더링은 코드 로직 검증으로 대체했으며 실브라우저
  상호작용으로는 확인하지 못했다**(§5 조건부 PASS와 동일 사유).
- **왕복 스왑 드리프트**: `20평 → 66.11570247933885㎡(raw) → 되돌리면 20평` 정확히 복원(오차
  <1e-9). `33.8℉ → 0.9999999999999984℃(raw) → 되돌리면 33.8℉` 정확히 복원. 스왑 시
  `String(rawResult)`로 표시용 반올림 값이 아니라 full-precision 값을 되먹인다는 코드 설명대로
  실제 동작하며, `Number(String(x)) === x` round-trip도 정밀도 손실 없이 확인됨(JS 부동소수 문자열
  변환의 spec 보장 특성과 일치).

## 5. 모바일/데스크톱 화면 — 조건부 PASS (실브라우저 미확인)

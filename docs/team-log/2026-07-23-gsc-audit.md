## 5. ★마스터 자기 오류 1건 (즉시 자가 검출·교정) — 바이트 vs 문자 길이
- 1차 desc/title 길이 집계에서 `awk length`·bash `${#var}`·`wc -m`을 썼는데 **43개 URL이 desc 160자 초과, 최대 485**로 나와 07-22(9개/최대 206)와 크게 어긋났다.
- **거짓 회귀를 보고하기 전에 계측기를 먼저 의심**(표준 ⓔ) → 알려진 4글자 한글 문자열 `주휴수당`으로 캘리브레이션: 이 환경은 **로케일 미설정(C locale)이라 bash·wc·awk가 전부 바이트를 센다**(한글 1자 = UTF-8 3바이트, `주휴수당`=12). 즉 485는 바이트, 실제 문자수는 ≈160.
- **python `len()`으로 재측정** → 최대 206, >160 9개, <70 16개로 **07-22와 정확히 일치**. 거짓 회귀 방지.
- **교훈(운영 표준 ⓘ 신규)**: 이 Git Bash 환경은 C 로케일이라 `wc -m`·`${#}`·`awk length`가 **멀티바이트 텍스트에서 문자수가 아니라 바이트수**를 센다. 한글 포함 길이 계측은 **반드시 python `len()`으로** 하고, `wc -m`을 믿지 말 것. (표준 ⓗ의 연장 — 사람이 읽는 텍스트 집계의 함정.)
